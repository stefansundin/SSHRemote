## SSH Remote बारेमा

यो अनुवाद GitHub Copilot (GPT-5.3-Codex) द्वारा गरिएको हो। यसमा अनुवादसम्बन्धी त्रुटिहरू हुन सक्छन्।

SSH Remote एउटा निःशुल्क र खुला स्रोत एप हो जसले तपाईंलाई SSH प्रयोग गरेर टाढाबाट कम्प्युटरहरू नियन्त्रण गर्न दिन्छ।

चलाइने आदेशहरू तपाईंले पूर्ण रूपमा अनुकूल बनाउन सक्नुहुन्छ, र सामान्य सेटअपहरूका लागि प्रिसेटहरू पनि उपलब्ध छन्।

म यो एप मेरो HTPC सेटअप नियन्त्रण गर्न प्रयोग गर्छु, जुन Raspberry Pi OS मा चलिरहेको छ। HTPC नियन्त्रण गर्नु यो एप अनुकूलित गरिएको मुख्य परिदृश्य हो।

यो एप टर्मिनल इमुलेटर होइन, तर आपतकालीन अवस्थामा यसले तपाईंलाई `apt-get install` चलाउन दिन्छ।

## सुरु गर्ने तरिका

यदि तपाईं जडान गर्न SSH कुञ्जी प्रयोग गर्न चाहनुहुन्छ भने, पहिले एपका सेटिङहरू खोल्नुहोस् र कुञ्जी आयात गर्नुहोस् वा उत्पन्न गर्नुहोस्।

तल दायाँ कुनामा रहेको `+` बटन थिचेर नयाँ होस्ट थप्नुहोस्। जडान विवरणहरू प्रविष्ट गर्नुहोस् र सेभ गर्नुहोस्।

पहिलो पटक जडान गर्दा, तपाईंलाई एउटा प्रिसेट चयन गर्न भनिनेछ। यो चयनले रिमोट कन्ट्रोल बटनहरूलाई विभिन्न प्रकारका कम्प्युटरहरूसँग राम्रोसँग काम गर्ने गरी कन्फिगर गर्छ। उपलब्ध प्रिसेटहरूको विवरणका लागि तल हेर्नुहोस्। यदि तपाईं चाहनुहुन्छ भने, `No preset` चयन गरेर कुनै कन्फिगरेसन बिना पनि सुरु गर्न सक्नुहुन्छ।

यदि तपाईंलाई तपाईंको Linux कम्प्युटर X11 चलिरहेको छ वा Wayland, थाहा छैन भने, टर्मिनलमा यो चलाउनुहोस्:

```shell
echo $XDG_SESSION_TYPE
```

यसले `x11` वा `wayland` आउटपुट दिनुपर्छ। तपाईंले यो डेस्कटप वातावरणभित्रै चलाउनुपर्छ।

## रिमोट कन्ट्रोल

जडान भएपछि, तपाईं आदेशहरू पठाउन रिमोट कन्ट्रोल इन्टरफेस प्रयोग गर्न सक्नुहुन्छ। विभिन्न इनपुट विधिहरूमा पहुँच गर्न ट्याबहरू बदल्नुहोस्।

प्रत्येक बटन थिच्दा होस्टमा एउटा आदेश चलाइन्छ। साधारण कुञ्जी थिचाइ जस्तो कुराका लागि पनि यो धेरै ओभरहेड हो, र साधारण किबोर्डको तुलनामा तपाईंले धेरै ढिलाइ अनुभव गर्न सक्नुहुन्छ। भविष्यका संस्करणहरूमा यसलाई सुधार्ने आशा राख्छु।

सम्पादन मोडमा जान मेनु प्रयोग गर्नुहोस्। हाल लेआउट वा बटन आइकनहरू सम्पादन गर्न सम्भव छैन। भविष्यको संस्करणमा यो सम्भव बनाउने आशा राख्छु।

## प्रिसेटहरू

तपाईंको डेस्कटप वातावरणका लागि आवश्यक टूल स्थापना गर्नुपर्नेछ।

मेरो सिफारिस `ydotool` हो, किनभने मेरो परीक्षणमा यसको कार्यसम्पादन सबैभन्दा राम्रो थियो, र यसले X11 र Wayland दुवैमा काम गर्छ।

### ydotool

`ydotool` ले कुनै पनि विन्डो प्रबन्धकमा काम गर्नुपर्छ, तर तपाईंलाई पृष्ठभूमिमा चलिरहेको सेवा चाहिन्छ। यदि तपाईंको वितरणले systemd user service उपलब्ध गराउँछ भने, यसलाई यसो गरेर सुरु गर्नुहोस्:

```shell
systemctl start --user ydotool
```

लगइन गर्दा सेवा स्वतः सुरु गर्न, यो चलाउनुहोस्:

```shell
systemctl enable --user ydotool
```

कृपया तपाईंले पर्याप्त नयाँ संस्करणको `ydotool` स्थापना गरिरहनुभएको छ भन्ने सुनिश्चित गर्नुहोस्। Ubuntu 26.04 भन्दा अघिका संस्करणहरूले धेरै पुराना संस्करणहरू उपलब्ध गराउँछन्। समाधानका लागि discussion thread हेर्नुहोस्।

कृपया `ydotool` सम्बन्धी छलफलका लागि यो discussion thread प्रयोग गर्नुहोस्: <https://github.com/stefansundin/SSHRemote/discussions/5>

### xdotool

`xdotool` X11 चलिरहेका कम्प्युटरहरूका लागि हो। X11 इतिहासमा धेरैजसो Linux कम्प्युटरहरूले प्रयोग गरेको प्रणाली हो, यद्यपि Wayland अहिले बढी लोकप्रिय हुँदैछ।

X11 को एउटा विशेषता के हो भने तपाईंले X server मा पहुँच अनुमति दिनुपर्ने हुन सक्छ। तपाईंले `Authorization required` त्रुटि पाएमा समस्या यही हुन सक्छ। यो समस्या समाधान गर्न धेरै विकल्पहरू छन्, यहाँ मेरो लागि काम गरेका दुई विकल्पहरू छन्:

यदि `xauth list` ले कुनै पनि प्रविष्टि नदेखाएमा, `.Xauthority` फाइल उत्पन्न गर्ने प्रयास गर्नुहोस्:

```shell
xauth generate :0 . trusted
```

यदि त्यसले काम गरेन भने, `xhost` प्रयोग गरेर पहुँच दिनुहोस्:

```shell
xhost +local:$USER
```

तपाईंले प्रत्येक बुटपछि `xhost` आदेश चलाउनुपर्छ। यसलाई स्वचालित बनाउन bash script बनाएर लगइनमा autostart हुने गरी कन्फिगर गर्न सक्नुहुन्छ।

कृपया `xdotool` सम्बन्धी छलफलका लागि यो discussion thread प्रयोग गर्नुहोस्: <https://github.com/stefansundin/SSHRemote/discussions/3>

### wtype

`wtype` `xdotool` जस्तै हो, तर Wayland का लागि।

सामान्यतया यसले माउस नियन्त्रण समर्थन गर्दैन, तर मैले एउटा परिमार्जित संस्करण बनाएको छु जसले माउस समर्थन थप्छ। यदि तपाईंलाई माउस समर्थन चाहिन्छ भने कृपया यसलाई स्थापना गर्नुहोस्: <https://github.com/stefansundin/wtype>

यदि तपाईंले `Compositor does not support the virtual keyboard protocol` त्रुटि पाउनुभयो भने, म तपाईंलाई `ydotool` जस्तो अर्को टूल प्रयास गर्न सुझाव दिन्छु।

कृपया `wtype` सम्बन्धी छलफलका लागि यो discussion thread प्रयोग गर्नुहोस्: <https://github.com/stefansundin/SSHRemote/discussions/4>

### dotool

`dotool` ले `ydotool` जस्तै कुनै पनि विन्डो प्रबन्धकमा काम गर्नुपर्छ। मेरो सीमित परीक्षणमा यो `ydotool` भन्दा धेरै ढिलो थियो।

कृपया `dotool` सम्बन्धी छलफलका लागि यो discussion thread प्रयोग गर्नुहोस्: <https://github.com/stefansundin/SSHRemote/discussions/6>

### cec-client

यो प्रिसेट प्रयोगात्मक छ किनभने मैले यसलाई आफ्नै हार्डवेयरमा पुष्टि गर्न सकेको छैन। प्रतिक्रिया स्वागत छ।

कृपया `cec-client` सम्बन्धी छलफलका लागि यो discussion thread प्रयोग गर्नुहोस्: <https://github.com/stefansundin/SSHRemote/discussions/7>

### macOS

macOS मा VLC नियन्त्रण गर्न AppleScript आदेशहरू प्रयोग गर्ने एउटा प्रिसेट उपलब्ध छ। किबोर्ड वा माउस इभेन्ट पठाउन समर्थन गर्ने कुनै टूलबारे मलाई थाहा छैन।

कृपया macOS सम्बन्धी छलफलका लागि यो discussion thread प्रयोग गर्नुहोस्: <https://github.com/stefansundin/SSHRemote/discussions/8>

### Android

यदि तपाईंको Android यन्त्रमा SSH server छ भने, तपाईं त्यसमा जडान गरेर input events पठाउन सक्नुहुन्छ। तपाईंको यन्त्र Raspberry Pi का लागि KonstaKANG जस्तो custom ROM मा चलिरहेको छ भने, यसको सम्भावना बढी हुन्छ।

माउस समर्थन काम गराउने तरिका मैले अझै पत्ता लगाउन सकेको छैन।

कृपया Android सम्बन्धी छलफलका लागि यो discussion thread प्रयोग गर्नुहोस्: <https://github.com/stefansundin/SSHRemote/discussions/9>

## स्मार्ट भोल्युम नियन्त्रण

रिमोट कन्ट्रोल सम्पादन गर्दा, मेनुमा तपाईंले “smart” volume control सेटिङहरू फेला पार्न सक्नुहुन्छ। यसले एपमा कम्प्युटरको हालको भोल्युम देखाउन सक्छ र स्लाइडर प्रयोग गरेर छिट्टै भोल्युम सेट गर्न दिन्छ। तपाईंले आफ्नो यन्त्रका हार्डवेयर बटनहरू प्रयोग गरेर पनि भोल्युम बढाउने/घटाउने आदेशहरू छिट्टै पठाउन सक्नुहुन्छ।

हालको भोल्युम पढ्ने र स्लाइडर प्रयोग गरेर नयाँ भोल्युम सेट गर्ने काम अहिले `pactl` प्रयोग गर्ने गरी hard-coded गरिएको छ।

`pactl` समावेश भएको प्याकेजलाई प्रायः `pulseaudio-utils` वा `libpulse` भनिन्छ।

## SSH कुञ्जीहरू

तपाईं एपका सेटिङहरूमा SSH कुञ्जीहरू आयात वा उत्पन्न गर्न सक्नुहुन्छ। पासवर्डको सट्टा SSH कुञ्जी प्रयोग गरेर जडान गर्नु बढी सुरक्षित हुन्छ।

कम्प्युटरबाट अवस्थित SSH कुञ्जी आयात गर्ने सबैभन्दा सजिलो तरिका QR कोड स्क्यान गर्नु हो। QR कोड छवि बनाउन तपाईं `qrencode` प्रोग्राम प्रयोग गर्न सक्नुहुन्छ। QR कोड उत्पन्न गर्न तलजस्तो आदेश चलाउनुहोस्:

```shell
# आफ्ना SSH कुञ्जीहरू भएको स्थानमा जानुहोस्:
cd ~/.ssh

# टर्मिनलमा QR कोड देखाउनुहोस्:
qrencode -r id_ed25519 -t ansiutf8

# वैकल्पिक रूपमा, एउटा छवि फाइल बनाउनुहोस्:
qrencode -r id_ed25519 -o qr.png

# 4096-bit RSA कुञ्जीहरू QR कोडमा अटाउन धेरै ठूला हुन्छन्। एउटा अटाउन जोगाउन तपाईं gzip प्रयोग गर्न सक्नुहुन्छ:
cat id_rsa_4096 | gzip | base64 | qrencode -o qr.png
```

तपाईं मेनुको `सार्वजनिक कुञ्जी पठाउनुहोस्` सुविधा प्रयोग गरेर server मा सार्वजनिक SSH कुञ्जीहरू पठाउन सक्नुहुन्छ। यसले चयन गरिएको SSH कुञ्जीलाई `~/.ssh/authorized_keys` फाइलमा append गर्नेछ। यसले पासवर्ड प्रयोग गरेर लगइन गर्ने अवस्थाबाट SSH कुञ्जी प्रयोग गरेर लगइन गर्ने अवस्थामा सजिलै सर्न मद्दत गर्छ।

तपाईं इन्क्रिप्ट गरिएका SSH कुञ्जीहरू आयात र प्रयोग गर्न सक्नुहुन्छ, तर अहिले एपभित्र यस्ता कुञ्जीहरू उत्पन्न गर्न सकिँदैन।

## सुरक्षा

एपबाट SSH कुञ्जीहरूको निजी भाग वा सुरक्षित गरिएका पासवर्डहरू निर्यात वा निकाल्न सम्भव छैन। यो डाटा 256-bit AES प्रयोग गरेर इन्क्रिप्ट गरिएको हुन्छ, र इन्क्रिप्सन कुञ्जी Android Keystore मा भण्डारण हुन्छ। इन्क्रिप्ट गरिएको डाटा Android ब्याकअपबाट बाहिर राखिन्छ।

यो एपमा कुनै crash reporting software छैन। कुनै telemetry छैन। कुनै विज्ञापन छैन। SSH जडान बाहेक अन्य कुनै network request हुँदैन।

यो एपको सुरक्षाको अडिट गरिएको छैन। यदि तपाईं Android सुरक्षा वा SSH सुरक्षामा अनुभवी हुनुहुन्छ भने, कृपया स्रोत कोड हेरेर आफ्नो निष्कर्ष यो GitHub issue मा रिपोर्ट गर्नुहोस्:

<https://github.com/stefansundin/SSHRemote/issues/1>
