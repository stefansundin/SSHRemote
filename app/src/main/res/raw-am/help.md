## ስለ SSH Remote

ይህ ትርጉም በAI (GitHub Copilot) ተዘጋጅቷል፣ ስለዚህ አንዳንድ የትርጉም ስህተቶች ሊኖሩ ይችላሉ።

SSH Remote በSSH በመጠቀም ኮምፒውተሮችን ከርቀት ለመቆጣጠር የሚያስችል ነፃ እና ክፍት ምንጭ መተግበሪያ ነው።

የሚፈጸሙትን ትእዛዞች ሙሉ በሙሉ ማበጀት ይችላሉ፣ እንዲሁም ለተለመዱ አቀማመጦች ቅድመ-ቅንብሮች አሉ።

እኔ ይህን መተግበሪያ በRaspberry Pi OS የሚሰራ የHTPC አቀማመጤን ለመቆጣጠር እጠቀማለሁ። HTPC መቆጣጠር መተግበሪያው በዋናነት የተመቻቸው ሁኔታ ነው።

ይህ መተግበሪያ ተርሚናል ኢሙሌተር አይደለም፣ ግን በአስቸኳይ ሁኔታ `apt-get install` እንዲያስኬዱ ያስችላል።

## መጀመር

ለመገናኘት SSH ቁልፍ መጠቀም ከፈለጉ መጀመሪያ የመተግበሪያውን ቅንብሮች ይክፈቱ እና ቁልፍ ያስመጡ ወይም ይፍጠሩ።

አዲስ አስተናጋጅ ለመጨመር በታች ቀኝ ጥግ ያለውን `+` አዝራር ይንኩ። የግንኙነት ዝርዝሮችን ያስገቡ እና ያስቀምጡ።

ለመጀመሪያ ጊዜ ሲገናኙ ቅድመ-ቅንብር እንዲመርጡ ይጠየቃሉ። ይህ ምርጫ የርቀት መቆጣጠሪያ አዝራሮች በተለያዩ የኮምፒውተር አይነቶች ላይ በጥሩ ሁኔታ እንዲሰሩ ያዋቅራል። ከታች ያሉትን ቅድመ-ቅንብሮች መግለጫ ይመልከቱ። ከፈለጉ `No preset` በመምረጥ ያለ ማዋቀር መጀመር ይችላሉ።

የLinux ኮምፒውተርዎ X11 ወይም Wayland ላይ እንደሚሰራ ካላወቁ በተርሚናል ይህን ያስኬዱ:

```shell
echo $XDG_SESSION_TYPE
```

ውጤቱ `x11` ወይም `wayland` መሆን አለበት። ይህን ትእዛዝ በዴስክቶፕ አካባቢ ውስጥ ማስኬድ አለብዎት።

## የርቀት መቆጣጠሪያ

ከተገናኙ በኋላ ትእዛዞችን ለመላክ የርቀት መቆጣጠሪያ በይነገጽን መጠቀም ይችላሉ። የተለያዩ የግቤት ዘዴዎችን ለመድረስ ትሮችን ይቀያይሩ።

እያንዳንዱ የአዝራር ግፊት በአስተናጋጁ ላይ ትእዛዝ ያስኬዳል። ለቁልፍ ግፊት ያህል ቀላል ነገር ብዙ ተጨማሪ ሂደት ስለሚፈልግ ከመደበኛ የቁልፍ ሰሌዳ ጋር ሲነጻጸር ከፍ ያለ የመዘግየት ጊዜ ሊሰማዎት ይችላል። ይህን በወደፊት ስሪቶች ለማሻሻል ተስፋ አደርጋለሁ።

የአርትዖት ሁኔታ ለመግባት ምናሌውን ይጠቀሙ። በአሁኑ ጊዜ የአቀማመጥ ወይም የአዝራር አዶዎችን ማርትዕ አይቻልም። ይህን በወደፊት ስሪት ማስቻል እመኛለሁ።

## ቅድመ-ቅንብሮች

ለዴስክቶፕ አካባቢዎ የሚፈለገውን መሣሪያ መጫን ያስፈልጋል።

በሙከራዬ መሰረት ምርጥ አፈጻጸም ስላለው እና በX11 እና Wayland ሁለቱም ላይ ስለሚሰራ `ydotool` እመክራለሁ።

### ydotool

`ydotool` ከማንኛውም የመስኮት አስተዳዳሪ ጋር መስራት አለበት፣ ግን በጀርባ የሚሰራ አገልግሎት ያስፈልጋል። የእርስዎ ስርጭት systemd የተጠቃሚ አገልግሎት ካቀረበ ይህን በማስኬድ ይጀምሩ:

```shell
systemctl start --user ydotool
```

ሲገቡ አገልግሎቱ በራስ-ሰር እንዲጀምር ይህን ያስኬዱ:

```shell
systemctl enable --user ydotool
```

በቂ አዲስ የሆነ የ`ydotool` ስሪት እየጫኑ መሆኑን እባክዎ ያረጋግጡ። ከ26.04 በፊት ያሉ የUbuntu ስሪቶች በጣም አሮጌ ስሪቶችን ያቀርባሉ። መፍትሔ ለማግኘት የውይይት ክርክሩን ይመልከቱ።

`ydotool` ስለ ውይይት እባክዎ ይህን ክር ይጠቀሙ: <https://github.com/stefansundin/SSHRemote/discussions/5>

### xdotool

`xdotool` በX11 ላይ ለሚሰሩ ኮምፒውተሮች ነው። X11 በታሪክ ብዙ የLinux ኮምፒውተሮች የተጠቀሙት ሲሆን Wayland ግን እየታወቀ እየጨመረ ነው።

ከX11 ጋር አንድ ልዩ ነገር ወደ X server መድረስ ፍቃድ መስጠት ሊፈልግ ይችላል። "Authorization required" የሚሉ ስህተቶች ከተቀበሉ ይህ ችግር ነው ብዙውን ጊዜ። ይህን ለመፍታት ጥቂት አማራጮች አሉ፤ ለእኔ የሰሩ ሁለት አማራጮች እነሆ:

`xauth list` ምንም መግቢያ ካላሳየ `.Xauthority` ፋይል ለመፍጠር ይህን ይሞክሩ:

```shell
xauth generate :0 . trusted
```

ያ ካልሰራ በ`xhost` ፍቃድ ለመስጠት ይሞክሩ:

```shell
xhost +local:$USER
```

`xhost` ትእዛዙን ከእያንዳንዱ ቡት በኋላ ማስኬድ ያስፈልጋል። ይህን በbash ስክሪፕት በመፍጠር እና በመግቢያ ላይ በራስ-ሰር እንዲጀምር በማዋቀር ማስራት ይችላሉ።

`xdotool` ስለ ውይይት እባክዎ ይህን ክር ይጠቀሙ: <https://github.com/stefansundin/SSHRemote/discussions/3>

### wtype

`wtype` እንደ `xdotool` ነው፣ ግን ለWayland ነው።

በመደበኛ ሁኔታ የmouse መቆጣጠር አይደግፍም፣ ግን የmouse ድጋፍ የሚጨምር የተሻሻለ ስሪት ፈጥራለሁ። የmouse ድጋፍ ከፈለጉ እባክዎ ይህን ይጫኑ: <https://github.com/stefansundin/wtype>

`Compositor does not support the virtual keyboard protocol` ስህተት ካገኙ እንደ `ydotool` ያለ ሌላ መሣሪያ እንዲሞክሩ እመክራለሁ።

`wtype` ስለ ውይይት እባክዎ ይህን ክር ይጠቀሙ: <https://github.com/stefansundin/SSHRemote/discussions/4>

### dotool

`dotool` እንደ `ydotool` ከማንኛውም የመስኮት አስተዳዳሪ ጋር መስራት አለበት። በአነስተኛ ሙከራዬ ውስጥ ግን ከ`ydotool` በጣም ዝግ ነበር።

`dotool` ስለ ውይይት እባክዎ ይህን ክር ይጠቀሙ: <https://github.com/stefansundin/SSHRemote/discussions/6>

### cec-client

ይህ ቅድመ-ቅንብር ሙከራዊ ነው፣ ምክንያቱም በራሴ ሃርድዌር ላይ ማረጋገጥ አልቻልኩም። አስተያየት በደህና ይቀበላል።

`cec-client` ስለ ውይይት እባክዎ ይህን ክር ይጠቀሙ: <https://github.com/stefansundin/SSHRemote/discussions/7>

### macOS

በmacOS ላይ VLC ለመቆጣጠር በAppleScript ትእዛዞች የሚሰራ ቅድመ-ቅንብር አለ። የቁልፍ ሰሌዳ ወይም የmouse ክስተቶችን መላክ የሚደግፍ መሣሪያ አላውቅም።

macOS ስለ ውይይት እባክዎ ይህን ክር ይጠቀሙ: <https://github.com/stefansundin/SSHRemote/discussions/8>

### Android

የAndroid መሣሪያዎ SSH server ካለው ከእሱ ጋር መገናኘት እና የግቤት ክስተቶችን መላክ ሊቻል ይችላል። መሣሪያዎ እንደ Raspberry Pi ላይ KonstaKANG ያለ ብጁ ROM ካለበት ይህ እድል የበለጠ ነው።

የmouse ድጋፍ እንዴት እንደሚሰራ ገና አልቻልኩም።

Android ስለ ውይይት እባክዎ ይህን ክር ይጠቀሙ: <https://github.com/stefansundin/SSHRemote/discussions/9>

## ስማርት የድምጽ መቆጣጠሪያ

የርቀት መቆጣጠሪያውን ሲያርትዑ በምናሌው ውስጥ "smart" የድምጽ መቆጣጠሪያ ቅንብሮችን ማግኘት ይችላሉ። ይህ የኮምፒውተሩን የአሁኑ ድምጽ መጠን በመተግበሪያው ላይ ሊያሳይ ይችላል፣ እንዲሁም ድምጹን በፍጥነት በslider አማካኝነት ማስተካከል ያስችላል። በተጨማሪም በመሣሪያዎ ሃርድዌር የድምጽ አዝራሮች በፍጥነት ድምጽ መቀነስ/መጨመር ትእዛዞችን ለመላክ ያስችላል። የአሁኑ ድምጽ ማንበብ እና በslider አማካኝነት አዲስ ድምጽ ማስተካከል በአሁኑ ጊዜ `pactl` ላይ ብቻ በhard-code ተደርጓል።

`pactl` ያለው ፓኬጅ ብዙ ጊዜ `pulseaudio-utils` ወይም `libpulse` ተብሎ ይጠራል።

## SSH ቁልፎች

በመተግበሪያው ቅንብሮች ውስጥ SSH ቁልፎችን ማስመጣት ወይም መፍጠር ይችላሉ። በSSH ቁልፍ መገናኘት ከይለፍ ቃል በላይ የበለጠ ደህንነት አለው።

ነባር SSH ቁልፍን ከኮምፒውተር ለማስመጣት ቀላሉ መንገድ QR ኮድ መቃኘት ነው። የQR ኮድ ምስል ለመፍጠር `qrencode` ፕሮግራም መጠቀም ይችላሉ። የQR ኮድ ለመፍጠር ከዚህ ዓይነት ትእዛዝ ያስኬዱ:

```shell
# ወደ SSH ቁልፎችዎ ይሂዱ:
cd ~/.ssh

# QR ኮዱን በተርሚናል ውስጥ ያሳዩ:
qrencode -r id_ed25519 -t ansiutf8

# ወይም የምስል ፋይል ይፍጠሩ:
qrencode -r id_ed25519 -o qr.png

# 4096-bit RSA ቁልፎች ለQR ኮድ በጣም ትልቅ ናቸው። አንዱን በጭንቅ ለማስገባት gzip መጠቀም ይችላሉ:
cat id_rsa_4096 | gzip | base64 | qrencode -o qr.png
```

በምናሌው ውስጥ ያለውን `የህዝብ ቁልፍ ግፋ` ባህሪ ተጠቅመው የህዝብ SSH ቁልፎችን ወደ ሰርቨር መግፋት ይችላሉ። ይህ የተመረጠውን SSH ቁልፍ ወደ `~/.ssh/authorized_keys` ፋይል መጨመር ያደርጋል። ይህ ከይለፍ ቃል መግቢያ ወደ SSH ቁልፍ መግቢያ በቀላሉ ለመቀየር ያግዛል።

የተመሰጠሩ SSH ቁልፎችን ማስመጣት እና መጠቀም ይችላሉ፣ ነገር ግን በመተግበሪያው ውስጥ አሁን መፍጠር አይቻልም።

## ደህንነት

የSSH ቁልፎች የግል ክፍል ወይም የተቀመጡ የይለፍ ቃላትን ከመተግበሪያው ማውጣት ወይም መቀየር አይቻልም። ይህ ውሂብ በ256-bit AES ተመስጥሯል፣ የምስጠራ ቁልፉም በAndroid Keystore ውስጥ ይቀመጣል። የተመሰጠረ ውሂብ ከAndroid ባክአፕ ውጪ ነው።

በዚህ መተግበሪያ ውስጥ የcrash ሪፖርት ሶፍትዌር የለም። telemetry የለም። ማስታወቂያ የለም። ከSSH ግንኙነት በስተቀር የኔትወርክ ጥያቄዎች የሉም።

የዚህ መተግበሪያ ደህንነት ኦዲት አልተደረገበትም። በAndroid ወይም SSH ደህንነት ልምድ ካለዎት ምንጭ ኮዱን ይመልከቱ እና የተገኙትን ጉዳዮች በዚህ GitHub issue ላይ ያሳውቁ:

<https://github.com/stefansundin/SSHRemote/issues/1>
