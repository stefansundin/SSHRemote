## SSH Remote کے بارے میں

یہ ترجمہ AI کی مدد سے تیار کیا گیا ہے، اور اس میں ترجمے کی کچھ غلطیاں موجود ہو سکتی ہیں. یہ ترجمہ GitHub Copilot (GPT-5.3-Codex) کے ذریعے بنایا گیا ہے۔

SSH Remote ایک مفت اور اوپن سورس ایپ ہے جو آپ کو SSH کے ذریعے کمپیوٹرز کو دور سے کنٹرول کرنے کی اجازت دیتی ہے۔

آپ چلائی جانے والی کمانڈز کو مکمل طور پر اپنی مرضی کے مطابق بنا سکتے ہیں، اور عام سیٹ اپس کے لیے پری سیٹس بھی موجود ہیں۔

میں اس ایپ کو اپنے HTPC سیٹ اپ کو کنٹرول کرنے کے لیے استعمال کرتا ہوں، جو Raspberry Pi OS پر چل رہا ہے۔ HTPC کو کنٹرول کرنا وہ بنیادی منظرنامہ ہے جس کے لیے یہ ایپ خاص طور پر بہتر بنائی گئی ہے۔

یہ ایپ ٹرمینل ایمولیٹر نہیں ہے، لیکن ہنگامی صورت میں یہ آپ کو `apt-get install` چلانے دے گی۔

## شروعات

اگر آپ کنیکٹ ہونے کے لیے SSH کلید استعمال کرنا چاہتے ہیں، تو پہلے ایپ کی ترتیبات کھولیں اور ایک کلید درآمد کریں یا بنائیں۔

نیچے دائیں جانب موجود `+` بٹن دبا کر ایک نیا ہوسٹ شامل کریں۔ کنکشن کی تفصیلات درج کریں اور محفوظ کریں۔

پہلی بار کنیکٹ ہونے پر، آپ سے ایک پری سیٹ منتخب کرنے کو کہا جائے گا۔ یہ انتخاب ریموٹ کنٹرول کے بٹنوں کو اس طرح ترتیب دیتا ہے کہ وہ مختلف قسم کے کمپیوٹرز کے ساتھ بہتر کام کریں۔ دستیاب پری سیٹس کی تفصیل کے لیے نیچے دیکھیں۔ اگر آپ چاہیں، تو `No preset` منتخب کر کے بغیر کسی ترتیب کے بھی آغاز کر سکتے ہیں۔

اگر آپ کو معلوم نہیں کہ آپ کا Linux کمپیوٹر X11 چلا رہا ہے یا Wayland، تو ٹرمینل میں یہ چلائیں:

```shell
echo $XDG_SESSION_TYPE
```

اس کا نتیجہ `x11` یا `wayland` ہونا چاہیے۔ یہ کمانڈ آپ کو ڈیسک ٹاپ ماحول کے اندر چلانی ہوگی۔

## ریموٹ کنٹرول

کنیکٹ ہونے کے بعد، آپ کمانڈز بھیجنے کے لیے ریموٹ کنٹرول انٹرفیس استعمال کر سکتے ہیں۔ مختلف ان پٹ طریقوں تک رسائی کے لیے ٹیبز تبدیل کریں۔

ہر بٹن دبانے پر ہوسٹ پر ایک کمانڈ چلتی ہے۔ کسی کلید کو دبانے جیسے سادہ کام کے لیے یہ کافی اضافی عمل ہے، اور اس وجہ سے آپ کو عام کی بورڈ کے مقابلے میں خاصی تاخیر محسوس ہو سکتی ہے۔ مجھے امید ہے کہ آئندہ ورژنز میں اسے بہتر بنایا جا سکے گا۔

ایڈیٹ موڈ میں جانے کے لیے مینو استعمال کریں۔ فی الحال لے آؤٹ یا بٹن آئیکنز میں ترمیم کرنا ممکن نہیں ہے۔ مجھے امید ہے کہ آئندہ ورژنز میں یہ ممکن ہو جائے گا۔

## پری سیٹس

آپ کو اپنے ڈیسک ٹاپ ماحول کے لیے درکار ٹول انسٹال کرنا ہوگا۔

میری سفارش `ydotool` ہے کیونکہ میری جانچ کے مطابق اس کی کارکردگی سب سے بہتر ہے، اور یہ X11 اور Wayland دونوں پر کام کرتا ہے۔

### ydotool

`ydotool` کسی بھی ونڈو مینیجر کے ساتھ کام کر سکتا ہے، لیکن اس کے لیے ایک بیک گراؤنڈ سروس کا چلنا ضروری ہے۔ اگر آپ کی ڈسٹری بیوشن systemd user service فراہم کرتی ہے تو اسے یہ چلا کر شروع کریں:

```shell
systemctl start --user ydotool
```

لاگ اِن پر سروس کو خودکار طور پر شروع کرنے کے لیے یہ چلائیں:

```shell
systemctl enable --user ydotool
```

براہ کرم یقینی بنائیں کہ آپ `ydotool` کا کافی نیا ورژن انسٹال کر رہے ہیں۔ Ubuntu 26.04 سے پہلے کے ورژنز میں بہت پرانا ورژن موجود ہوتا ہے۔ متبادل حل کے لیے discussion thread دیکھیں۔

براہ کرم `ydotool` پر اس discussion thread میں گفتگو کریں: <https://github.com/stefansundin/SSHRemote/discussions/5>

### xdotool

`xdotool` اُن کمپیوٹرز کے لیے ہے جو X11 چلا رہے ہیں۔ تاریخی طور پر زیادہ تر Linux کمپیوٹر X11 استعمال کرتے رہے ہیں، اگرچہ Wayland اب زیادہ مقبول ہو رہا ہے۔

X11 کی ایک خاص بات یہ ہے کہ آپ کو کبھی کبھی X server تک رسائی دینی پڑ سکتی ہے۔ اگر آپ کو `Authorization required` کی خرابی ملتی ہے تو مسئلہ یہی ہے۔ اس مسئلے کے حل کے لیے آپ کے پاس چند اختیارات ہیں، ان میں سے دو جو میرے لیے کام کر چکے ہیں یہ ہیں:

اگر `xauth list` کوئی اندراجات نہیں دکھاتا تو `.Xauthority` فائل بنانے کی کوشش کریں:

```shell
xauth generate :0 . trusted
```

اگر اس سے مسئلہ حل نہ ہو تو `xhost` کے ذریعے رسائی دینے کی کوشش کریں:

```shell
xhost +local:$USER
```

آپ کو `xhost` کمانڈ ہر بوٹ کے بعد چلانی ہوگی۔ آپ ایک bash script بنا کر اور اسے لاگ اِن پر خودکار طور پر چلانے کے لیے ترتیب دے کر اس عمل کو خودکار بنا سکتے ہیں۔

براہ کرم `xdotool` پر اس discussion thread میں گفتگو کریں: <https://github.com/stefansundin/SSHRemote/discussions/3>

### wtype

`wtype`، `xdotool` جیسا ہے، مگر Wayland کے لیے۔

عام طور پر یہ ماؤس کنٹرول کی معاونت نہیں کرتا، لیکن میں نے اس کا ایک ترمیم شدہ ورژن بنایا ہے جو ماؤس سپورٹ شامل کرتا ہے۔ اگر آپ کو ماؤس سپورٹ چاہیے تو براہ کرم اسے انسٹال کریں: <https://github.com/stefansundin/wtype>

اگر آپ کو `Compositor does not support the virtual keyboard protocol` کی خرابی ملے تو میری تجویز ہے کہ آپ کوئی اور ٹول آزمائیں، جیسے `ydotool`۔

براہ کرم `wtype` پر اس discussion thread میں گفتگو کریں: <https://github.com/stefansundin/SSHRemote/discussions/4>

### dotool

`dotool`، `ydotool` کی طرح، کسی بھی ونڈو مینیجر کے ساتھ کام کر سکتا ہے۔ میری محدود جانچ میں یہ `ydotool` کے مقابلے میں کافی سست تھا۔

براہ کرم `dotool` پر اس discussion thread میں گفتگو کریں: <https://github.com/stefansundin/SSHRemote/discussions/6>

### cec-client

یہ پری سیٹ تجرباتی ہے کیونکہ میں اسے اپنے ہارڈویئر پر خود جانچ نہیں سکا۔ آپ کی رائے خوش آمدید ہے۔

براہ کرم `cec-client` پر اس discussion thread میں گفتگو کریں: <https://github.com/stefansundin/SSHRemote/discussions/7>

### macOS

macOS پر VLC کو کنٹرول کرنے کے لیے ایک پری سیٹ موجود ہے، جو AppleScript کمانڈز استعمال کرتا ہے۔ مجھے کسی ایسے ٹول کا علم نہیں جو کی بورڈ یا ماؤس ایونٹس بھیجنے کی معاونت کرتا ہو۔

براہ کرم macOS پر اس discussion thread میں گفتگو کریں: <https://github.com/stefansundin/SSHRemote/discussions/8>

### Android

اگر آپ کے Android ڈیوائس میں SSH server موجود ہے، تو ہو سکتا ہے کہ آپ اس سے کنیکٹ ہو کر ان پٹ ایونٹس بھیج سکیں۔ اس کا امکان زیادہ ہے اگر آپ کا ڈیوائس Raspberry Pi پر KonstaKANG جیسے custom ROM پر چل رہا ہو۔

میں ابھی تک یہ معلوم نہیں کر سکا کہ ماؤس سپورٹ کو کیسے کام کروایا جائے۔

براہ کرم Android پر اس discussion thread میں گفتگو کریں: <https://github.com/stefansundin/SSHRemote/discussions/9>

## سمارٹ والیوم کنٹرول

جب آپ ریموٹ کنٹرول میں ترمیم کرتے ہیں، تو مینو میں آپ کو volume control کی "smart" ترتیبات مل سکتی ہیں۔ یہ ایپ میں کمپیوٹر کا موجودہ والیوم دکھا سکتی ہیں اور آپ کو slider کے ذریعے جلدی سے والیوم سیٹ کرنے دیتی ہیں۔ آپ اپنے ڈیوائس کے ہارڈویئر بٹنوں سے بھی جلدی volume up/down کمانڈز بھیج سکتے ہیں۔

موجودہ والیوم پڑھنا اور slider کے ذریعے نیا والیوم سیٹ کرنا فی الحال `pactl` کے استعمال کے لیے hard-coded ہے۔

`pactl` والا پیکیج عموماً `pulseaudio-utils` یا `libpulse` کہلاتا ہے۔

## SSH کلیدیں

آپ ایپ کی ترتیبات میں SSH کلیدیں درآمد کر سکتے ہیں یا بنا سکتے ہیں۔ SSH کلید کے ساتھ کنیکٹ ہونا پاس ورڈ استعمال کرنے سے زیادہ محفوظ ہے۔

کمپیوٹر سے موجودہ SSH کلید درآمد کرنے کا سب سے آسان طریقہ QR کوڈ اسکین کرنا ہے۔ آپ QR کوڈ کی تصویر بنانے کے لیے `qrencode` پروگرام استعمال کر سکتے ہیں۔ QR کوڈ بنانے کے لیے اس طرح کی کمانڈ چلائیں:

```shell
# اپنی SSH کلیدوں کے فولڈر میں جائیں:
cd ~/.ssh

# ٹرمینل میں QR کوڈ دکھائیں:
qrencode -r id_ed25519 -t ansiutf8

# متبادل طور پر، ایک تصویر فائل بنائیں:
qrencode -r id_ed25519 -o qr.png

# 4096-bit RSA کلیدیں QR کوڈ میں فٹ نہیں ہوتیں۔ ایک کو بمشکل فٹ کرنے کے لیے gzip استعمال کیا جا سکتا ہے:
cat id_rsa_4096 | gzip | base64 | qrencode -o qr.png
```

آپ مینو میں موجود `عوامی کلید بھیجیں` فیچر کے ذریعے کسی سرور پر عوامی SSH کلید بھیج سکتے ہیں۔ یہ منتخب شدہ SSH کلید کو `~/.ssh/authorized_keys` فائل میں شامل کر دے گا۔ اس سے آپ آسانی سے پاس ورڈ کے ذریعے لاگ اِن سے SSH کلید کے ذریعے لاگ اِن کی طرف منتقل ہو سکتے ہیں۔

آپ خفیہ کردہ SSH کلیدیں درآمد کر کے استعمال کر سکتے ہیں، لیکن فی الحال آپ انہیں ایپ میں بنا نہیں سکتے۔

## سلامتی

ایپ سے SSH کلیدوں کا نجی حصہ یا محفوظ شدہ پاس ورڈز برآمد یا نکالنا ممکن نہیں ہے۔ یہ ڈیٹا 256-bit AES کے ذریعے خفیہ کیا جاتا ہے، اور خفیہ کاری کی کلید Android Keystore میں محفوظ کی جاتی ہے۔ خفیہ کردہ ڈیٹا Android بیک اپس میں شامل نہیں ہوتا۔

اس ایپ میں کوئی crash reporting software نہیں ہے۔ کوئی telemetry نہیں ہے۔ کوئی اشتہارات نہیں ہیں۔ SSH کنکشن کے علاوہ کوئی network requests نہیں کی جاتیں۔

اس ایپ کی سلامتی کا آڈٹ نہیں کیا گیا۔ اگر آپ کو Android security یا SSH security کا تجربہ ہے، تو براہ کرم سورس کوڈ دیکھیں اور اپنی رائے اس GitHub issue میں درج کریں:

<https://github.com/stefansundin/SSHRemote/issues/1>
