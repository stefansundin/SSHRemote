## SSH Remote ಬಗ್ಗೆ

ಈ ಅನುವಾದವನ್ನು GitHub Copilot ಬಳಸಿ AI ಸಹಾಯದಿಂದ ಮಾಡಲಾಗಿದೆ. ಈ ಪರಿಸರದಲ್ಲಿ ನಿಖರ ಮಾದರಿ ಮಾಹಿತಿಯನ್ನು ದೃಢೀಕರಿಸಲಾಗಲಿಲ್ಲ. ಅನುವಾದ ದೋಷಗಳು ಇರಬಹುದು.

SSH Remote ಒಂದು ಉಚಿತ ಮತ್ತು ಮುಕ್ತ ಮೂಲದ ಆಪ್ ಆಗಿದ್ದು, SSH ಬಳಸಿ ದೂರದಿಂದ ಕಂಪ್ಯೂಟರ್‌ಗಳನ್ನು ನಿಯಂತ್ರಿಸಲು ನಿಮಗೆ ಅವಕಾಶ ನೀಡುತ್ತದೆ.

ಚಲಾಯಿಸಲ್ಪಡುವ ಆಜ್ಞೆಗಳನ್ನು ನೀವು ಸಂಪೂರ್ಣವಾಗಿ ಕಸ್ಟಮೈಸ್ ಮಾಡಬಹುದು, ಮತ್ತು ಸಾಮಾನ್ಯ ವ್ಯವಸ್ಥೆಗಳಿಗಾಗಿ ಪ್ರೀಸೆಟ್‌ಗಳೂ ಇವೆ.

ನಾನು ಈ ಆಪ್ ಅನ್ನು Raspberry Pi OS ನಲ್ಲಿ ನಡೆಯುವ ನನ್ನ HTPC ವ್ಯವಸ್ಥೆಯನ್ನು ನಿಯಂತ್ರಿಸಲು ಬಳಸುತ್ತೇನೆ. HTPC ನಿಯಂತ್ರಿಸುವುದು ಈ ಆಪ್ ವಿಶೇಷವಾಗಿ ಹೊಂದಿಕೊಳ್ಳುವ ಮುಖ್ಯ ಸಂದರ್ಭವಾಗಿದೆ.

ಈ ಆಪ್ ಒಂದು ಟರ್ಮಿನಲ್ ಎಮ್ಯುಲೇಟರ್ ಅಲ್ಲ, ಆದರೆ ತುರ್ತು ಪರಿಸ್ಥಿತಿಯಲ್ಲಿ `apt-get install` ಅನ್ನು ಚಲಾಯಿಸಲು ಇದು ನಿಮಗೆ ಸಹಾಯ ಮಾಡುತ್ತದೆ.

## ಪ್ರಾರಂಭಿಸುವುದು

ಸಂಪರ್ಕಿಸಲು ನೀವು SSH ಕೀ ಬಳಸಲು ಬಯಸಿದರೆ, ಮೊದಲು ಆಪ್ ಸೆಟ್ಟಿಂಗ್‌ಗಳನ್ನು ತೆರೆಯಿರಿ ಮತ್ತು ಒಂದು ಕೀ ಅನ್ನು ಆಮದು ಮಾಡಿ ಅಥವಾ ರಚಿಸಿ.

ಕೆಳಗಿನ ಬಲ ಕೋನದಲ್ಲಿರುವ `+` ಬಟನ್ ಒತ್ತಿ ಹೊಸ ಹೋಸ್ಟ್ ಸೇರಿಸಿ. ಸಂಪರ್ಕ ವಿವರಗಳನ್ನು ನಮೂದಿಸಿ ಮತ್ತು ಉಳಿಸಿ.

ಮೊದಲ ಬಾರಿ ಸಂಪರ್ಕಿಸಿದಾಗ, ನಿಮಗೆ ಒಂದು ಪ್ರೀಸೆಟ್ ಆಯ್ಕೆಮಾಡಲು ಕೇಳಲಾಗುತ್ತದೆ. ಈ ಆಯ್ಕೆ ರಿಮೋಟ್ ಕಂಟ್ರೋಲ್ ಬಟನ್‌ಗಳನ್ನು ವಿವಿಧ ರೀತಿಯ ಕಂಪ್ಯೂಟರ್‌ಗಳ ಜೊತೆ ಸರಿಯಾಗಿ ಕೆಲಸ ಮಾಡುವಂತೆ ಸಂರಚಿಸುತ್ತದೆ. ಲಭ್ಯವಿರುವ ಪ್ರೀಸೆಟ್‌ಗಳ ವಿವರಕ್ಕಾಗಿ ಕೆಳಗೆ ನೋಡಿ. ನೀವು ಬಯಸಿದರೆ, `No preset` ಆಯ್ಕೆಮಾಡಿ ಯಾವುದೇ ಸಂರಚನೆಯಿಲ್ಲದೆ ಆರಂಭಿಸಬಹುದು.

ನಿಮ್ಮ Linux ಕಂಪ್ಯೂಟರ್ X11 ಅಥವಾ Wayland ನಲ್ಲಿ ಚಲಿಸುತ್ತಿದೆಯೇ ಎಂಬುದು ಗೊತ್ತಿಲ್ಲದಿದ್ದರೆ, ಟರ್ಮಿನಲ್‌ನಲ್ಲಿ ಇದನ್ನು ಚಲಾಯಿಸಿ:

```shell
echo $XDG_SESSION_TYPE
```

ಇದರಿಂದ `x11` ಅಥವಾ `wayland` ಎಂದು ಔಟ್‌ಪುಟ್ ಬರಬೇಕು. ನೀವು ಇದನ್ನು ಡೆಸ್ಕ್‌ಟಾಪ್ ಪರಿಸರದ ಒಳಗಿನಿಂದಲೇ ಚಲಾಯಿಸಬೇಕು.

## ರಿಮೋಟ್ ಕಂಟ್ರೋಲ್

ಒಮ್ಮೆ ಸಂಪರ್ಕವಾದ ನಂತರ, ಆಜ್ಞೆಗಳನ್ನು ಕಳುಹಿಸಲು ನೀವು ರಿಮೋಟ್ ಕಂಟ್ರೋಲ್ ಇಂಟರ್‌ಫೇಸ್ ಅನ್ನು ಬಳಸಬಹುದು. ವಿವಿಧ ಇನ್‌ಪುಟ್ ವಿಧಾನಗಳಿಗೆ ಪ್ರವೇಶಿಸಲು ಟ್ಯಾಬ್‌ಗಳನ್ನು ಬದಲಿಸಿ.

ಪ್ರತಿ ಬಟನ್ ಒತ್ತುವಿಕೆಯೂ ಹೋಸ್ಟ್‌ನಲ್ಲಿ ಒಂದು ಆಜ್ಞೆಯನ್ನು ಚಲಾಯಿಸುತ್ತದೆ. ಸರಳ ಕೀ ಒತ್ತುವಿಕೆಯಂತಹ ಕೆಲಸಕ್ಕೂ ಇದು ಸಾಕಷ್ಟು ಹೆಚ್ಚುವರಿ ಕಾರ್ಯಭಾರವನ್ನುಂಟುಮಾಡುತ್ತದೆ, ಆದ್ದರಿಂದ ಸಾಮಾನ್ಯ ಕೀಬೋರ್ಡ್‌ನೊಂದಿಗೆ ಹೋಲಿಸಿದರೆ ನಿಮಗೆ ಹೆಚ್ಚಿನ ವಿಳಂಬ ಅನುಭವವಾಗಬಹುದು. ಭವಿಷ್ಯದ ಆವೃತ್ತಿಗಳಲ್ಲಿ ಇದನ್ನು ಸುಧಾರಿಸಲು ನಾನು ಆಶಿಸುತ್ತೇನೆ.

ಸಂಪಾದನಾ ಮೋಡ್‌ಗೆ ಹೋಗಲು ಮೆನು ಬಳಸಿ. ಈಗಿನಂತಿಗೆ ವಿನ್ಯಾಸ ಅಥವಾ ಬಟನ್ ಐಕಾನ್‌ಗಳನ್ನು ಸಂಪಾದಿಸುವುದು ಸಾಧ್ಯವಿಲ್ಲ. ಮುಂದಿನ ಆವೃತ್ತಿಯಲ್ಲಿ ಇದನ್ನು ಸಾಧ್ಯವಾಗಿಸಲು ನಾನು ಆಶಿಸುತ್ತೇನೆ.

## ಪ್ರೀಸೆಟ್‌ಗಳು

ನಿಮ್ಮ ಡೆಸ್ಕ್‌ಟಾಪ್ ಪರಿಸರಕ್ಕೆ ಅಗತ್ಯವಾದ ಸಾಧನವನ್ನು ನೀವು ಸ್ಥಾಪಿಸಬೇಕು.

ನನ್ನ ಶಿಫಾರಸು `ydotool`, ಏಕೆಂದರೆ ನನ್ನ ಪರೀಕ್ಷೆಗಳಲ್ಲಿ ಅದು ಅತ್ಯುತ್ತಮ ಕಾರ್ಯಕ್ಷಮತೆಯನ್ನು ನೀಡಿತು, ಮತ್ತು ಅದು X11 ಹಾಗು Wayland ಎರಡರಲ್ಲೂ ಕೆಲಸ ಮಾಡುತ್ತದೆ.

### ydotool

`ydotool` ಯಾವುದೇ window manager ಜೊತೆ ಕೆಲಸ ಮಾಡಬೇಕು, ಆದರೆ ಹಿನ್ನಲೆಯಲ್ಲಿ ಒಂದು ಸೇವೆ ಓಡುತ್ತಿರಬೇಕು. ನಿಮ್ಮ ಡಿಸ್ಟ್ರೋ systemd user service ನೀಡುತ್ತಿದ್ದರೆ, ಇದನ್ನು ಚಲಾಯಿಸಿ ಅದನ್ನು ಪ್ರಾರಂಭಿಸಿ:

```shell
systemctl start --user ydotool
```

ಲಾಗಿನ್ ಸಂದರ್ಭದಲ್ಲಿ ಸೇವೆಯನ್ನು ಸ್ವಯಂಚಾಲಿತವಾಗಿ ಪ್ರಾರಂಭಿಸಲು, ಇದನ್ನು ಚಲಾಯಿಸಿ:

```shell
systemctl enable --user ydotool
```

ದಯವಿಟ್ಟು ನೀವು ಸಾಕಷ್ಟು ಹೊಸ ಆವೃತ್ತಿಯ `ydotool` ಅನ್ನು ಸ್ಥಾಪಿಸುತ್ತಿದ್ದೀರಿ ಎಂದು ಖಚಿತಪಡಿಸಿಕೊಳ್ಳಿ. Ubuntu 26.04 ಗಿಂತ ಹಳೆಯ ಆವೃತ್ತಿಗಳಲ್ಲಿ ತುಂಬಾ ಹಳೆಯ ಆವೃತ್ತಿಗಳಿರುತ್ತವೆ. ಪರಿಹಾರಕ್ಕಾಗಿ discussion thread ನೋಡಿ.

ದಯವಿಟ್ಟು `ydotool` ಕುರಿತು ಈ discussion thread ನಲ್ಲಿ ಚರ್ಚಿಸಿ: <https://github.com/stefansundin/SSHRemote/discussions/5>

### xdotool

`xdotool` ಎಂಬುದು X11 ನಲ್ಲಿ ನಡೆಯುವ ಕಂಪ್ಯೂಟರ್‌ಗಳಿಗೆ. X11 ಇತಿಹಾಸಾತ್ಮಕವಾಗಿ ಹೆಚ್ಚಿನ Linux ಕಂಪ್ಯೂಟರ್‌ಗಳು ಬಳಸಿದ ವ್ಯವಸ್ಥೆ, ಆದರೂ Wayland ಹೆಚ್ಚು ಜನಪ್ರಿಯವಾಗುತ್ತಿದೆ.

X11 ನಲ್ಲಿ ಒಂದು ವೈಶಿಷ್ಟ್ಯವೆಂದರೆ, X server ಗೆ ಪ್ರವೇಶವನ್ನು ನೀವು ಅನುಮತಿಸಬೇಕಾಗಬಹುದು. ನಿಮಗೆ "Authorization required" ದೋಷಗಳು ಬಂದರೆ ಸಮಸ್ಯೆ ಇದೇ ಆಗಿರಬಹುದು. ಇದನ್ನು ಸರಿಪಡಿಸಲು ನಿಮಗೆ ಕೆಲವು ಆಯ್ಕೆಗಳಿವೆ; ನನಗೆ ಕೆಲಸ ಮಾಡಿದ ಎರಡು ವಿಧಾನಗಳು ಇಲ್ಲಿವೆ:

`xauth list` ಯಾವುದೇ ನಮೂದುಗಳನ್ನು ತೋರಿಸದಿದ್ದರೆ, `.Xauthority` ಫೈಲ್ ರಚಿಸಲು ಪ್ರಯತ್ನಿಸಿ:

```shell
xauth generate :0 . trusted
```

ಅದು ಕೆಲಸ ಮಾಡದಿದ್ದರೆ, `xhost` ಬಳಸಿ ಪ್ರವೇಶ ನೀಡಲು ಪ್ರಯತ್ನಿಸಿ:

```shell
xhost +local:$USER
```

ಪ್ರತಿ ಬೂಟ್‌ನ ನಂತರ ನೀವು `xhost` ಆಜ್ಞೆಯನ್ನು ಮತ್ತೆ ಚಲಾಯಿಸಬೇಕಾಗುತ್ತದೆ. ಇದನ್ನು ಸ್ವಯಂಚಾಲಿತಗೊಳಿಸಲು ಒಂದು bash script ರಚಿಸಿ ಮತ್ತು ಲಾಗಿನ್ ಸಮಯದಲ್ಲಿ ಸ್ವಯಂಚಾಲಿತವಾಗಿ ಪ್ರಾರಂಭವಾಗುವಂತೆ ಸಂರಚಿಸಬಹುದು.

ದಯವಿಟ್ಟು `xdotool` ಕುರಿತು ಈ discussion thread ನಲ್ಲಿ ಚರ್ಚಿಸಿ: <https://github.com/stefansundin/SSHRemote/discussions/3>

### wtype

`wtype` ಅಂದರೆ Wayland ಗಾಗಿ `xdotool` ತರಹ.

ಸಾಮಾನ್ಯವಾಗಿ, ಇದು ಮೌಸ್ ನಿಯಂತ್ರಣಕ್ಕೆ ಬೆಂಬಲ ನೀಡುವುದಿಲ್ಲ, ಆದರೆ ನಾನು ಮೌಸ್ ಬೆಂಬಲ ಸೇರಿಸಿದ ಪರಿವರ್ತಿತ ಆವೃತ್ತಿಯನ್ನು ಸೃಷ್ಟಿಸಿದ್ದೇನೆ. ನಿಮಗೆ ಮೌಸ್ ಬೆಂಬಲ ಬೇಕಿದ್ದರೆ ದಯವಿಟ್ಟು ಅದನ್ನು ಸ್ಥಾಪಿಸಿ: <https://github.com/stefansundin/wtype>

`Compositor does not support the virtual keyboard protocol` ಎಂಬ ದೋಷ ಬಂದರೆ, `ydotool` ಹೋಲುವ ಮತ್ತೊಂದು ಸಾಧನವನ್ನು ಪ್ರಯತ್ನಿಸುವಂತೆ ನಾನು ಸಲಹೆ ನೀಡುತ್ತೇನೆ.

ದಯವಿಟ್ಟು `wtype` ಕುರಿತು ಈ discussion thread ನಲ್ಲಿ ಚರ್ಚಿಸಿ: <https://github.com/stefansundin/SSHRemote/discussions/4>

### dotool

`dotool` ಯಾವುದೇ window manager ಜೊತೆ ಕೆಲಸ ಮಾಡಬೇಕು, `ydotool` ತರಹ. ನನ್ನ ಸೀಮಿತ ಪರೀಕ್ಷೆಗಳಲ್ಲಿ ಇದು `ydotool` ಗಿಂತ ಬಹಳ ನಿಧಾನವಾಗಿತ್ತು.

ಈ ಪ್ರೀಸೆಟ್ ಪ್ರಯೋಗಾತ್ಮಕವಾಗಿದೆ, ಏಕೆಂದರೆ ನನ್ನ ಸ್ವಂತ ಹಾರ್ಡ್‌ವೇರ್‌ನಲ್ಲಿ ಇದನ್ನು ಪರಿಶೀಲಿಸಲು ನನಗೆ ಸಾಧ್ಯವಾಗಿಲ್ಲ. ನಿಮ್ಮ ಪ್ರತಿಕ್ರಿಯೆ ಸ್ವಾಗತಾರ್ಹ.

### Android

ನಿಮ್ಮ Android ಸಾಧನದಲ್ಲಿ SSH server ಇದ್ದರೆ, ನೀವು ಅದಕ್ಕೆ ಸಂಪರ್ಕಿಸಿ ಇನ್‌ಪುಟ್ ಈವೆಂಟ್‌ಗಳನ್ನು ಕಳುಹಿಸಲು ಸಾಧ್ಯವಾಗಬಹುದು. ನಿಮ್ಮ ಸಾಧನವು Raspberry Pi ಯ KonstaKANG ತರಹದ custom ROM ಅನ್ನು ಬಳಸುತ್ತಿದ್ದರೆ ಇದು ಹೆಚ್ಚು ಸಾಧ್ಯತೆ ಇದೆ.

ಮೌಸ್ ಬೆಂಬಲವನ್ನು ಹೇಗೆ ಕಾರ್ಯನಿರ್ವಹಿಸುವಂತೆ ಮಾಡುವುದು ಎಂದು ನನಗೆ ಇನ್ನೂ ಗೊತ್ತಾಗಿಲ್ಲ.

## ಸ್ಮಾರ್ಟ್ ಶಬ್ದಮಟ್ಟ ನಿಯಂತ್ರಣ

ರಿಮೋಟ್ ಕಂಟ್ರೋಲ್ ತಿದ್ದುತ್ತಿರುವಾಗ, ಮೆನುದಲ್ಲಿ "smart" volume control ಸೆಟ್ಟಿಂಗ್‌ಗಳನ್ನು ಕಂಡುಹಿಡಿಯಬಹುದು. ಇದರಿಂದ ಆಪ್‌ನಲ್ಲಿ ಕಂಪ್ಯೂಟರ್‌ನ ಪ್ರಸ್ತುತ ಶಬ್ದಮಟ್ಟವನ್ನು ತೋರಿಸಬಹುದು ಮತ್ತು ಸ್ಲೈಡರ್ ಬಳಸಿ ಅದನ್ನು ವೇಗವಾಗಿ ಹೊಂದಿಸಬಹುದು. ನಿಮ್ಮ ಸಾಧನದ ಹಾರ್ಡ್‌ವೇರ್ ಬಟನ್‌ಗಳನ್ನು ಬಳಸಿ volume up/down ಆಜ್ಞೆಗಳನ್ನು ತ್ವರಿತವಾಗಿ ಕಳುಹಿಸಬಹುದೂ ಹೌದು.

ಪ್ರಸ್ತುತ ಶಬ್ದಮಟ್ಟವನ್ನು ಓದುವುದು ಮತ್ತು ಸ್ಲೈಡರ್ ಬಳಸಿ ಹೊಸ ಶಬ್ದಮಟ್ಟವನ್ನು ಹೊಂದಿಸುವುದು ಈಗ `pactl` ಬಳಕೆಗೆ hard-coded ಆಗಿದೆ.

`pactl` ಹೊಂದಿರುವ ಪ್ಯಾಕೇಜ್ ಸಾಮಾನ್ಯವಾಗಿ `pulseaudio-utils` ಅಥವಾ `libpulse` ಎಂದು ಕರೆಯಲ್ಪಡುತ್ತದೆ.

## SSH ಕೀಲಿಗಳು

ನೀವು ಆಪ್ ಸೆಟ್ಟಿಂಗ್‌ಗಳಲ್ಲಿ SSH ಕೀಲಿಗಳನ್ನು ಆಮದು ಮಾಡಬಹುದು ಅಥವಾ ರಚಿಸಬಹುದು. SSH ಕೀ ಬಳಸಿ ಸಂಪರ್ಕಿಸುವುದು ಪಾಸ್ವರ್ಡ್ ಬಳಕೆಯಿಗಿಂತ ಹೆಚ್ಚು ಸುರಕ್ಷಿತವಾಗಿದೆ.

ಕಂಪ್ಯೂಟರ್‌ನಿಂದ ಈಗಿರುವ SSH ಕೀ ಅನ್ನು ಆಮದು ಮಾಡುವ ಸುಲಭವಾದ ವಿಧಾನವೆಂದರೆ QR ಕೋಡ್ ಸ್ಕ್ಯಾನ್ ಮಾಡುವುದು. QR ಕೋಡ್ ಚಿತ್ರವನ್ನು ರಚಿಸಲು `qrencode` ಪ್ರೋಗ್ರಾಂ ಬಳಸಬಹುದು. ಕೆಳಗಿನಂತೆ ಒಂದು ಆಜ್ಞೆಯನ್ನು ಚಲಾಯಿಸಿ QR ಕೋಡ್ ರಚಿಸಬಹುದು:

```shell
# ನಿಮ್ಮ SSH ಕೀಲಿಗಳಿರುವ ಸ್ಥಳಕ್ಕೆ ಹೋಗಿ:
cd ~/.ssh

# ಟರ್ಮಿನಲ್‌ನಲ್ಲಿ QR ಕೋಡ್ ಅನ್ನು ತೋರಿಸಿ:
qrencode -r id_ed25519 -t ansiutf8

# ಪರ್ಯಾಯವಾಗಿ, ಒಂದು image ಫೈಲ್ ರಚಿಸಿ:
qrencode -r id_ed25519 -o qr.png

# 4096-bit RSA ಕೀಲಿಗಳು QR ಕೋಡ್‌ಗೆ ತುಂಬಾ ದೊಡ್ಡವು. ಒಂದು ಕಷ್ಟಕ್ಕೆ ಸರಿಹೊಂದಿಸಲು gzip ಬಳಸಬಹುದು:
cat id_rsa_4096 | gzip | base64 | qrencode -o qr.png
```

ಮೆನುದಲ್ಲಿರುವ `Push public key` ವೈಶಿಷ್ಟ್ಯವನ್ನು ಬಳಸಿ ನೀವು ಸಾರ್ವಜನಿಕ SSH ಕೀಲಿಗಳನ್ನು server ಗೆ ಕಳುಹಿಸಬಹುದು. ಇದು ಆಯ್ಕೆಮಾಡಿದ SSH ಕೀ ಅನ್ನು `~/.ssh/authorized_keys` ಫೈಲ್‌ಗೆ append ಮಾಡುತ್ತದೆ. ಹೀಗೆ ಪಾಸ್ವರ್ಡ್ ಮೂಲಕ ಲಾಗಿನ್ ಮಾಡುವುದರಿಂದ SSH ಕೀ ಮೂಲಕ ಲಾಗಿನ್ ಮಾಡುವುದಕ್ಕೆ ಸುಲಭವಾಗಿ ವರ್ಗಾಯಿಸಬಹುದು.

ನೀವು ಎನ್ಕ್ರಿಪ್ಟ್ ಮಾಡಿದ SSH ಕೀಲಿಗಳನ್ನು ಆಮದು ಮಾಡಬಹುದು ಮತ್ತು ಬಳಸಬಹುದು, ಆದರೆ ಈಗಿನಂತಿಗೆ ಆಪ್‌ನಲ್ಲೇ ಅವನ್ನು ರಚಿಸಲು ಸಾಧ್ಯವಿಲ್ಲ.

## ಭದ್ರತೆ

ಆಪ್‌ನಿಂದ SSH ಕೀಲಿಗಳ ಖಾಸಗಿ ಭಾಗವನ್ನು, ಅಥವಾ ಸಂಗ್ರಹಿತ ಪಾಸ್ವರ್ಡ್‌ಗಳನ್ನು, ರಫ್ತು ಮಾಡುವುದು ಅಥವಾ ಹೊರತೆಗೆದು ಕೊಳ್ಳುವುದು ಸಾಧ್ಯವಿಲ್ಲ. ಈ ದತ್ತಾಂಶವನ್ನು 256-bit AES ಬಳಸಿ ಎನ್ಕ್ರಿಪ್ಟ್ ಮಾಡಲಾಗುತ್ತದೆ, ಮತ್ತು ಎನ್ಕ್ರಿಪ್ಷನ್ ಕೀ ಅನ್ನು Android Keystore ನಲ್ಲಿ ಸಂಗ್ರಹಿಸಲಾಗುತ್ತದೆ. ಎನ್ಕ್ರಿಪ್ಟ್ ಮಾಡಲಾದ ದತ್ತಾಂಶವನ್ನು Android backup ಗಳಿಂದ ಹೊರತುಪಡಿಸಲಾಗುತ್ತದೆ.

ಈ ಆಪ್‌ನಲ್ಲಿ ಯಾವುದೇ crash reporting software ಇಲ್ಲ. ಯಾವುದೇ telemetry ಇಲ್ಲ. ಯಾವುದೇ ಜಾಹೀರಾತುಗಳಿಲ್ಲ. SSH ಸಂಪರ್ಕ ಹೊರತುಪಡಿಸಿ ಯಾವುದೇ network request ಗಳು ಇಲ್ಲ.

ಈ ಆಪ್‌ನ ಭದ್ರತೆಯನ್ನು audit ಮಾಡಲಾಗಿಲ್ಲ. ನೀವು Android security ಅಥವಾ SSH security ನಲ್ಲಿ ಅನುಭವ ಹೊಂದಿದ್ದರೆ, ದಯವಿಟ್ಟು ಮೂಲ ಕೋಡ್ ನೋಡಿ ಮತ್ತು ನಿಮ್ಮ ಕಂಡುಹಿಡಿಕೆಗಳನ್ನು ಈ GitHub issue ನಲ್ಲಿ ವರದಿ ಮಾಡಿ:

<https://github.com/stefansundin/SSHRemote/issues/1>

ದಯವಿಟ್ಟು ಗೌರವಪೂರ್ವಕವಾಗಿರಿ. ನಾನು ಈ ಆಪ್ ಅನ್ನು ನನ್ನ ಖಾಲಿ ಸಮಯದಲ್ಲಿ ನಿರ್ಮಿಸಿದ್ದೇನೆ ಮತ್ತು ಇದನ್ನು ಉಚಿತವಾಗಿ ನೀಡುತ್ತಿದ್ದೇನೆ. ಮೊದಲನೆಯದಾಗಿ, ನಾನು ಈ ಆಪ್ ಅನ್ನು ನನ್ನ ಸ್ವಂತ ಬಳಕೆಗೆ ನಿರ್ಮಿಸುತ್ತಿದ್ದೇನೆ.
