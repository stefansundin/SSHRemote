## SSH Remote жөнүндө

Бул котормо GitHub Copilot тарабынан AI жардамы менен жасалды; колдонулган модель аталышы редактор чөйрөсүндө так көрсөтүлгөн эмес. Котормодо каталар болушу мүмкүн.

SSH Remote — SSH аркылуу компьютерлерди алыстан башкарууга мүмкүндүк берген акысыз жана ачык булактуу колдонмо.

Аткарылуучу буйруктарды толугу менен ыңгайлаштыра аласыз, ошондой эле кеңири колдонулган орнотуулар үчүн алдын ала орнотуулар бар.

Мен бул колдонмону Raspberry Pi OS иштеткен HTPC системамды башкаруу үчүн колдоном. HTPC'ни башкаруу — бул колдонмо эң көп ылайыкташтырылган негизги сценарий.

Бул колдонмо терминал эмулятору эмес, бирок шашылыш учурда `apt-get install` иштетүүгө мүмкүнчүлүк берет.

## Баштоо

Эгер туташуу үчүн SSH ачкычын колдонгуңуз келсе, адегенде колдонмонун жөндөөлөрүн ачып, ачкычты импорттоңуз же түзүңүз.

Төмөнкү оң бурчтагы `+` баскычын таптап, жаңы хост кошуңуз. Туташуу маалыматтарын киргизип, сактаңыз.

Биринчи жолу туташканыңызда, сизден алдын ала орнотууну тандоо суралат. Бул тандоо пульттагы баскычтарды ар кандай компьютер түрлөрү менен жакшы иштей тургандай конфигурациялайт. Төмөндө жеткиликтүү алдын ала орнотуулардын сүрөттөмөсү келтирилген. Кааласаңыз, `Алдын ала орнотуу жок` дегенди тандап, эч кандай конфигурациясыз баштасаңыз да болот.

Эгер Linux компьютериңиз X11 же Wayland иштетип жатканын билбесеңиз, терминалда төмөнкүлөрдү иштетиңиз:

```shell
echo $XDG_SESSION_TYPE
```

Натыйжада `x11` же `wayland` чыгышы керек. Бул буйрукту сөзсүз түрдө иш такта чөйрөсүнүн ичинде иштетиңиз.

## Пульт

Туташкандан кийин, буйруктарды жөнөтүү үчүн пульт интерфейсин колдоно аласыз. Ар кандай киргизүү ыкмаларына жетүү үчүн өтмөктөрдү алмаштырыңыз.

Ар бир баскыч басылганда хостто буйрук аткарылат. Жөнөкөй эле баскыч басууга салыштырмалуу бул бир топ кошумча чыгымды талап кылат, ошондуктан кадимки баскычтопко салыштырмалуу кечигүү кыйла жогору болушу мүмкүн. Муну келечектеги версияларда жакшыртам деп үмүттөнөм.

Түзөтүү режимине кирүү үчүн менюну колдонуңуз. Азырынча жайгашууну же баскыч сөлөкөттөрүн түзөтүү мүмкүн эмес. Муну да келечектеги версияларда мүмкүн кылам деп үмүттөнөм.

## Алдын ала орнотуулар

Иш такта чөйрөңүзгө керектүү куралды орнотушуңуз керек болот.

Мен `ydotool`'ду сунуштайм, анткени тесттеримде ал эң жакшы иштеди жана X11 менен Wayland'дын экөөндө тең иштейт.

### ydotool

`ydotool` каалаган терезе башкаргычы менен иштеши керек, бирок фондо кызмат иштеп турушу зарыл. Эгер дистрибутивиңиз systemd колдонуучу кызматын берсе, анда аны төмөнкү буйрук менен баштаңыз:

```shell
systemctl start --user ydotool
```

Киргенде кызмат автоматтык түрдө башталышы үчүн төмөнкү буйрукту иштетиңиз:

```shell
systemctl enable --user ydotool
```

`ydotool`'дун жетишерлик жаңы версиясын орнотуп жатканыңызды текшериңиз. Ubuntu'нун 26.04'кө чейинки версияларындагы варианттар өтө эски. Чечүү жолун талкуу темасынан караңыз.

`ydotool` боюнча талкууну бул жерден жүргүзүңүз: <https://github.com/stefansundin/SSHRemote/discussions/5>

### xdotool

`xdotool` X11 иштеткен компьютерлер үчүн. Тарыхый жактан алганда көпчүлүк Linux компьютерлери X11 колдонуп келген, бирок азыр Wayland барган сайын популярдуу болуп жатат.

X11'деги өзгөчөлүктөрдүн бири — сизге X серверине кирүүгө уруксат берүү талап кылынышы мүмкүн. Эгер `Authorization required` каталарын көрсөңүз, маселе ушунда болушу мүмкүн. Бул көйгөйдү чечүүнүн бир нече жолу бар, алардын экөө мага жардам берген:

Эгер `xauth list` эч кандай жазуу көрсөтпөсө, `.Xauthority` файлын түзүүгө аракет кылып көрүңүз:

```shell
xauth generate :0 . trusted
```

Эгер бул иштебесе, анда `xhost` аркылуу уруксат берүүгө аракет кылыңыз:

```shell
xhost +local:$USER
```

`xhost` буйругун ар бир жүктөөдөн кийин кайра иштетишиңиз керек болот. Муну автоматташтыруу үчүн bash скриптин түзүп, аны киргенде автоматтык ишке кирүүгө коюңуз.

`xdotool` боюнча талкууну бул жерден жүргүзүңүз: <https://github.com/stefansundin/SSHRemote/discussions/3>

### wtype

`wtype` — Wayland үчүн `xdotool` сыяктуу курал.

Адатта ал чычканды башкарууну колдобойт, бирок мен чычкан колдоосун кошкон өзгөртүлгөн версиясын жасагам. Эгер чычкан колдоосу керек болсо, аны орнотуңуз: <https://github.com/stefansundin/wtype>

Эгер `Compositor does not support the virtual keyboard protocol` катасын көрсөңүз, анда `ydotool` сыяктуу башка куралды колдонуп көрүүнү сунуштайм.

`wtype` боюнча талкууну бул жерден жүргүзүңүз: <https://github.com/stefansundin/SSHRemote/discussions/4>

### dotool

`dotool` дагы `ydotool` сыяктуу каалаган терезе башкаргычы менен иштеши керек. Менин чектелүү тесттеримде ал `ydotool`'га караганда кыйла жайыраак болду.

`dotool` боюнча талкууну бул жерден жүргүзүңүз: <https://github.com/stefansundin/SSHRemote/discussions/6>

### cec-client

Бул алдын ала орнотуу эксперименталдык, анткени мен аны өз жабдуумда текшере алган жокмун. Пикирлер кабыл алынат.

`cec-client` боюнча талкууну бул жерден жүргүзүңүз: <https://github.com/stefansundin/SSHRemote/discussions/7>

### macOS

VLC'ни macOS'то AppleScript буйруктары аркылуу башкаруу үчүн алдын ала орнотуу бар. Азырынча баскычтоп же чычкан окуяларын жөнөтүүнү колдогон куралды билбейм.

macOS боюнча талкууну бул жерден жүргүзүңүз: <https://github.com/stefansundin/SSHRemote/discussions/8>

### Android

Эгер Android түзмөгүңүздө SSH сервери бар болсо, ага туташып, киргизүү окуяларын жөнөтө алышыңыз мүмкүн. Бул айрыкча KonstaKANG сыяктуу кастомдук ROM колдонгон түзмөктөрдө, мисалы Raspberry Pi'де, ыктымалы жогору.

Чычкан колдоосун кантип иштетүүнү азырынча таба алган жокмун.

Android боюнча талкууну бул жерден жүргүзүңүз: <https://github.com/stefansundin/SSHRemote/discussions/9>

## Акылдуу үн башкаруусу

Пультту түзөтүп жатканда, менюдан үн үчүн "акылдуу" жөндөөлөрдү таба аласыз. Бул компьютердин учурдагы үн деңгээлин колдонмодо көрсөтүп, сыдырма аркылуу үндү тез өзгөртүүгө мүмкүндүк берет. Ошондой эле түзмөгүңүздүн аппараттык баскычтарын колдонуп, үнүн көбөйтүү/азайтуу буйруктарын бат жөнөтө аласыз.

Учурдагы үн деңгээлин окуу жана сыдырма аркылуу жаңы маанини коюу азырынча `pactl` колдонууга катуу байланган.

`pactl` камтылган пакет адатта `pulseaudio-utils` же `libpulse` деп аталат.

## SSH ачкычтары

SSH ачкычтарын колдонмонун жөндөөлөрүнөн импорттосоңуз же түзсөңүз болот. SSH ачкычы менен туташуу сырсөз колдонууга караганда коопсузураак.

Компьютерден бар болгон SSH ачкычын импорттоонун эң оңой жолу — QR кодду скандоо. QR код сүрөтүн түзүү үчүн `qrencode` программасын колдоно аласыз. QR кодду түзүү үчүн төмөнкү сыяктуу буйрукту иштетиңиз:

```shell
# SSH ачкычтарыңыз жайгашкан каталогго өтүңүз:
cd ~/.ssh

# QR кодду терминалда көрсөтүңүз:
qrencode -r id_ed25519 -t ansiutf8

# Же болбосо, сүрөт файлын түзүңүз:
qrencode -r id_ed25519 -o qr.png

# 4096-биттик RSA ачкычтары QR кодго батпайт. Gzip колдонсоңуз, араң батышы мүмкүн:
cat id_rsa_4096 | gzip | base64 | qrencode -o qr.png
```

Менюдагы `Ачык ачкычты жиберүү` мүмкүнчүлүгүн колдонуп, ачык SSH ачкычтарын серверге жибере аласыз. Бул тандалган SSH ачкычын `~/.ssh/authorized_keys` файлынын аягына кошот. Мунун жардамы менен сырсөз аркылуу кирүүдөн SSH ачкычы аркылуу кирүүгө оңой өтө аласыз.

Шифрленген SSH ачкычтарын импорттоп колдонсоңуз болот, бирок азырынча колдонмонун ичинде андай ачкычтарды түзө албайсыз.

## Коопсуздук

Колдонмодон SSH ачкычтарынын жеке бөлүгүн же сакталган сырсөздөрдү экспорттоо же чыгарып алуу мүмкүн эмес. Бул маалымат 256-биттик AES менен шифрленет, ал эми шифрлөө ачкычы Android Keystore'до сакталат. Шифрленген маалыматтар Android резервдик көчүрмөлөрүнө кирбейт.

Бул колдонмодо кыйроолор тууралуу отчет берүүчү программа жок. Телеметрия жок. Жарнама жок. SSH туташуусунан башка тармактык сурамдар жасалбайт.

Бул колдонмонун коопсуздугу аудиттен өткөн эмес. Эгер Android коопсуздугу же SSH коопсуздугу боюнча тажрыйбаңыз болсо, баштапкы кодду карап чыгып, табылгаларыңызды ушул GitHub issue'га жазыңыз:

<https://github.com/stefansundin/SSHRemote/issues/1>

## Функция сунуштары

GitHub репозиторийине функция сунуштарын жана ката тууралуу билдирүүлөрдү жөнөтө бериңиз. Сураныч, англис тилин колдонуңуз. Комментарийлериңиз сылык болсун. Урматсыз комментарийлер өчүрүлөт, ал эми колдонуучуларга репозиторийден бөгөт коюлушу мүмкүн.

Сурооңуз буга чейин берилип же жооп берилип калган жокпу — ушуну көрүү үчүн бар болгон issue'ларды жана талкуу темаларын карап чыгыңыз.

Сураныч, урмат-сый менен жазыңыз. Мен бул колдонмону бош убактымда жасап, аны акысыз таратып жатам. Бул колдонмону биринчи кезекте өз муктаждыктарым үчүн иштеп чыгам.

Сураныч, суроолорду мага электрондук почта менен жибербеңиз. Мүмкүн болушунча сүйлөшүүлөр GitHub'да болсун, анткени бул башкаларга да жардам берет! Суроолоруңуз болсо, GitHub'дагы талкуу бөлүмүндө берсеңиз болот.

Өзүңүздүн функцияларыңызды ишке ашыруу үчүн колдонмону форк кылууга ар дайым болот. Бул — үйрөнүүнүн эң сонун жолу. Пайдалуу функциялар менен салым кошууну ойлонуп көрүңүз.
