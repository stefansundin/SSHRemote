## SSH Remote ਬਾਰੇ

ਇਹ ਅਨੁਵਾਦ AI ਦੀ ਮਦਦ ਨਾਲ ਕੀਤਾ ਗਿਆ ਹੈ, ਖਾਸ ਤੌਰ ਤੇ GitHub Copilot (GPT-5.3-Codex) ਦੀ ਵਰਤੋਂ ਕਰਕੇ। ਇਸ ਵਿੱਚ ਅਨੁਵਾਦ ਸੰਬੰਧੀ ਗਲਤੀਆਂ ਹੋ ਸਕਦੀਆਂ ਹਨ।

SSH Remote ਇੱਕ ਮੁਫ਼ਤ ਅਤੇ ਓਪਨ ਸੋਰਸ ਐਪ ਹੈ ਜੋ ਤੁਹਾਨੂੰ SSH ਦੀ ਵਰਤੋਂ ਕਰਕੇ ਕੰਪਿਊਟਰਾਂ ਨੂੰ ਦੂਰੋਂ ਕੰਟਰੋਲ ਕਰਨ ਦੀ ਆਗਿਆ ਦਿੰਦੀ ਹੈ।

ਤੁਸੀਂ ਚਲਾਈਆਂ ਜਾਣ ਵਾਲੀਆਂ ਕਮਾਂਡਾਂ ਨੂੰ ਪੂਰੀ ਤਰ੍ਹਾਂ ਅਨੁਕੂਲਿਤ ਕਰ ਸਕਦੇ ਹੋ, ਅਤੇ ਆਮ ਸੈੱਟਅੱਪਾਂ ਲਈ ਪ੍ਰੀਸੈਟ ਵੀ ਉਪਲਬਧ ਹਨ।

ਮੈਂ ਇਸ ਐਪ ਨੂੰ ਆਪਣੇ HTPC ਸੈੱਟਅੱਪ ਨੂੰ ਕੰਟਰੋਲ ਕਰਨ ਲਈ ਵਰਤਦਾ ਹਾਂ, ਜੋ Raspberry Pi OS ਤੇ ਚੱਲ ਰਿਹਾ ਹੈ। HTPC ਨੂੰ ਕੰਟਰੋਲ ਕਰਨਾ ਇਸ ਐਪ ਦਾ ਮੁੱਖ ਦ੍ਰਿਸ਼ ਹੈ ਜਿਸ ਲਈ ਇਹ ਸਭ ਤੋਂ ਵੱਧ ਅਨੁਕੂਲਿਤ ਕੀਤੀ ਗਈ ਹੈ।

ਇਹ ਐਪ ਟਰਮੀਨਲ ਇਮੀੂਲੇਟਰ ਨਹੀਂ ਹੈ, ਪਰ ਜ਼ਰੂਰਤ ਪੈਣ ਤੇ ਇਹ ਤੁਹਾਨੂੰ `apt-get install` ਚਲਾਉਣ ਦੇਵੇਗੀ।

## ਸ਼ੁਰੂਆਤ

ਜੇ ਤੁਸੀਂ ਕਨੈਕਟ ਕਰਨ ਲਈ SSH ਕੀ ਵਰਤਣਾ ਚਾਹੁੰਦੇ ਹੋ, ਤਾਂ ਪਹਿਲਾਂ ਐਪ ਦੀਆਂ ਸੈਟਿੰਗਾਂ ਖੋਲ੍ਹੋ ਅਤੇ ਇੱਕ ਕੀ ਆਯਾਤ ਕਰੋ ਜਾਂ ਤਿਆਰ ਕਰੋ।

ਹੇਠਾਂ ਸੱਜੇ ਕੋਨੇ ਵਿੱਚ `+` ਬਟਨ ਦਬਾ ਕੇ ਨਵਾਂ ਹੋਸਟ ਜੋੜੋ। ਕਨੈਕਸ਼ਨ ਵੇਰਵੇ ਭਰੋ ਅਤੇ ਸੰਭਾਲੋ।

ਜਦੋਂ ਤੁਸੀਂ ਪਹਿਲੀ ਵਾਰ ਕਨੈਕਟ ਕਰੋਗੇ, ਤੁਹਾਨੂੰ ਇੱਕ ਪ੍ਰੀਸੈਟ ਚੁਣਨ ਲਈ ਕਿਹਾ ਜਾਵੇਗਾ। ਇਹ ਚੋਣ ਰਿਮੋਟ ਕੰਟਰੋਲ ਦੇ ਬਟਨਾਂ ਨੂੰ ਵੱਖ-ਵੱਖ ਕਿਸਮ ਦੇ ਕੰਪਿਊਟਰਾਂ ਉੱਤੇ ਠੀਕ ਤਰ੍ਹਾਂ ਕੰਮ ਕਰਨ ਲਈ ਸੰਰਚਿਤ ਕਰਦੀ ਹੈ। ਉਪਲਬਧ ਪ੍ਰੀਸੈਟਾਂ ਦੇ ਵੇਰਵੇ ਲਈ ਹੇਠਾਂ ਵੇਖੋ। ਜੇ ਤੁਸੀਂ ਚਾਹੋ, ਤਾਂ `No preset` ਚੁਣ ਕੇ ਬਿਨਾਂ ਕਿਸੇ ਸੰਰਚਨਾ ਤੋਂ ਵੀ ਸ਼ੁਰੂ ਕਰ ਸਕਦੇ ਹੋ।

ਜੇ ਤੁਹਾਨੂੰ ਨਹੀਂ ਪਤਾ ਕਿ ਤੁਹਾਡਾ Linux ਕੰਪਿਊਟਰ X11 ਚਲਾ ਰਿਹਾ ਹੈ ਜਾਂ Wayland, ਤਾਂ ਟਰਮੀਨਲ ਵਿੱਚ ਇਹ ਚਲਾਓ:

```shell
echo $XDG_SESSION_TYPE
```

ਇਸ ਦਾ ਨਤੀਜਾ `x11` ਜਾਂ `wayland` ਹੋਣਾ ਚਾਹੀਦਾ ਹੈ। ਤੁਹਾਨੂੰ ਇਹ ਕਮਾਂਡ ਡੈਸਕਟਾਪ ਮਾਹੌਲ ਦੇ ਅੰਦਰ ਚਲਾਉਣੀ ਪਵੇਗੀ।

## ਰਿਮੋਟ ਕੰਟਰੋਲ

ਕਨੈਕਟ ਹੋਣ ਤੋਂ ਬਾਅਦ, ਤੁਸੀਂ ਕਮਾਂਡਾਂ ਭੇਜਣ ਲਈ ਰਿਮੋਟ ਕੰਟਰੋਲ ਇੰਟਰਫੇਸ ਵਰਤ ਸਕਦੇ ਹੋ। ਵੱਖ-ਵੱਖ ਇਨਪੁੱਟ ਢੰਗਾਂ ਤੱਕ ਪਹੁੰਚ ਲਈ ਟੈਬ ਬਦਲੋ।

ਹਰ ਬਟਨ ਦਬਾਉਣ ਨਾਲ ਹੋਸਟ ਉੱਤੇ ਇੱਕ ਕਮਾਂਡ ਚੱਲੇਗੀ। ਸਿਰਫ਼ ਇੱਕ ਕੀ ਦਬਾਉਣ ਵਰਗੀ ਸਧਾਰਣ ਕਾਰਵਾਈ ਲਈ ਇਹ ਕਾਫ਼ੀ ਵਾਧੂ ਪ੍ਰਕਿਰਿਆ ਹੈ, ਇਸ ਲਈ ਤੁਹਾਨੂੰ ਆਮ ਕੀਬੋਰਡ ਨਾਲੋਂ ਕਾਫ਼ੀ ਵੱਧ ਡੀਲੇ ਮਹਿਸੂਸ ਹੋ ਸਕਦਾ ਹੈ। ਮੈਨੂੰ ਆਸ ਹੈ ਕਿ ਭਵਿੱਖ ਦੇ ਵਰਜਨਾਂ ਵਿੱਚ ਮੈਂ ਇਸ ਨੂੰ ਹੋਰ ਬਿਹਤਰ ਕਰਾਂਗਾ।

ਸੋਧ ਮੋਡ ਵਿੱਚ ਜਾਣ ਲਈ ਮੀਨੂ ਵਰਤੋ। ਇਸ ਵੇਲੇ ਲੇਆਉਟ ਜਾਂ ਬਟਨਾਂ ਦੇ ਆਇਕਨ ਸੋਧਣਾ ਸੰਭਵ ਨਹੀਂ ਹੈ। ਮੈਨੂੰ ਆਸ ਹੈ ਕਿ ਭਵਿੱਖ ਵਿੱਚ ਇਹ ਸੰਭਵ ਬਣਾਵਾਂਗਾ।

## ਪ੍ਰੀਸੈਟ

ਤੁਹਾਨੂੰ ਆਪਣੇ ਡੈਸਕਟਾਪ ਮਾਹੌਲ ਲਈ ਲੋੜੀਂਦਾ ਟੂਲ ਇੰਸਟਾਲ ਕਰਨਾ ਪਵੇਗਾ।

ਮੈਂ `ydotool` ਦੀ ਸਿਫਾਰਸ਼ ਕਰਦਾ ਹਾਂ ਕਿਉਂਕਿ ਮੇਰੀ ਜਾਂਚ ਅਨੁਸਾਰ ਇਸ ਦੀ ਕਾਰਗੁਜ਼ਾਰੀ ਸਭ ਤੋਂ ਵਧੀਆ ਹੈ, ਅਤੇ ਇਹ X11 ਅਤੇ Wayland ਦੋਹਾਂ ਉੱਤੇ ਕੰਮ ਕਰਦਾ ਹੈ।

### ydotool

`ydotool` ਕਿਸੇ ਵੀ ਵਿੰਡੋ ਮੈਨੇਜਰ ਨਾਲ ਕੰਮ ਕਰਨਾ ਚਾਹੀਦਾ ਹੈ, ਪਰ ਤੁਹਾਨੂੰ ਬੈਕਗ੍ਰਾਊਂਡ ਸਰਵਿਸ ਚੱਲਦੀ ਹੋਣੀ ਲਾਜ਼ਮੀ ਹੈ। ਜੇ ਤੁਹਾਡਾ ਡਿਸਟ੍ਰੀਬਿਊਸ਼ਨ systemd ਯੂਜ਼ਰ ਸਰਵਿਸ ਪ੍ਰਦਾਨ ਕਰਦਾ ਹੈ, ਤਾਂ ਇਸਨੂੰ ਇਹ ਚਲਾ ਕੇ ਸ਼ੁਰੂ ਕਰੋ:

```shell
systemctl start --user ydotool
```

ਲੋਗਇਨ ਸਮੇਂ ਸਰਵਿਸ ਆਪਣੇ ਆਪ ਸ਼ੁਰੂ ਕਰਨ ਲਈ ਇਹ ਚਲਾਓ:

```shell
systemctl enable --user ydotool
```

ਕਿਰਪਾ ਕਰਕੇ ਯਕੀਨੀ ਬਣਾਓ ਕਿ ਤੁਸੀਂ `ydotool` ਦਾ ਕਾਫ਼ੀ ਨਵਾਂ ਵਰਜਨ ਇੰਸਟਾਲ ਕਰ ਰਹੇ ਹੋ। Ubuntu 26.04 ਤੋਂ ਪਹਿਲਾਂ ਦੇ ਵਰਜਨਾਂ ਵਿੱਚ ਦਿੱਤੇ ਵਰਜਨ ਬਹੁਤ ਪੁਰਾਣੇ ਹਨ। ਹੱਲ ਲਈ ਚਰਚਾ ਥ੍ਰੇਡ ਵੇਖੋ।

ਕਿਰਪਾ ਕਰਕੇ `ydotool` ਬਾਰੇ ਇਸ ਚਰਚਾ ਥ੍ਰੇਡ ਵਿੱਚ ਗੱਲ ਕਰੋ: <https://github.com/stefansundin/SSHRemote/discussions/5>

### xdotool

`xdotool` X11 ਚਲਾਉਣ ਵਾਲੇ ਕੰਪਿਊਟਰਾਂ ਲਈ ਹੈ। ਇਤਿਹਾਸਕ ਤੌਰ ਤੇ ਬਹੁਤ ਸਾਰੇ Linux ਕੰਪਿਊਟਰ X11 ਵਰਤਦੇ ਆਏ ਹਨ, ਹਾਲਾਂਕਿ Wayland ਹੁਣ ਹੋਰ ਲੋਕਪ੍ਰਿਯ ਹੋ ਰਿਹਾ ਹੈ।

X11 ਨਾਲ ਇੱਕ ਖਾਸ ਗੱਲ ਇਹ ਹੈ ਕਿ ਤੁਹਾਨੂੰ X ਸਰਵਰ ਲਈ ਪਹੁੰਚ ਦੀ ਆਗਿਆ ਦੇਣੀ ਪੈ ਸਕਦੀ ਹੈ। ਜੇ ਤੁਹਾਨੂੰ "Authorization required" ਗਲਤੀਆਂ ਮਿਲਦੀਆਂ ਹਨ ਤਾਂ ਸੰਭਵ ਹੈ ਇਹੀ ਸਮੱਸਿਆ ਹੋਵੇ। ਇਸਨੂੰ ਠੀਕ ਕਰਨ ਲਈ ਤੁਹਾਡੇ ਕੋਲ ਕੁਝ ਵਿਕਲਪ ਹਨ, ਇੱਥੇ ਦੋ ਅਜੇਹੇ ਵਿਕਲਪ ਹਨ ਜਿਹੜੇ ਮੇਰੇ ਲਈ ਕੰਮ ਕਰ ਚੁੱਕੇ ਹਨ:

ਜੇ `xauth list` ਕੋਈ ਐਂਟਰੀ ਨਹੀਂ ਦਿਖਾਉਂਦਾ, ਤਾਂ `.Xauthority` ਫਾਈਲ ਤਿਆਰ ਕਰਨ ਦੀ ਕੋਸ਼ਿਸ਼ ਕਰੋ:

```shell
xauth generate :0 . trusted
```

ਜੇ ਇਹ ਕੰਮ ਨਾ ਕਰੇ, ਤਾਂ `xhost` ਨਾਲ ਪਹੁੰਚ ਦੇਣ ਦੀ ਕੋਸ਼ਿਸ਼ ਕਰੋ:

```shell
xhost +local:$USER
```

ਤੁਹਾਨੂੰ ਹਰ ਬੂਟ ਤੋਂ ਬਾਅਦ `xhost` ਕਮਾਂਡ ਚਲਾਉਣੀ ਪਵੇਗੀ। ਤੁਸੀਂ ਇੱਕ bash ਸਕ੍ਰਿਪਟ ਬਣਾ ਕੇ ਅਤੇ ਲੋਗਇਨ ਵੇਲੇ ਆਪਣੇ ਆਪ ਚਲਣ ਲਈ ਸੈੱਟ ਕਰਕੇ ਇਸ ਨੂੰ ਆਟੋਮੇਟ ਕਰ ਸਕਦੇ ਹੋ।

ਕਿਰਪਾ ਕਰਕੇ `xdotool` ਬਾਰੇ ਇਸ ਚਰਚਾ ਥ੍ਰੇਡ ਵਿੱਚ ਗੱਲ ਕਰੋ: <https://github.com/stefansundin/SSHRemote/discussions/3>

### wtype

`wtype` `xdotool` ਵਰਗਾ ਹੈ, ਪਰ Wayland ਲਈ।

ਆਮ ਤੌਰ ਤੇ, ਇਹ ਮਾਊਸ ਕੰਟਰੋਲ ਦਾ ਸਮਰਥਨ ਨਹੀਂ ਕਰਦਾ, ਪਰ ਮੈਂ ਇੱਕ ਸੋਧਿਆ ਹੋਇਆ ਵਰਜਨ ਬਣਾਇਆ ਹੈ ਜੋ ਮਾਊਸ ਸਮਰਥਨ ਜੋੜਦਾ ਹੈ। ਜੇ ਤੁਹਾਨੂੰ ਮਾਊਸ ਸਮਰਥਨ ਚਾਹੀਦਾ ਹੈ, ਤਾਂ ਕਿਰਪਾ ਕਰਕੇ ਇਸਨੂੰ ਇੰਸਟਾਲ ਕਰੋ: <https://github.com/stefansundin/wtype>

ਜੇ ਤੁਹਾਨੂੰ `Compositor does not support the virtual keyboard protocol` ਗਲਤੀ ਮਿਲੇ, ਤਾਂ ਮੈਂ ਸੁਝਾਅ ਦੇਵਾਂਗਾ ਕਿ ਤੁਸੀਂ `ydotool` ਵਰਗਾ ਹੋਰ ਟੂਲ ਅਜ਼ਮਾਓ।

ਕਿਰਪਾ ਕਰਕੇ `wtype` ਬਾਰੇ ਇਸ ਚਰਚਾ ਥ੍ਰੇਡ ਵਿੱਚ ਗੱਲ ਕਰੋ: <https://github.com/stefansundin/SSHRemote/discussions/4>

### dotool

`dotool` ਕਿਸੇ ਵੀ ਵਿੰਡੋ ਮੈਨੇਜਰ ਨਾਲ ਕੰਮ ਕਰਨਾ ਚਾਹੀਦਾ ਹੈ, `ydotool` ਵਾਂਗ। ਮੇਰੀ ਸੀਮਿਤ ਜਾਂਚ ਵਿੱਚ ਇਹ `ydotool` ਨਾਲੋਂ ਕਾਫ਼ੀ ਹੌਲਾ ਸੀ।

ਕਿਰਪਾ ਕਰਕੇ `dotool` ਬਾਰੇ ਇਸ ਚਰਚਾ ਥ੍ਰੇਡ ਵਿੱਚ ਗੱਲ ਕਰੋ: <https://github.com/stefansundin/SSHRemote/discussions/6>

### cec-client

ਇਹ ਪ੍ਰੀਸੈਟ ਪ੍ਰਯੋਗਾਤਮਕ ਹੈ ਕਿਉਂਕਿ ਮੈਂ ਇਸਨੂੰ ਆਪਣੇ ਹਾਰਡਵੇਅਰ ਤੇ ਖੁਦ ਪੜਤਾਲ ਨਹੀਂ ਕਰ ਸਕਿਆ। ਤੁਹਾਡਾ ਫੀਡਬੈਕ ਸਵਾਗਤਯੋਗ ਹੈ।

ਕਿਰਪਾ ਕਰਕੇ `cec-client` ਬਾਰੇ ਇਸ ਚਰਚਾ ਥ੍ਰੇਡ ਵਿੱਚ ਗੱਲ ਕਰੋ: <https://github.com/stefansundin/SSHRemote/discussions/7>

### macOS

macOS ਉੱਤੇ VLC ਨੂੰ ਕੰਟਰੋਲ ਕਰਨ ਲਈ ਇੱਕ ਪ੍ਰੀਸੈਟ ਹੈ, ਜੋ AppleScript ਕਮਾਂਡਾਂ ਵਰਤਦਾ ਹੈ। ਮੈਨੂੰ ਕਿਸੇ ਅਜੇਹੇ ਟੂਲ ਦੀ ਜਾਣਕਾਰੀ ਨਹੀਂ ਜੋ ਕੀਬੋਰਡ ਜਾਂ ਮਾਊਸ ਇਵੈਂਟ ਭੇਜਣ ਦਾ ਸਮਰਥਨ ਕਰਦਾ ਹੋਵੇ।

ਕਿਰਪਾ ਕਰਕੇ macOS ਬਾਰੇ ਇਸ ਚਰਚਾ ਥ੍ਰੇਡ ਵਿੱਚ ਗੱਲ ਕਰੋ: <https://github.com/stefansundin/SSHRemote/discussions/8>

### Android

ਜੇ ਤੁਹਾਡੇ Android ਡਿਵਾਈਸ ਵਿੱਚ SSH ਸਰਵਰ ਆਉਂਦਾ ਹੈ, ਤਾਂ ਹੋ ਸਕਦਾ ਹੈ ਤੁਸੀਂ ਇਸ ਨਾਲ ਕਨੈਕਟ ਹੋ ਸਕੋ ਅਤੇ ਇਨਪੁੱਟ ਇਵੈਂਟ ਭੇਜ ਸਕੋ। ਇਹ ਹੋਰ ਸੰਭਾਵਨਾ ਹੈ ਜੇ ਤੁਹਾਡੀ ਡਿਵਾਈਸ Raspberry Pi ਉੱਤੇ KonstaKANG ਵਰਗਾ ਕਸਟਮ ROM ਚਲਾ ਰਹੀ ਹੋਵੇ।

ਮੈਂ ਅਜੇ ਤੱਕ ਮਾਊਸ ਸਮਰਥਨ ਨੂੰ ਕੰਮ ਕਰਨ ਲਾਇਕ ਨਹੀਂ ਬਣਾ ਸਕਿਆ।

ਕਿਰਪਾ ਕਰਕੇ Android ਬਾਰੇ ਇਸ ਚਰਚਾ ਥ੍ਰੇਡ ਵਿੱਚ ਗੱਲ ਕਰੋ: <https://github.com/stefansundin/SSHRemote/discussions/9>

## ਸਮਾਰਟ ਵਾਲਿਊਮ ਕੰਟਰੋਲ

ਰਿਮੋਟ ਕੰਟਰੋਲ ਸੋਧਦੇ ਸਮੇਂ, ਤੁਸੀਂ ਮੀਨੂ ਵਿੱਚ "smart" ਵਾਲਿਊਮ ਕੰਟਰੋਲ ਸੈਟਿੰਗਾਂ ਲੱਭ ਸਕਦੇ ਹੋ। ਇਹ ਐਪ ਵਿੱਚ ਕੰਪਿਊਟਰ ਦੀ ਮੌਜੂਦਾ ਆਵਾਜ਼ ਦਿਖਾ ਸਕਦੀਆਂ ਹਨ ਅਤੇ ਸਲਾਈਡਰ ਦੀ ਮਦਦ ਨਾਲ ਤੁਹਾਨੂੰ ਜਲਦੀ ਨਾਲ ਆਵਾਜ਼ ਸੈੱਟ ਕਰਨ ਦੇਂਦੀਆਂ ਹਨ। ਤੁਸੀਂ ਆਪਣੇ ਡਿਵਾਈਸ ਦੇ ਹਾਰਡਵੇਅਰ ਬਟਨਾਂ ਦੀ ਮਦਦ ਨਾਲ ਵਾਲਿਊਮ ਵਧਾਉਣ/ਘਟਾਉਣ ਵਾਲੀਆਂ ਕਮਾਂਡਾਂ ਵੀ ਜਲਦੀ ਭੇਜ ਸਕਦੇ ਹੋ।

ਮੌਜੂਦਾ ਆਵਾਜ਼ ਪੜ੍ਹਨਾ ਅਤੇ ਸਲਾਈਡਰ ਰਾਹੀਂ ਨਵੀਂ ਆਵਾਜ਼ ਸੈੱਟ ਕਰਨਾ ਇਸ ਵੇਲੇ `pactl` ਵਰਤਣ ਲਈ ਹਾਰਡ-ਕੋਡ ਕੀਤਾ ਗਿਆ ਹੈ।

`pactl` ਵਾਲਾ ਪੈਕੇਜ ਆਮ ਤੌਰ ਤੇ `pulseaudio-utils` ਜਾਂ `libpulse` ਕਿਹਾ ਜਾਂਦਾ ਹੈ।

## SSH ਕੀਆਂ

ਤੁਸੀਂ ਐਪ ਦੀਆਂ ਸੈਟਿੰਗਾਂ ਵਿੱਚ SSH ਕੀਆਂ ਆਯਾਤ ਕਰ ਸਕਦੇ ਹੋ ਜਾਂ ਤਿਆਰ ਕਰ ਸਕਦੇ ਹੋ। SSH ਕੀ ਨਾਲ ਕਨੈਕਟ ਕਰਨਾ ਪਾਸਵਰਡ ਵਰਤਣ ਨਾਲੋਂ ਜ਼ਿਆਦਾ ਸੁਰੱਖਿਅਤ ਹੈ।

ਕਿਸੇ ਕੰਪਿਊਟਰ ਤੋਂ ਮੌਜੂਦਾ SSH ਕੀ ਆਯਾਤ ਕਰਨ ਦਾ ਸਭ ਤੋਂ ਆਸਾਨ ਤਰੀਕਾ QR ਕੋਡ ਸਕੈਨ ਕਰਨਾ ਹੈ। ਤੁਸੀਂ QR ਕੋਡ ਚਿੱਤਰ ਤਿਆਰ ਕਰਨ ਲਈ `qrencode` ਪ੍ਰੋਗਰਾਮ ਵਰਤ ਸਕਦੇ ਹੋ। QR ਕੋਡ ਤਿਆਰ ਕਰਨ ਲਈ ਹੇਠਾਂ ਦਿੱਤੀ ਵਰਗੀ ਕਮਾਂਡ ਚਲਾਓ:

```shell
# ਆਪਣੀਆਂ SSH ਕੀਆਂ ਦੇ ਫੋਲਡਰ ਵਿੱਚ ਜਾਓ:
cd ~/.ssh

# ਟਰਮੀਨਲ ਵਿੱਚ QR ਕੋਡ ਦਿਖਾਓ:
qrencode -r id_ed25519 -t ansiutf8

# ਨਹੀਂ ਤਾਂ, ਇੱਕ ਇਮੇਜ ਫਾਈਲ ਬਣਾਓ:
qrencode -r id_ed25519 -o qr.png

# 4096-ਬਿੱਟ RSA ਕੀਆਂ QR ਕੋਡ ਵਿੱਚ ਫਿੱਟ ਨਹੀਂ ਹੁੰਦੀਆਂ। gzip ਨਾਲ ਉਹ ਬਸ-ਬਸ ਫਿੱਟ ਹੋ ਸਕਦੀਆਂ ਹਨ:
cat id_rsa_4096 | gzip | base64 | qrencode -o qr.png
```

ਤੁਸੀਂ ਮੀਨੂ ਵਿੱਚ `ਪਬਲਿਕ ਕੀ ਭੇਜੋ` ਵਿਸ਼ੇਸ਼ਤਾ ਦੀ ਵਰਤੋਂ ਕਰਕੇ ਸਰਵਰ ਉੱਤੇ ਪਬਲਿਕ SSH ਕੀਆਂ ਭੇਜ ਸਕਦੇ ਹੋ। ਇਹ ਚੁਣੀ ਹੋਈ SSH ਕੀ ਨੂੰ `~/.ssh/authorized_keys` ਫਾਈਲ ਵਿੱਚ ਜੋੜ ਦੇਵੇਗੀ। ਇਸ ਨਾਲ ਤੁਸੀਂ ਪਾਸਵਰਡ ਨਾਲ ਲੌਗਇਨ ਕਰਨ ਤੋਂ SSH ਕੀ ਨਾਲ ਲੌਗਇਨ ਕਰਨ ਵੱਲ ਆਸਾਨੀ ਨਾਲ ਜਾ ਸਕਦੇ ਹੋ।

ਤੁਸੀਂ ਇਨਕ੍ਰਿਪਟ ਕੀਤੀਆਂ SSH ਕੀਆਂ ਆਯਾਤ ਕਰਕੇ ਵਰਤ ਸਕਦੇ ਹੋ, ਪਰ ਇਸ ਵੇਲੇ ਐਪ ਵਿੱਚ ਇਨ੍ਹਾਂ ਨੂੰ ਤਿਆਰ ਨਹੀਂ ਕੀਤਾ ਜਾ ਸਕਦਾ।

## ਸੁਰੱਖਿਆ

ਐਪ ਵਿੱਚੋਂ SSH ਕੀਆਂ ਦੇ ਪ੍ਰਾਈਵੇਟ ਹਿੱਸੇ ਜਾਂ ਸੰਭਾਲੇ ਪਾਸਵਰਡ ਨਿਰਯਾਤ ਕਰਨਾ ਜਾਂ ਕੱਢਣਾ ਸੰਭਵ ਨਹੀਂ ਹੈ। ਇਹ ਡਾਟਾ 256-ਬਿੱਟ AES ਨਾਲ ਇਨਕ੍ਰਿਪਟ ਕੀਤਾ ਜਾਂਦਾ ਹੈ, ਅਤੇ ਇਨਕ੍ਰਿਪਸ਼ਨ ਕੀ Android Keystore ਵਿੱਚ ਸੰਭਾਲੀ ਜਾਂਦੀ ਹੈ। ਇਨਕ੍ਰਿਪਟ ਕੀਤਾ ਡਾਟਾ Android ਬੈਕਅੱਪ ਤੋਂ ਬਾਹਰ ਰੱਖਿਆ ਜਾਂਦਾ ਹੈ।

ਇਸ ਐਪ ਵਿੱਚ ਕੋਈ ਕਰੈਸ਼ ਰਿਪੋਰਟਿੰਗ ਸੌਫਟਵੇਅਰ ਨਹੀਂ ਹੈ। ਕੋਈ ਟੈਲੀਮੇਟਰੀ ਨਹੀਂ ਹੈ। ਕੋਈ ਵਿਗਿਆਪਨ ਨਹੀਂ ਹਨ। SSH ਕਨੈਕਸ਼ਨ ਤੋਂ ਇਲਾਵਾ ਕੋਈ ਹੋਰ ਨੈੱਟਵਰਕ ਬੇਨਤੀ ਨਹੀਂ ਕੀਤੀ ਜਾਂਦੀ।

ਇਸ ਐਪ ਦੀ ਸੁਰੱਖਿਆ ਦਾ ਅਜੇ ਔਡਿਟ ਨਹੀਂ ਹੋਇਆ। ਜੇ ਤੁਹਾਨੂੰ Android ਸੁਰੱਖਿਆ ਜਾਂ SSH ਸੁਰੱਖਿਆ ਦਾ ਅਨੁਭਵ ਹੈ, ਤਾਂ ਕਿਰਪਾ ਕਰਕੇ ਸੋਰਸ ਕੋਡ ਵੇਖੋ ਅਤੇ ਆਪਣੇ ਨਤੀਜੇ ਇਸ GitHub issue ਵਿੱਚ ਦਰਜ ਕਰੋ:

<https://github.com/stefansundin/SSHRemote/issues/1>
