## เกี่ยวกับ SSH Remote

คำแปลภาษาไทยนี้จัดทำด้วย AI โดย GitHub Copilot และอาจมีข้อผิดพลาดในการแปลอยู่บ้าง หากมีข้อสงสัย โปรดตรวจสอบกับต้นฉบับภาษาอังกฤษด้วย

SSH Remote เป็นแอปฟรีและโอเพนซอร์สที่ให้คุณควบคุมคอมพิวเตอร์จากระยะไกลผ่าน SSH ได้

คุณสามารถปรับแต่งคำสั่งที่จะถูกเรียกใช้ได้ทั้งหมด และมีพรีเซ็ตสำหรับการตั้งค่าทั่วไปให้เลือก

ฉันใช้แอปนี้เพื่อควบคุมชุด HTPC ของตัวเอง ซึ่งรันบน Raspberry Pi OS การควบคุม HTPC เป็นสถานการณ์หลักที่แอปนี้ถูกออกแบบมาให้เหมาะสมที่สุด

แอปนี้ไม่ใช่โปรแกรมจำลองเทอร์มินัล แต่ก็ยังช่วยให้คุณรัน `apt-get install` ได้ในยามฉุกเฉิน

## เริ่มต้นใช้งาน

หากคุณต้องการใช้คีย์ SSH ในการเชื่อมต่อ ให้เปิดการตั้งค่าแอปก่อน แล้วนำเข้าหรือสร้างคีย์

เพิ่มโฮสต์ใหม่โดยแตะปุ่ม `+` ที่มุมขวาล่าง กรอกรายละเอียดการเชื่อมต่อ แล้วบันทึก

ครั้งแรกที่คุณเชื่อมต่อ คุณจะถูกขอให้เลือกพรีเซ็ต การเลือกนี้จะตั้งค่าปุ่มต่างๆ บนรีโมตคอนโทรลให้ทำงานได้ดีกับคอมพิวเตอร์หลายประเภท ดูคำอธิบายของพรีเซ็ตที่มีอยู่ด้านล่าง หากต้องการ คุณสามารถเริ่มต้นโดยไม่มีการตั้งค่าใดๆ ได้ด้วยการเลือก `No preset`

หากคุณไม่ทราบว่าคอมพิวเตอร์ Linux ของคุณกำลังรัน X11 หรือ Wayland ให้รันคำสั่งนี้ในเทอร์มินัล:

```shell
echo $XDG_SESSION_TYPE
```

ผลลัพธ์ควรเป็น `x11` หรือ `wayland` คุณต้องรันคำสั่งนี้ภายในเดสก์ท็อปเอนวายรอนเมนต์

## รีโมตคอนโทรล

เมื่อเชื่อมต่อแล้ว คุณสามารถใช้อินเทอร์เฟซรีโมตคอนโทรลเพื่อส่งคำสั่งได้ สลับแท็บเพื่อเข้าถึงวิธีป้อนข้อมูลแบบต่างๆ

การกดปุ่มแต่ละครั้งจะเรียกใช้คำสั่งบนโฮสต์ ซึ่งมีค่าใช้จ่ายค่อนข้างมากสำหรับสิ่งที่ง่ายอย่างการกดปุ่มหนึ่งครั้ง และคุณอาจพบความหน่วงค่อนข้างสูงเมื่อเทียบกับแป้นพิมพ์ปกติ ฉันหวังว่าจะปรับปรุงเรื่องนี้ได้ในเวอร์ชันต่อๆ ไป

ใช้เมนูเพื่อเข้าสู่โหมดแก้ไข ตอนนี้ยังไม่สามารถแก้ไขเลย์เอาต์หรือไอคอนของปุ่มได้ ฉันหวังว่าจะทำให้สิ่งนี้เป็นไปได้ในเวอร์ชันอนาคต

## พรีเซ็ต

คุณจะต้องติดตั้งเครื่องมือที่จำเป็นสำหรับเดสก์ท็อปเอนวายรอนเมนต์ของคุณ

ฉันแนะนำ `ydotool` เพราะจากการทดสอบของฉัน มันให้ประสิทธิภาพดีที่สุด และใช้งานได้ทั้งบน X11 และ Wayland

### ydotool

`ydotool` ควรทำงานได้กับตัวจัดการหน้าต่างทุกแบบ แต่คุณต้องมีบริการเบื้องหลังที่กำลังทำงานอยู่ หากดิสทริบิวชันของคุณมี systemd user service ให้เริ่มต้นโดยรัน:

```shell
systemctl start --user ydotool
```

ให้บริการเริ่มอัตโนมัติเมื่อเข้าสู่ระบบโดยรัน:

```shell
systemctl enable --user ydotool
```

โปรดตรวจสอบให้แน่ใจว่าคุณกำลังติดตั้ง `ydotool` เวอร์ชันที่ใหม่พอ Ubuntu เวอร์ชันก่อน 26.04 มีเวอร์ชันที่เก่าเกินไป โปรดดูเธรดสนทนาสำหรับวิธีแก้ปัญหา

โปรดพูดคุยเรื่อง `ydotool` ในเธรดสนทนานี้: <https://github.com/stefansundin/SSHRemote/discussions/5>

### xdotool

`xdotool` สำหรับคอมพิวเตอร์ที่รัน X11 โดย X11 คือสิ่งที่คอมพิวเตอร์ Linux ส่วนใหญ่ใช้กันมาในอดีต แม้ว่า Wayland จะได้รับความนิยมมากขึ้นเรื่อยๆ

ข้อสังเกตอย่างหนึ่งของ X11 คือคุณอาจต้องอนุญาตการเข้าถึง X server หากคุณพบข้อผิดพลาด "Authorization required" นี่คือสาเหตุ คุณมีหลายวิธีในการแก้ปัญหานี้ ด้านล่างนี้คือสองวิธีที่ใช้ได้ผลสำหรับฉัน:

หาก `xauth list` ไม่แสดงรายการใดๆ ให้ลองสร้างไฟล์ `.Xauthority`:

```shell
xauth generate :0 . trusted
```

หากยังไม่ได้ผล ให้ลองอนุญาตการเข้าถึงโดยใช้ `xhost`:

```shell
xhost +local:$USER
```

คุณจะต้องรันคำสั่ง `xhost` หลังจากบูตทุกครั้ง คุณสามารถทำให้เป็นอัตโนมัติได้โดยสร้างสคริปต์ bash แล้วตั้งค่าให้เริ่มอัตโนมัติเมื่อเข้าสู่ระบบ

โปรดพูดคุยเรื่อง `xdotool` ในเธรดสนทนานี้: <https://github.com/stefansundin/SSHRemote/discussions/3>

### wtype

`wtype` คล้ายกับ `xdotool` แต่สำหรับ Wayland

ตามปกติแล้ว มันไม่รองรับการควบคุมเมาส์ แต่ฉันได้สร้างเวอร์ชันดัดแปลงที่เพิ่มการรองรับเมาส์ไว้ โปรดติดตั้งหากคุณต้องการการรองรับเมาส์: <https://github.com/stefansundin/wtype>

หากคุณพบข้อผิดพลาด `Compositor does not support the virtual keyboard protocol` ฉันแนะนำให้ลองใช้เครื่องมืออื่น เช่น `ydotool`

โปรดพูดคุยเรื่อง `wtype` ในเธรดสนทนานี้: <https://github.com/stefansundin/SSHRemote/discussions/4>

### dotool

`dotool` ควรทำงานได้กับตัวจัดการหน้าต่างทุกแบบ คล้ายกับ `ydotool` จากการทดสอบที่มีอยู่อย่างจำกัดของฉัน มันช้ากว่า `ydotool` มาก

โปรดพูดคุยเรื่อง `dotool` ในเธรดสนทนานี้: <https://github.com/stefansundin/SSHRemote/discussions/6>

### cec-client

พรีเซ็ตนี้ยังอยู่ในขั้นทดลอง เพราะฉันยังไม่สามารถยืนยันการทำงานกับฮาร์ดแวร์ของตัวเองได้ ยินดีรับคำติชม

โปรดพูดคุยเรื่อง `cec-client` ในเธรดสนทนานี้: <https://github.com/stefansundin/SSHRemote/discussions/7>

### macOS

มีพรีเซ็ตสำหรับควบคุม VLC บน macOS โดยใช้คำสั่ง AppleScript ฉันยังไม่ทราบถึงเครื่องมือที่รองรับการส่งเหตุการณ์จากแป้นพิมพ์หรือเมาส์

โปรดพูดคุยเรื่อง macOS ในเธรดสนทนานี้: <https://github.com/stefansundin/SSHRemote/discussions/8>

### Android

หากอุปกรณ์ Android ของคุณมาพร้อมกับ SSH server คุณอาจสามารถเชื่อมต่อกับมันและส่งเหตุการณ์อินพุตได้ เรื่องนี้มีแนวโน้มเป็นไปได้มากขึ้นหากอุปกรณ์ของคุณรัน custom ROM เช่น KonstaKANG บน Raspberry Pi

ฉันยังหาวิธีให้การรองรับเมาส์ทำงานไม่ได้

โปรดพูดคุยเรื่อง Android ในเธรดสนทนานี้: <https://github.com/stefansundin/SSHRemote/discussions/9>

## การควบคุมระดับเสียงอัจฉริยะ

เมื่อแก้ไขรีโมตคอนโทรล คุณจะพบการตั้งค่าการควบคุมระดับเสียงแบบ "อัจฉริยะ" ได้ในเมนู ฟีเจอร์นี้สามารถแสดงระดับเสียงปัจจุบันของคอมพิวเตอร์ในแอป และให้คุณตั้งค่าระดับเสียงอย่างรวดเร็วด้วยแถบเลื่อน คุณยังสามารถใช้ปุ่มฮาร์ดแวร์ของอุปกรณ์เพื่อส่งคำสั่งเพิ่ม/ลดระดับเสียงได้อย่างรวดเร็ว

การอ่านระดับเสียงปัจจุบันและการตั้งค่าระดับเสียงใหม่ด้วยแถบเลื่อน ปัจจุบันถูกกำหนดให้ใช้ `pactl` แบบตายตัว

แพ็กเกจที่มี `pactl` มักมีชื่อว่า `pulseaudio-utils` หรือ `libpulse`

## คีย์ SSH

คุณสามารถนำเข้าหรือสร้างคีย์ SSH ได้ในการตั้งค่าแอป การเชื่อมต่อด้วยคีย์ SSH ปลอดภัยกว่าการใช้รหัสผ่าน

วิธีที่ง่ายที่สุดในการนำเข้าคีย์ SSH ที่มีอยู่จากคอมพิวเตอร์คือการสแกน QR code คุณสามารถใช้โปรแกรม `qrencode` เพื่อสร้างภาพ QR code ได้ รันคำสั่งลักษณะดังต่อไปนี้เพื่อสร้าง QR code:

```shell
# ไปยังโฟลเดอร์คีย์ SSH ของคุณ:
cd ~/.ssh

# แสดง QR code ในเทอร์มินัล:
qrencode -r id_ed25519 -t ansiutf8

# หรือสร้างเป็นไฟล์ภาพ:
qrencode -r id_ed25519 -o qr.png

# คีย์ RSA ขนาด 4096 บิต ใหญ่เกินไปสำหรับ QR code คุณสามารถใช้ gzip เพื่อให้พอใส่ได้แบบเฉียดๆ:
cat id_rsa_4096 | gzip | base64 | qrencode -o qr.png
```

คุณสามารถส่งคีย์สาธารณะไปยังเซิร์ฟเวอร์ได้ด้วยฟีเจอร์ `ส่งคีย์สาธารณะ` ในเมนู ระบบจะนำคีย์ SSH ที่เลือกไปต่อท้ายในไฟล์ `~/.ssh/authorized_keys` วิธีนี้ช่วยให้คุณย้ายจากการล็อกอินด้วยรหัสผ่านไปเป็นการล็อกอินด้วยคีย์ SSH ได้อย่างง่ายดาย

คุณสามารถนำเข้าและใช้คีย์ SSH ที่เข้ารหัสได้ แต่ตอนนี้ยังไม่สามารถสร้างคีย์ประเภทนี้ในแอปได้

## ความปลอดภัย

ไม่สามารถส่งออกหรือดึงส่วน private ของคีย์ SSH หรือรหัสผ่านที่จัดเก็บไว้ ออกจากแอปได้ ข้อมูลนี้ถูกเข้ารหัสด้วย AES 256 บิต และกุญแจเข้ารหัสจะถูกเก็บไว้ใน Android Keystore ข้อมูลที่เข้ารหัสจะไม่ถูกรวมอยู่ในการสำรองข้อมูลของ Android

แอปนี้ไม่มีซอฟต์แวร์รายงานการแครช ไม่มี telemetry ไม่มีโฆษณา และไม่มีการร้องขอเครือข่ายใดๆ นอกเหนือจากการเชื่อมต่อ SSH

ความปลอดภัยของแอปนี้ยังไม่ได้ผ่านการตรวจสอบ หากคุณมีประสบการณ์ด้านความปลอดภัย Android หรือ SSH โปรดช่วยดูซอร์สโค้ดและรายงานสิ่งที่คุณพบใน GitHub issue นี้:

<https://github.com/stefansundin/SSHRemote/issues/1>
