## SSH Remote-ի մասին

Այս թարգմանությունը կատարվել է AI-ի միջոցով (GitHub Copilot, GPT-5-Codex), ուստի հնարավոր են թարգմանական կամ ձևակերպման սխալներ։

SSH Remote-ը անվճար և բաց կոդով հավելված է, որը թույլ է տալիս հեռակառավարել համակարգիչներ SSH-ի միջոցով։

Դուք կարող եք ամբողջությամբ հարմարեցնել կատարվող հրամանները, և կան նախադրվածքներ սովորական օգտագործման տարբերակների համար։

Ես այս հավելվածը օգտագործում եմ իմ HTPC համակարգը կառավարելու համար, որը աշխատում է Raspberry Pi OS-ով։ HTPC-ի կառավարումը հիմնական սցենարն է, որի համար հավելվածը օպտիմիզացված է։

Այս հավելվածը տերմինալ էմուլյատոր չէ, բայց արտակարգ իրավիճակում թույլ կտա գործարկել `apt-get install`։

## Սկսելու համար

Եթե ուզում եք միանալ SSH բանալիով, նախ բացեք հավելվածի կարգավորումները և ներմուծեք կամ ստեղծեք բանալի։

Ավելացրեք նոր հոսթ՝ ներքևի աջ անկյունում սեղմելով `+` կոճակը։ Մուտքագրեք կապի տվյալները և պահպանեք։

Առաջին կապի ժամանակ ձեզ կառաջարկվի ընտրել նախադրվածք։ Այդ ընտրությունը կարգավորում է հեռակառավարման կոճակները, որպեսզի լավ աշխատեն տարբեր տեսակի համակարգիչների վրա։ Ստորև տեսեք հասանելի նախադրվածքների նկարագրությունը։ Ցանկության դեպքում կարող եք սկսել առանց կարգավորման՝ ընտրելով `No preset`։

Եթե չգիտեք՝ ձեր Linux համակարգիչը աշխատում է X11-ո՞վ, թե Wayland-ով, տերմինալում գործարկեք՝

```shell
echo $XDG_SESSION_TYPE
```

Արդյունքում պետք է ստանաք `x11` կամ `wayland`։ Սա պետք է գործարկել հենց desktop միջավայրի ներսում։

## Հեռակառավարում

Կապվելուց հետո կարող եք օգտագործել հեռակառավարման միջերեսը՝ հրամաններ ուղարկելու համար։ Տարբեր մուտքագրման եղանակների համար փոխեք ներդիրները։

Յուրաքանչյուր կոճակի սեղմում կհանգեցնի հոսթի վրա հրամանի կատարմանը։ Սա բավական մեծ ծախս է նման պարզ գործողության համար, ինչպիսին ստեղնի սեղմումն է, և կարող եք զգալ բավական բարձր ուշացում՝ սովորական ստեղնաշարի համեմատ։ Հույս ունեմ սա բարելավել ապագա տարբերակներում։

Խմբագրման ռեժիմ մտնելու համար օգտագործեք մենյուն։ Ներկայում հնարավոր չէ խմբագրել դասավորությունը կամ կոճակների պատկերակները։ Հույս ունեմ սա նույնպես ավելացնել ապագայում։

## Նախադրվածքներ

Ձեզ պետք է տեղադրել այն գործիքը, որը պահանջվում է ձեր desktop միջավայրի համար։

Խորհուրդ եմ տալիս `ydotool`, որովհետև իմ փորձարկումներում այն ապահովում է լավագույն արդյունավետությունը և աշխատում է ինչպես X11-ի, այնպես էլ Wayland-ի հետ։

### ydotool

`ydotool`-ը պետք է աշխատի ցանկացած window manager-ի հետ, բայց անհրաժեշտ է աշխատող հետին ծառայություն։ Եթե ձեր բաշխումն ունի systemd user service, ապա գործարկեք այն՝

```shell
systemctl start --user ydotool
```

Մուտքի ժամանակ ծառայությունը ավտոմատ գործարկելու համար՝

```shell
systemctl enable --user ydotool
```

Խնդրում ենք համոզվել, որ տեղադրում եք բավարար նոր տարբերակ `ydotool`։ Ubuntu-ի 26.04-ից հին տարբերակները տրամադրում են չափազանց հին տարբերակներ։ Շրջանցման համար տեսեք քննարկման թեման։

`ydotool`-ի քննարկումը վարեք այստեղ՝ <https://github.com/stefansundin/SSHRemote/discussions/5>

### xdotool

`xdotool`-ը նախատեսված է X11-ով աշխատող համակարգիչների համար։ Պատմականորեն Linux համակարգիչների մեծ մասը օգտագործել է X11, թեև Wayland-ը դառնում է ավելի տարածված։

X11-ի մի առանձնահատկություն այն է, որ հնարավոր է պետք լինի թույլատրել մուտքը X server-ին։ Եթե ստանում եք "Authorization required" սխալ, ապա խնդիրը հենց դա է։ Կան մի քանի տարբերակ, ահա երկուսը, որոնք ինձ մոտ աշխատել են՝

Եթե `xauth list`-ը որևէ գրառում չի ցույց տալիս, փորձեք ստեղծել `.Xauthority` ֆայլ՝

```shell
xauth generate :0 . trusted
```

Եթե դա չաշխատի, փորձեք մուտք թույլատրել `xhost`-ով՝

```shell
xhost +local:$USER
```

`xhost` հրամանը պետք է գործարկեք ամեն բեռնումից հետո։ Կարող եք ավտոմատացնել սա՝ ստեղծելով bash script և կարգավորելով այն ավտոմատ մեկնարկի համար։

`xdotool`-ի քննարկումը վարեք այստեղ՝ <https://github.com/stefansundin/SSHRemote/discussions/3>

### wtype

`wtype`-ը նման է `xdotool`-ին, բայց Wayland-ի համար։

Սովորաբար այն չի աջակցում մկնիկի կառավարում, բայց ես ստեղծել եմ փոփոխված տարբերակ, որը ավելացնում է մկնիկի աջակցություն։ Եթե մկնիկի աջակցություն է պետք, տեղադրեք այն՝ <https://github.com/stefansundin/wtype>

Եթե ստանում եք `Compositor does not support the virtual keyboard protocol` սխալը, խորհուրդ եմ տալիս փորձել այլ գործիք, օրինակ՝ `ydotool`։

`wtype`-ի քննարկումը վարեք այստեղ՝ <https://github.com/stefansundin/SSHRemote/discussions/4>

### dotool

`dotool`-ը պետք է աշխատի ցանկացած window manager-ի հետ, ինչպես `ydotool`-ը։ Իմ սահմանափակ փորձարկումներում այն զգալիորեն դանդաղ էր `ydotool`-ից։

`dotool`-ի քննարկումը վարեք այստեղ՝ <https://github.com/stefansundin/SSHRemote/discussions/6>

### cec-client

Այս նախադրվածքը փորձարարական է, քանի որ չեմ կարողացել ստուգել այն իմ սեփական սարքավորման վրա։ Հետադարձ կապը ողջունելի է։

`cec-client`-ի քննարկումը վարեք այստեղ՝ <https://github.com/stefansundin/SSHRemote/discussions/7>

### macOS

macOS-ի համար կա նախադրվածք, որը VLC-ն կառավարում է AppleScript հրամաններով։ Ինձ հայտնի չէ որևէ գործիք, որը աջակցում է ստեղնաշարի կամ մկնիկի իրադարձությունների ուղարկում։

macOS-ի քննարկումը վարեք այստեղ՝ <https://github.com/stefansundin/SSHRemote/discussions/8>

### Android

Եթե ձեր Android սարքն ունի SSH սերվեր, ապա գուցե կարողանաք միանալ և ուղարկել մուտքային իրադարձություններ։ Սա ավելի հավանական է, եթե սարքը աշխատում է custom ROM-ով, օրինակ՝ KonstaKANG Raspberry Pi-ի վրա։

Դեռ չեմ կարողացել հասկանալ՝ ինչպես աշխատեցնել մկնիկի աջակցությունը։

Android-ի քննարկումը վարեք այստեղ՝ <https://github.com/stefansundin/SSHRemote/discussions/9>

## Խելացի ձայնի կառավարում

Հեռակառավարումը խմբագրելիս մենյուում կարող եք գտնել "խելացի" ձայնի կարգավորումները։ Դրանք կարող են հավելվածում ցույց տալ համակարգչի ընթացիկ ձայնը և թույլ տալ արագ փոխել ձայնը սահիչով։ Կարող եք նաև օգտագործել սարքի ֆիզիկական կոճակները՝ ձայնը բարձրացնելու/նվազեցնելու հրամաններ արագ ուղարկելու համար։

Ընթացիկ ձայնի ընթերցումը և սահիչով նոր ձայնի սահմանումը այս պահին կոշտ կերպով կապված են `pactl`-ի հետ։

`pactl` պարունակող փաթեթը սովորաբար կոչվում է `pulseaudio-utils` կամ `libpulse`։

## SSH բանալիներ

Հավելվածի կարգավորումներում կարող եք ներմուծել կամ ստեղծել SSH բանալիներ։ SSH բանալիով կապվելն ավելի անվտանգ է, քան գաղտնաբառով։

Գոյություն ունեցող SSH բանալի ներմուծելու ամենահեշտ ճանապարհը QR կոդ սկանավորելն է։ QR կոդի պատկեր ստեղծելու համար կարող եք օգտագործել `qrencode` ծրագիրը։ Օրինակ՝

```shell
# Անցեք ձեր SSH բանալիների թղթապանակը:
cd ~/.ssh

# Ցույց տվեք QR կոդը տերմինալում:
qrencode -r id_ed25519 -t ansiutf8

# Կամ ստեղծեք նկար:
qrencode -r id_ed25519 -o qr.png

# 4096-բիթ RSA բանալիները չափազանց մեծ են QR կոդի համար։ gzip-ով կարելի է հազիվ տեղավորել մեկը.
cat id_rsa_4096 | gzip | base64 | qrencode -o qr.png
```

Հանրային SSH բանալիները կարող եք ուղարկել սերվեր `Ուղարկել հանրային բանալի` գործառույթով՝ մենյուից։ Սա ընտրված SSH բանալին կավելացնի `~/.ssh/authorized_keys` ֆայլին։ Այսպիսով հեշտ է գաղտնաբառային մուտքից անցնել SSH բանալիով մուտքի։

Կարող եք ներմուծել և օգտագործել կոդավորված SSH բանալիներ, բայց ներկայում հավելվածը դրանք ստեղծել չի կարող։

## Անվտանգություն

Հնարավոր չէ հավելվածից արտահանել կամ հանել SSH բանալիների մասնավոր մասը կամ պահպանված գաղտնաբառերը։ Այս տվյալները կոդավորվում են 256-բիթ AES-ով, իսկ կոդավորման բանալին պահվում է Android Keystore-ում։ Կոդավորված տվյալները Android կրկնօրինակների մեջ չեն ներառվում։

Այս հավելվածում crash reporting ծրագիրակազմ չկա։ Telemetry չկա։ Գովազդ չկա։ Ցանցային հարցումներ էլ չկան, բացի SSH կապից։

Այս հավելվածի անվտանգությունը չի անցել audit։ Եթե փորձառու եք Android կամ SSH անվտանգության ոլորտում, խնդրում ենք նայել աղբյուրային կոդը և ձեր դիտարկումները գրել այս GitHub issue-ում՝

<https://github.com/stefansundin/SSHRemote/issues/1>

## Գործառույթի հարցումներ

Ազատորեն ուղարկեք գործառույթի առաջարկներ և սխալների զեկույցներ GitHub-ի պահոցում։ Խնդրում եմ օգտագործեք անգլերեն։ Խնդրում եմ պահպանեք հարգալից խոսելակերպ։ Անհարգալից մեկնաբանությունները կհեռացվեն, և օգտատերերը կարող են արգելափակվել պահոցից։

Խնդրում եմ նախ ստուգեք գոյություն ունեցող issue-ներն ու քննարկումները, որ տեսնեք՝ ձեր հարցը արդեն տրվե՞լ կամ պատասխանվե՞լ է։

Խնդրում եմ հարգալից եղեք։ Ես այս հավելվածը ստեղծել եմ իմ ազատ ժամանակ և անվճար տրամադրում եմ։ Առաջին հերթին այս հավելվածը ստեղծում եմ իմ սեփական օգտագործման համար։

Խնդրում եմ ինձ հարցեր մի ուղարկեք էլփոստով։ Փորձեք քննարկումները պահել GitHub-ում, քանի որ դա օգնում է նաև ուրիշներին։ Կարող եք հարցեր տալ GitHub-ի discussions բաժնում։

Դուք միշտ ազատ եք fork անել հավելվածը՝ ձեր հնարավորությունները ավելացնելու համար։ Սա սովորելու հիանալի ճանապարհ է։ Խնդրում եմ մտածեք օգտակար հնարավորություններով ներդրում ունենալու մասին։

Աղբյուրային կոդը լիցենզավորված է GNU GPLv3-ով։ Եթե տարածում եք այս հավելվածի փոփոխված տարբերակները, պետք է նաև տրամադրեք աղբյուրային կոդը։
