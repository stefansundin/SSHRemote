## SSH Remote பற்றி

இந்த தமிழாக்கம் GitHub Copilot மூலம் செய்யப்பட்டது; மொழிபெயர்ப்பில் பிழைகள் இருக்கக்கூடும்.

SSH Remote என்பது SSH-ஐப் பயன்படுத்தி கணினிகளை தொலைவிலிருந்து கட்டுப்படுத்த அனுமதிக்கும் இலவச மற்றும் திறந்த மூல பயன்பாடு.

இயக்கப்படும் கட்டளைகளை நீங்கள் முழுமையாக தனிப்பயனாக்கலாம்; பொதுவாகப் பயன்படுத்தப்படும் அமைப்புகளுக்கான முன்னமைப்புகளும் உள்ளன.

நான் இந்த பயன்பாட்டை Raspberry Pi OS இயங்கும் என் HTPC அமைப்பை கட்டுப்படுத்த பயன்படுத்துகிறேன். HTPC-ஐ கட்டுப்படுத்துவது தான் இந்த பயன்பாடு சிறப்பாக வேலை செய்யும்படி மேம்படுத்தப்பட்ட முக்கிய சூழல்.

இந்த பயன்பாடு terminal emulator அல்ல, ஆனால் அவசரநிலையிலே `apt-get install` போன்றவற்றை இயக்க உதவும்.

## தொடங்குவது எப்படி

SSH விசையைப் பயன்படுத்தி இணைக்க விரும்பினால், முதலில் பயன்பாட்டு அமைப்புகளைத் திறந்து ஒரு விசையை இறக்குமதி செய்யவோ உருவாக்கவோ செய்யவும்.

கீழ் வலது மூலையில் உள்ள `+` பொத்தானைத் தட்டி புதிய ஹோஸ்டைச் சேர்க்கவும். இணைப்பு விவரங்களை உள்ளிட்டு சேமிக்கவும்.

முதன்முறையாக இணைக்கும் போது, ஒரு முன்னமைப்பைத் தேர்வு செய்யச் சொல்லப்படும். இந்தத் தேர்வு, பலவிதமான கணினிகளுடன் நன்றாக வேலை செய்ய ரிமோட் கட்டுப்பாட்டு பொத்தான்களை அமைக்கும். கிடைக்கும் முன்னமைப்புகளின் விளக்கத்திற்காக கீழே பார்க்கவும். விருப்பமிருந்தால், `No preset` என்பதைத் தேர்வு செய்து எந்த அமைப்பும் இல்லாமல் தொடங்கலாம்.

உங்கள் Linux கணினி X11-ஆ அல்லது Wayland-ஆ இயக்கப்படுகிறது என்பதைத் தெரியாவிட்டால், terminal-ல் இதை இயக்கவும்:

```shell
echo $XDG_SESSION_TYPE
```

இதன் வெளியீடு `x11` அல்லது `wayland` ஆக இருக்க வேண்டும். இதை desktop environment-க்குள் இருந்து இயக்க வேண்டும்.

## ரிமோட் கட்டுப்பாடு

இணைக்கப்பட்ட பிறகு, கட்டளைகளை அனுப்ப ரிமோட் கட்டுப்பாட்டு இடைமுகத்தைப் பயன்படுத்தலாம். பல்வேறு உள்ளீட்டு முறைகளை அணுக தாவல்களை மாற்றவும்.

ஒவ்வொரு பொத்தானை அழுத்தும் போதும், ஹோஸ்டில் ஒரு கட்டளை இயக்கப்படும். ஒரு விசை அழுத்தம் போன்ற எளிய செயலுக்கே இது அதிக செயல்முறைச் சுமையை உருவாக்கும்; எனவே, சாதாரண விசைப்பலகையுடன் ஒப்பிடும்போது அதிக தாமதம் இருக்கலாம். எதிர்கால பதிப்புகளில் இதை மேம்படுத்த முடியும் என்று நம்புகிறேன்.

திருத்து நிலைக்கு செல்ல மெனுவைப் பயன்படுத்தவும். தற்போது layout அல்லது பொத்தான் icon-களைத் திருத்த முடியாது. எதிர்கால பதிப்பில் இதையும் சாத்தியமாக்க விரும்புகிறேன்.

## முன்னமைப்புகள்

உங்கள் desktop environment-க்கு தேவையான கருவியை நிறுவ வேண்டும்.

என் சோதனைகளில் `ydotool` சிறந்த செயல்திறன் கொடுத்தது; மேலும் அது X11 மற்றும் Wayland இரண்டிலும் வேலை செய்கிறது என்பதால் அதைப் பரிந்துரைக்கிறேன்.

### ydotool

`ydotool` எந்த window manager-உடனும் வேலை செய்ய வேண்டும்; ஆனால் பின்னணியில் ஒரு சேவை இயங்க வேண்டும். உங்கள் distribution systemd user service-ஐ வழங்கினால், இதைப் பயன்படுத்தி அதைத் தொடங்கவும்:

```shell
systemctl start --user ydotool
```

உள்நுழையும்போது சேவை தானாகத் தொடங்க இதை இயக்கவும்:

```shell
systemctl enable --user ydotool
```

போதுமான அளவு புதிய `ydotool` பதிப்பை நீங்கள் நிறுவியுள்ளீர்கள் என்பதை உறுதிப்படுத்தவும். Ubuntu 26.04-க்கு முன்புள்ள பதிப்புகள் மிகவும் பழைய பதிப்புகளை வழங்குகின்றன. மாற்று வழிக்கான விவாதத் தொடரைப் பார்க்கவும்.

`ydotool` குறித்து இந்த விவாதத் தொடரில் பேசவும்: <https://github.com/stefansundin/SSHRemote/discussions/5>

### xdotool

`xdotool` என்பது X11-இல் இயங்கும் கணினிகளுக்கானது. வரலாற்றாக பல Linux கணினிகள் X11-ஐப் பயன்படுத்தினாலும், Wayland இப்போது அதிகமாகப் பிரபலமாகிறது.

X11-இன் ஒரு சிக்கல் என்னவெனில், நீங்கள் X server-க்கு அணுகலை அனுமதிக்க வேண்டியிருக்கும். `Authorization required` போன்ற பிழைகள் கிடைத்தால் இதுவே காரணமாக இருக்கலாம். இந்தச் சிக்கலை சரிசெய்ய சில வழிகள் உள்ளன; எனக்கு வேலை செய்த இரண்டு வழிகள் இதோ:

`xauth list` எந்த பதிவையும் காட்டவில்லை என்றால், `.Xauthority` கோப்பை உருவாக்க முயற்சிக்கவும்:

```shell
xauth generate :0 . trusted
```

அது வேலை செய்யவில்லை என்றால், `xhost` மூலம் அணுகலை வழங்க முயற்சிக்கவும்:

```shell
xhost +local:$USER
```

ஒவ்வொரு boot-க்கும் பிறகும் `xhost` கட்டளையை இயக்க வேண்டும். இதை தானியக்கமாக்க bash script ஒன்றை உருவாக்கி, login-இல் தானாகத் தொடங்குமாறு அமைக்கலாம்.

`xdotool` குறித்து இந்த விவாதத் தொடரில் பேசவும்: <https://github.com/stefansundin/SSHRemote/discussions/3>

### wtype

`wtype` என்பது `xdotool` போலவே, ஆனால் Wayland-க்கானது.

பொதுவாக இது mouse control-ஐ ஆதரிக்காது; ஆனால் mouse support சேர்க்கும் மாற்றியமைக்கப்பட்ட பதிப்பை நான் உருவாக்கியுள்ளேன். உங்களுக்கு mouse support தேவைப்பட்டால் அதை நிறுவவும்: <https://github.com/stefansundin/wtype>

`Compositor does not support the virtual keyboard protocol` என்ற பிழை வந்தால், `ydotool` போன்ற வேறு கருவியை முயற்சிக்குமாறு பரிந்துரைக்கிறேன்.

`wtype` குறித்து இந்த விவாதத் தொடரில் பேசவும்: <https://github.com/stefansundin/SSHRemote/discussions/4>

### dotool

`dotool` என்பது `ydotool` போலவே எந்த window manager-உடனும் வேலை செய்ய வேண்டும். என் குறைந்தளவான சோதனைகளில் இது `ydotool`-ஐவிட மிகவும் மந்தமாக இருந்தது.

`dotool` குறித்து இந்த விவாதத் தொடரில் பேசவும்: <https://github.com/stefansundin/SSHRemote/discussions/6>

### cec-client

இந்த முன்னமைப்பு பரிசோதனை நிலையிலுள்ளது; ஏனெனில் இதை என் hardware-ல் நான் சரிபார்க்க முடியவில்லை. கருத்துக்கள் வரவேற்கப்படுகின்றன.

`cec-client` குறித்து இந்த விவாதத் தொடரில் பேசவும்: <https://github.com/stefansundin/SSHRemote/discussions/7>

### macOS

macOS-இல் VLC-ஐ கட்டுப்படுத்த AppleScript கட்டளைகளைப் பயன்படுத்தும் ஒரு முன்னமைப்பு உள்ளது. keyboard அல்லது mouse நிகழ்வுகளை அனுப்ப ஆதரிக்கும் கருவி எனக்கு தெரியவில்லை.

macOS குறித்து இந்த விவாதத் தொடரில் பேசவும்: <https://github.com/stefansundin/SSHRemote/discussions/8>

### Android

உங்கள் Android சாதனத்தில் SSH server இருந்தால், அதனுடன் இணைந்து input event-களை அனுப்ப முடிந்திருக்கலாம். Raspberry Pi-க்கு உள்ள KonstaKANG போன்ற custom ROM பயன்படுத்தினால் இதற்கான வாய்ப்பு அதிகம்.

mouse support-ஐச் செயல்படுத்துவது எப்படி என்று நான் இன்னும் கண்டுபிடிக்கவில்லை.

Android குறித்து இந்த விவாதத் தொடரில் பேசவும்: <https://github.com/stefansundin/SSHRemote/discussions/9>

## ஸ்மார்ட் ஒலி அளவு கட்டுப்பாடு

ரிமோட் கட்டுப்பாட்டைத் திருத்தும்போது, மெனுவில் “smart” volume control settings-ஐ காணலாம். இது கணினியின் தற்போதைய ஒலி அளவை பயன்பாட்டில் காட்டவும், slider மூலம் ஒலி அளவை விரைவாக அமைக்கவும் உதவும். உங்கள் சாதனத்தின் hardware buttons மூலம் volume up/down கட்டளைகளையும் விரைவாக அனுப்பலாம்.

தற்போதைய ஒலி அளவைப் படிப்பதும், slider மூலம் புதிய ஒலி அளவை அமைப்பதும் இப்போது `pactl`-ஐ பயன்படுத்துமாறு hard-code செய்யப்பட்டுள்ளன.

`pactl` உள்ள package பொதுவாக `pulseaudio-utils` அல்லது `libpulse` என்று அழைக்கப்படும்.

## SSH விசைகள்

பயன்பாட்டு அமைப்புகளில் SSH விசைகளை இறக்குமதி செய்யவோ உருவாக்கவோலாம். கடவுச்சொல்லைப் பயன்படுத்துவதைக் காட்டிலும் SSH விசையுடன் இணைப்பது பாதுகாப்பானது.

ஏற்கனவே உள்ள SSH விசையை கணினியிலிருந்து இறக்குமதி செய்வதற்கான எளிய வழி QR குறியீட்டை ஸ்கேன் செய்வது. QR குறியீட்டு படத்தை உருவாக்க `qrencode` நிரலைப் பயன்படுத்தலாம். இதைப் போன்ற ஒரு கட்டளையை இயக்கி QR குறியீட்டை உருவாக்கலாம்:

```shell
# உங்கள் SSH விசைகள் உள்ள இடத்துக்கு செல்லவும்:
cd ~/.ssh

# QR குறியீட்டை terminal-ல் காட்ட:
qrencode -r id_ed25519 -t ansiutf8

# மாற்றாக, ஒரு image file உருவாக்க:
qrencode -r id_ed25519 -o qr.png

# 4096-bit RSA விசைகள் QR குறியீட்டில் அடங்காது. மிகக் கடினமாக பொருத்த gzip-ஐ பயன்படுத்தலாம்:
cat id_rsa_4096 | gzip | base64 | qrencode -o qr.png
```

மெனுவில் உள்ள `பொது விசையை அனுப்பு` அம்சத்தைப் பயன்படுத்தி public SSH விசைகளை server-க்கு அனுப்பலாம். இது தேர்ந்தெடுக்கப்பட்ட SSH விசையை `~/.ssh/authorized_keys` கோப்பின் முடிவில் சேர்க்கும். இதன் மூலம் கடவுச்சொல் உள்நுழைவிலிருந்து SSH விசை உள்நுழைவிற்கு எளிதாக மாறலாம்.

குறியாக்கப்பட்ட SSH விசைகளையும் இறக்குமதி செய்து பயன்படுத்தலாம்; ஆனால் இப்போது பயன்பாட்டுக்குள் அவற்றை உருவாக்க முடியாது.

## பாதுகாப்பு

பயன்பாட்டிலிருந்து SSH விசைகளின் private பகுதியையோ, சேமிக்கப்பட்ட கடவுச்சொற்களையோ ஏற்றுமதி செய்யவோ எடுத்துக்கொள்ளவோ முடியாது. இந்தத் தரவு 256-bit AES மூலம் குறியாக்கம் செய்யப்படுகிறது; குறியாக்க விசை Android Keystore-இல் சேமிக்கப்படுகிறது. குறியாக்கப்பட்ட தரவு Android backups-இல் சேர்க்கப்படாது.

இந்த பயன்பாட்டில் crash reporting software இல்லை. telemetry இல்லை. விளம்பரங்கள் இல்லை. SSH இணைப்பைத் தவிர வேறு network request-களும் இல்லை.

இந்த பயன்பாட்டின் பாதுகாப்பு இன்னும் audit செய்யப்படவில்லை. Android security அல்லது SSH security குறித்து அனுபவமுள்ளவராக இருந்தால், source code-ஐப் பார்த்து உங்கள் கண்டுபிடிப்புகளை இந்த GitHub issue-ல் தெரிவிக்கவும்:

<https://github.com/stefansundin/SSHRemote/issues/1>
