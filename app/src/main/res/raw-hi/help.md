## SSH Remote के बारे में

यह अनुवाद GitHub Copilot की सहायता से किया गया है और इसमें अनुवाद संबंधी त्रुटियाँ हो सकती हैं।

SSH Remote एक मुफ़्त और ओपन सोर्स ऐप है जो आपको SSH का उपयोग करके कंप्यूटरों को दूर से नियंत्रित करने देता है।

आप चलाए जाने वाले कमांड को पूरी तरह अनुकूलित कर सकते हैं, और सामान्य सेटअप के लिए प्रीसेट उपलब्ध हैं।

मैं इस ऐप का उपयोग अपने HTPC सेटअप को नियंत्रित करने के लिए करता हूँ, जो Raspberry Pi OS चला रहा है। HTPC को नियंत्रित करना वह मुख्य परिदृश्य है जिसके लिए यह ऐप सबसे अधिक अनुकूलित है।

यह ऐप टर्मिनल एमुलेटर नहीं है, लेकिन आपात स्थिति में यह आपको `apt-get install` चलाने देगा।

## शुरुआत

यदि आप कनेक्ट करने के लिए SSH कुंजी का उपयोग करना चाहते हैं, तो पहले ऐप सेटिंग्स खोलें और एक कुंजी आयात करें या जनरेट करें।

नीचे दाईं ओर `+` बटन दबाकर एक नया होस्ट जोड़ें। कनेक्शन विवरण दर्ज करें और सहेजें।

पहली बार कनेक्ट करते समय आपसे एक प्रीसेट चुनने के लिए कहा जाएगा। यह चयन रिमोट कंट्रोल बटन को विभिन्न प्रकार के कंप्यूटरों पर अच्छे से काम करने के लिए कॉन्फ़िगर करता है। उपलब्ध प्रीसेट के विवरण के लिए नीचे देखें। यदि आप चाहें, तो `कोई प्रीसेट नहीं` चुनकर बिना किसी कॉन्फ़िगरेशन के भी शुरुआत कर सकते हैं।

यदि आपको नहीं पता कि आपका Linux कंप्यूटर X11 चला रहा है या Wayland, तो टर्मिनल में यह चलाएँ:

```shell
echo $XDG_SESSION_TYPE
```

इसका आउटपुट `x11` या `wayland` होना चाहिए। आपको यह डेस्कटॉप वातावरण के अंदर चलाना होगा।

## रिमोट कंट्रोल

कनेक्ट होने के बाद आप कमांड भेजने के लिए रिमोट कंट्रोल इंटरफ़ेस का उपयोग कर सकते हैं। विभिन्न इनपुट विधियों तक पहुँचने के लिए टैब बदलें।

हर बटन दबाने पर होस्ट पर एक कमांड चलेगा। साधारण key press जैसी चीज़ के लिए यह काफ़ी ओवरहेड है, और आपको सामान्य कीबोर्ड की तुलना में काफ़ी अधिक latency महसूस हो सकती है। मैं भविष्य के संस्करणों में इसे बेहतर बनाने की उम्मीद करता हूँ।

संपादन मोड में जाने के लिए मेनू का उपयोग करें। अभी लेआउट या बटन आइकन संपादित करना संभव नहीं है। मैं भविष्य के किसी संस्करण में इसे संभव बनाने की उम्मीद करता हूँ।

## प्रीसेट

आपको अपने डेस्कटॉप वातावरण के लिए आवश्यक टूल इंस्टॉल करना होगा।

मैं `ydotool` की सिफारिश करता हूँ क्योंकि मेरी जाँच में इसका प्रदर्शन सबसे अच्छा रहा, और यह X11 तथा Wayland दोनों पर काम करता है।

### ydotool

`ydotool` किसी भी window manager के साथ काम करना चाहिए, लेकिन इसके लिए एक background service चल रही होनी चाहिए। यदि आपका distribution systemd user service प्रदान करता है, तो इसे चलाकर शुरू करें:

```shell
systemctl start --user ydotool
```

लॉगिन पर इस सेवा को अपने आप शुरू करने के लिए यह चलाएँ:

```shell
systemctl enable --user ydotool
```

कृपया सुनिश्चित करें कि आप `ydotool` का पर्याप्त नया संस्करण इंस्टॉल कर रहे हैं। Ubuntu 26.04 से पहले के संस्करण बहुत पुराने पैकेज देते हैं। समाधान के लिए चर्चा थ्रेड देखें।

कृपया `ydotool` पर इस चर्चा थ्रेड में चर्चा करें: <https://github.com/stefansundin/SSHRemote/discussions/5>

### xdotool

`xdotool` X11 चलाने वाले कंप्यूटरों के लिए है। ऐतिहासिक रूप से अधिकांश Linux कंप्यूटर X11 का उपयोग करते रहे हैं, हालाँकि Wayland अब अधिक लोकप्रिय हो रहा है।

X11 की एक खास बात यह है कि आपको X server तक पहुँच की अनुमति देनी पड़ सकती है। यदि आपको `Authorization required` त्रुटियाँ मिल रही हैं, तो यही समस्या है। इस समस्या को ठीक करने के लिए आपके पास कुछ विकल्प हैं; यहाँ दो विकल्प दिए गए हैं जो मेरे लिए काम कर चुके हैं:

यदि `xauth list` कोई प्रविष्टि नहीं दिखाता, तो `.Xauthority` फ़ाइल बनाने का प्रयास करें:

```shell
xauth generate :0 . trusted
```

यदि उससे काम नहीं बना, तो `xhost` का उपयोग करके पहुँच प्रदान करने का प्रयास करें:

```shell
xhost +local:$USER
```

आपको हर बूट के बाद `xhost` कमांड चलाना होगा। आप इसके लिए एक bash script बनाकर और उसे login पर autostart के लिए कॉन्फ़िगर करके इसे स्वचालित कर सकते हैं।

कृपया `xdotool` पर इस चर्चा थ्रेड में चर्चा करें: <https://github.com/stefansundin/SSHRemote/discussions/3>

### wtype

`wtype`, `xdotool` जैसा है, लेकिन Wayland के लिए।

सामान्यतः यह mouse control का समर्थन नहीं करता, लेकिन मैंने एक संशोधित संस्करण बनाया है जो mouse support जोड़ता है। यदि आपको mouse support चाहिए, तो इसे इंस्टॉल करें: <https://github.com/stefansundin/wtype>

यदि आपको `Compositor does not support the virtual keyboard protocol` त्रुटि मिलती है, तो मैं सुझाव दूँगा कि आप `ydotool` जैसे किसी अन्य टूल को आज़माएँ।

कृपया `wtype` पर इस चर्चा थ्रेड में चर्चा करें: <https://github.com/stefansundin/SSHRemote/discussions/4>

### dotool

`dotool` को `ydotool` की तरह किसी भी window manager के साथ काम करना चाहिए। मेरी सीमित जाँच में यह `ydotool` की तुलना में काफ़ी धीमा था।

कृपया `dotool` पर इस चर्चा थ्रेड में चर्चा करें: <https://github.com/stefansundin/SSHRemote/discussions/6>

### cec-client

यह प्रीसेट प्रायोगिक है क्योंकि मैं इसे अपने हार्डवेयर पर सत्यापित नहीं कर पाया हूँ। प्रतिक्रिया स्वागत योग्य है।

कृपया `cec-client` पर इस चर्चा थ्रेड में चर्चा करें: <https://github.com/stefansundin/SSHRemote/discussions/7>

### macOS

macOS पर VLC को नियंत्रित करने के लिए एक प्रीसेट है, जो AppleScript कमांड का उपयोग करता है। मुझे किसी ऐसे टूल की जानकारी नहीं है जो कीबोर्ड या माउस इवेंट भेजने का समर्थन करता हो।

कृपया macOS पर इस चर्चा थ्रेड में चर्चा करें: <https://github.com/stefansundin/SSHRemote/discussions/8>

### Android

यदि आपके Android डिवाइस में SSH server है, तो आप उससे कनेक्ट करके input events भेजने में सक्षम हो सकते हैं। यह अधिक संभव है यदि आपका डिवाइस किसी custom ROM पर चल रहा हो, जैसे Raspberry Pi पर KonstaKANG।

मैं अभी तक mouse support को काम कराने का तरीका नहीं ढूँढ पाया हूँ।

कृपया Android पर इस चर्चा थ्रेड में चर्चा करें: <https://github.com/stefansundin/SSHRemote/discussions/9>

## स्मार्ट वॉल्यूम नियंत्रण

रिमोट कंट्रोल संपादित करते समय आपको मेनू में “स्मार्ट” वॉल्यूम नियंत्रण सेटिंग्स मिलेंगी। यह ऐप में कंप्यूटर की वर्तमान वॉल्यूम दिखा सकता है और आपको स्लाइडर का उपयोग करके जल्दी से वॉल्यूम सेट करने देता है। आप अपने डिवाइस के हार्डवेयर बटन का उपयोग करके भी जल्दी से वॉल्यूम बढ़ाने/घटाने के कमांड भेज सकते हैं।

वर्तमान वॉल्यूम पढ़ना और स्लाइडर का उपयोग करके नई वॉल्यूम सेट करना अभी hard-coded है और `pactl` का उपयोग करता है।

`pactl` वाला पैकेज आमतौर पर `pulseaudio-utils` या `libpulse` कहलाता है।

## SSH कुंजियाँ

आप ऐप सेटिंग्स में SSH कुंजियाँ आयात या जनरेट कर सकते हैं। पासवर्ड की बजाय SSH कुंजी से कनेक्ट करना अधिक सुरक्षित है।

किसी कंप्यूटर से मौजूदा SSH कुंजी आयात करने का सबसे आसान तरीका QR कोड स्कैन करना है। आप QR कोड छवि बनाने के लिए `qrencode` प्रोग्राम का उपयोग कर सकते हैं। QR कोड बनाने के लिए नीचे जैसा कमांड चलाएँ:

```shell
# अपनी SSH कुंजियों पर जाएँ:
cd ~/.ssh

# QR कोड को टर्मिनल में प्रदर्शित करें:
qrencode -r id_ed25519 -t ansiutf8

# वैकल्पिक रूप से, एक इमेज फ़ाइल बनाएँ:
qrencode -r id_ed25519 -o qr.png

# 4096-bit RSA keys QR कोड के लिए बहुत बड़ी होती हैं। आप gzip का इस्तेमाल करके, किसी तरह एक key को इसमें समा सकते हैं:
cat id_rsa_4096 | gzip | base64 | qrencode -o qr.png
```

आप मेनू में `सार्वजनिक कुंजी भेजें` सुविधा का उपयोग करके सार्वजनिक SSH कुंजियाँ सर्वर पर भेज सकते हैं। यह चुनी गई SSH कुंजी को `~/.ssh/authorized_keys` फ़ाइल में जोड़ देगा। इससे पासवर्ड से लॉगिन करने से SSH कुंजी से लॉगिन करने की ओर जाना आसान हो जाता है।

आप एन्क्रिप्टेड SSH कुंजियाँ आयात और उपयोग कर सकते हैं, लेकिन अभी ऐप में इन्हें जनरेट नहीं किया जा सकता।

## सुरक्षा

ऐप से SSH कुंजियों का निजी भाग या सहेजे गए पासवर्ड निर्यात या निकाला नहीं जा सकता। यह डेटा 256-bit AES का उपयोग करके एन्क्रिप्ट किया जाता है, और एन्क्रिप्शन कुंजी Android Keystore में संग्रहीत होती है। एन्क्रिप्टेड डेटा Android backups से बाहर रखा जाता है।

इस ऐप में कोई crash reporting software नहीं है। कोई telemetry नहीं है। कोई ads नहीं हैं। SSH कनेक्शन के अलावा कोई network request नहीं होती।

इस ऐप की सुरक्षा का ऑडिट नहीं किया गया है। यदि आपको Android सुरक्षा या SSH सुरक्षा का अनुभव है, तो कृपया source code देखें और अपने निष्कर्ष इस GitHub issue में रिपोर्ट करें:

<https://github.com/stefansundin/SSHRemote/issues/1>
