## SSH Remote बद्दल

हा अनुवाद AI च्या मदतीने, विशेषतः GitHub Copilot (GPT-5.3-Codex) वापरून केला आहे. यामध्ये अनुवादातील काही चुका असू शकतात.

SSH Remote हे एक मोफत आणि मुक्त स्त्रोत अ‍ॅप आहे जे तुम्हाला SSH वापरून संगणक दूरस्थपणे नियंत्रित करण्याची परवानगी देते.

तुम्ही चालवल्या जाणाऱ्या कमांड्स पूर्णपणे सानुकूल करू शकता, आणि सामान्य सेटअपसाठी प्रीसेट देखील उपलब्ध आहेत.

मी हे अ‍ॅप माझ्या HTPC सेटअपला नियंत्रित करण्यासाठी वापरतो, जो Raspberry Pi OS वर चालतो. HTPC नियंत्रित करणे ही या अ‍ॅपसाठी अनुकूलित केलेली मूलभूत परिस्थिती आहे.

हे अ‍ॅप टर्मिनल एम्युलेटर नाही, पण गरज पडल्यास ते तुम्हाला `apt-get install` चालवू देईल.

## सुरुवात

जर तुम्हाला कनेक्ट होण्यासाठी SSH की वापरायची असेल, तर प्रथम अ‍ॅप सेटिंग्ज उघडा आणि की आयात करा किंवा तयार करा.

खाली उजव्या कोपऱ्यातील `+` बटण टॅप करून नवीन होस्ट जोडा. कनेक्शन तपशील भरा आणि जतन करा.

तुम्ही प्रथमच कनेक्ट केल्यावर, तुम्हाला एक प्रीसेट निवडण्यास सांगितले जाईल. ही निवड रिमोट कंट्रोलची बटणे विविध प्रकारच्या संगणकांवर व्यवस्थित काम करतील अशा प्रकारे संरचीत करते. उपलब्ध प्रीसेटचे वर्णन खाली पहा. तुम्हाला हवे असल्यास, `No preset` निवडून तुम्ही कोणतीही संरचना न करता सुरुवात करू शकता.

तुमचा Linux संगणक X11 चालवत आहे की Wayland हे माहित नसेल, तर टर्मिनलमध्ये हे चालवा:

```shell
echo $XDG_SESSION_TYPE
```

याचे आउटपुट `x11` किंवा `wayland` असे असावे. ही कमांड तुम्हाला डेस्कटॉप वातावरणाच्या आत चालवावी लागेल.

## रिमोट कंट्रोल

कनेक्ट झाल्यावर, तुम्ही कमांड्स पाठवण्यासाठी रिमोट कंट्रोल इंटरफेस वापरू शकता. विविध इनपुट पद्धतींमध्ये प्रवेश करण्यासाठी टॅब बदला.

प्रत्येक बटण दाबल्यावर होस्टवर एक कमांड चालवली जाईल. फक्त की दाबण्याइतक्या साध्या गोष्टीसाठी हा खूप मोठा अतिरिक्त भार आहे, आणि तुम्हाला साध्या कीबोर्डच्या तुलनेत जास्त विलंब जाणवू शकतो. भविष्यातील आवृत्त्यांमध्ये हे सुधारण्याची मला आशा आहे.

संपादन मोडमध्ये जाण्यासाठी मेनू वापरा. सध्या लेआउट किंवा बटणांच्या आयकॉन्सचे संपादन करणे शक्य नाही. भविष्यात हे शक्य करण्याची मला आशा आहे.

## प्रीसेट

तुमच्या डेस्कटॉप वातावरणासाठी आवश्यक साधन तुम्हाला इन्स्टॉल करावे लागेल.

माझ्या चाचण्यांनुसार `ydotool` ची कार्यक्षमता सर्वोत्तम आहे, आणि ते X11 तसेच Wayland दोन्हीवर काम करते, म्हणून मी त्याची शिफारस करतो.

### ydotool

`ydotool` कोणत्याही विंडो मॅनेजरसह कार्य करायला हवे, पण त्यासाठी पार्श्वभूमी सेवा चालू असणे आवश्यक आहे. तुमचा डिस्ट्रिब्युशन systemd वापरकर्ता सेवा पुरवत असेल, तर ती सुरू करण्यासाठी हे चालवा:

```shell
systemctl start --user ydotool
```

लॉगिनवेळी ही सेवा आपोआप सुरू करण्यासाठी हे चालवा:

```shell
systemctl enable --user ydotool
```

कृपया खात्री करा की तुम्ही `ydotool` ची पुरेशी नवीन आवृत्ती इन्स्टॉल करत आहात. Ubuntu 26.04 पूर्वीच्या आवृत्त्यांमध्ये मिळणाऱ्या आवृत्त्या खूप जुन्या आहेत. उपायासाठी चर्चा धागा पहा.

कृपया `ydotool` बद्दल या चर्चा धाग्यात बोला: <https://github.com/stefansundin/SSHRemote/discussions/5>

### xdotool

`xdotool` हा X11 चालवणाऱ्या संगणकांसाठी आहे. इतिहासात बहुतेक Linux संगणक X11 वापरत आले आहेत, तरी Wayland अधिक लोकप्रिय होत आहे.

X11 मधील एक वैशिष्ट्य म्हणजे तुम्हाला X सर्व्हरला प्रवेश द्यावा लागू शकतो. तुम्हाला "Authorization required" अशा त्रुटी मिळत असतील तर हीच समस्या असू शकते. ही समस्या सोडवण्यासाठी काही पर्याय आहेत, त्यापैकी माझ्यासाठी काम केलेले दोन पर्याय येथे दिले आहेत:

जर `xauth list` कोणत्याही नोंदी दाखवत नसेल, तर `.Xauthority` फाइल तयार करून पाहा:

```shell
xauth generate :0 . trusted
```

ते काम करत नसेल, तर `xhost` वापरून प्रवेश देण्याचा प्रयत्न करा:

```shell
xhost +local:$USER
```

प्रत्येक बूटनंतर तुम्हाला `xhost` कमांड चालवावी लागेल. तुम्ही bash स्क्रिप्ट तयार करून आणि लॉगिनवेळी ती आपोआप सुरू होईल असे संरचीत करून हे स्वयंचलित करू शकता.

कृपया `xdotool` बद्दल या चर्चा धाग्यात बोला: <https://github.com/stefansundin/SSHRemote/discussions/3>

### wtype

`wtype` हे `xdotool` सारखेच आहे, पण Wayland साठी.

सामान्यतः ते माऊस नियंत्रणाला समर्थन देत नाही, पण मी माऊस समर्थन जोडणारी एक सुधारित आवृत्ती तयार केली आहे. तुम्हाला माऊस समर्थन हवे असल्यास, कृपया ती इन्स्टॉल करा: <https://github.com/stefansundin/wtype>

जर तुम्हाला `Compositor does not support the virtual keyboard protocol` ही त्रुटी मिळाली, तर मी `ydotool` सारखे दुसरे साधन वापरून पाहण्याचा सल्ला देईन.

कृपया `wtype` बद्दल या चर्चा धाग्यात बोला: <https://github.com/stefansundin/SSHRemote/discussions/4>

### dotool

`dotool` हे `ydotool` प्रमाणे कोणत्याही विंडो मॅनेजरसह कार्य करायला हवे. माझ्या मर्यादित चाचणीत ते `ydotool` पेक्षा खूप धीमे होते.

कृपया `dotool` बद्दल या चर्चा धाग्यात बोला: <https://github.com/stefansundin/SSHRemote/discussions/6>

### cec-client

हा प्रीसेट प्रायोगिक आहे कारण मी तो माझ्या स्वतःच्या हार्डवेअरवर तपासू शकलो नाही. अभिप्रायाचे स्वागत आहे.

कृपया `cec-client` बद्दल या चर्चा धाग्यात बोला: <https://github.com/stefansundin/SSHRemote/discussions/7>

### macOS

macOS वर VLC नियंत्रित करण्यासाठी AppleScript कमांड्स वापरणारा एक प्रीसेट आहे. कीबोर्ड किंवा माऊस इव्हेंट पाठवण्यास समर्थन देणारे साधन मला माहित नाही.

कृपया macOS बद्दल या चर्चा धाग्यात बोला: <https://github.com/stefansundin/SSHRemote/discussions/8>

### Android

जर तुमच्या Android डिव्हाइससोबत SSH सर्व्हर मिळत असेल, तर तुम्ही त्याच्याशी कनेक्ट होऊन इनपुट इव्हेंट पाठवू शकता. तुमचे डिव्हाइस Raspberry Pi वर KonstaKANG सारखा कस्टम ROM चालवत असेल तर ही शक्यता अधिक असते.

माऊस समर्थन कसे कार्यान्वित करायचे हे मी अजून शोधू शकलो नाही.

कृपया Android बद्दल या चर्चा धाग्यात बोला: <https://github.com/stefansundin/SSHRemote/discussions/9>

## स्मार्ट आवाज नियंत्रण

रिमोट कंट्रोल संपादित करताना, तुम्हाला मेनूमध्ये "smart" आवाज नियंत्रण सेटिंग्ज सापडतील. यामुळे अ‍ॅपमध्ये संगणकाचा सध्याचा आवाज दाखवता येतो आणि स्लायडर वापरून पटकन आवाज सेट करता येतो. तुम्ही तुमच्या डिव्हाइसवरील हार्डवेअर बटणे वापरून आवाज वाढवा किंवा कमी करा अशा कमांड्स पटकन पाठवू शकता.

सध्याचा आवाज वाचणे आणि स्लायडर वापरून नवीन आवाज सेट करणे सध्या `pactl` वापरण्यासाठी हार्ड-कोड केलेले आहे.

`pactl` असलेले पॅकेज साधारणपणे `pulseaudio-utils` किंवा `libpulse` असे म्हणतात.

## SSH की

तुम्ही अ‍ॅप सेटिंग्जमध्ये SSH की आयात किंवा तयार करू शकता. SSH की वापरून कनेक्ट करणे हे पासवर्ड वापरण्यापेक्षा अधिक सुरक्षित आहे.

संगणकावरून अस्तित्वात असलेली SSH की आयात करण्याचा सर्वात सोपा मार्ग म्हणजे QR कोड स्कॅन करणे. QR कोड प्रतिमा तयार करण्यासाठी तुम्ही `qrencode` प्रोग्राम वापरू शकता. QR कोड तयार करण्यासाठी खालीलप्रमाणे कमांड चालवा:

```shell
# तुमच्या SSH कींच्या फोल्डरमध्ये जा:
cd ~/.ssh

# टर्मिनलमध्ये QR कोड दाखवा:
qrencode -r id_ed25519 -t ansiutf8

# किंवा, प्रतिमा फाइल तयार करा:
qrencode -r id_ed25519 -o qr.png

# 4096-बिट RSA की QR कोडसाठी खूप मोठ्या असतात. gzip वापरल्यास एक कसाबसा मावू शकतो:
cat id_rsa_4096 | gzip | base64 | qrencode -o qr.png
```

मेनूमधील `सार्वजनिक की पाठवा` वैशिष्ट्य वापरून तुम्ही सार्वजनिक SSH की सर्व्हरवर पाठवू शकता. यामुळे निवडलेली SSH की `~/.ssh/authorized_keys` फाइलमध्ये जोडली जाईल. त्यामुळे पासवर्डने लॉगिन करण्यापासून SSH कीने लॉगिन करण्याकडे सहज बदल करता येतो.

तुम्ही एन्क्रिप्ट केलेल्या SSH की आयात करून वापरू शकता, पण सध्या अ‍ॅपमध्ये त्या तयार करता येत नाहीत.

## सुरक्षा

अ‍ॅपमधून SSH कींचा खाजगी भाग किंवा साठवलेले पासवर्ड निर्यात करणे किंवा बाहेर काढणे शक्य नाही. हा डेटा 256-बिट AES वापरून एन्क्रिप्ट केला जातो, आणि एन्क्रिप्शन की Android Keystore मध्ये साठवली जाते. एन्क्रिप्ट केलेला डेटा Android बॅकअपमधून वगळला जातो.

या अ‍ॅपमध्ये कोणतेही क्रॅश रिपोर्टिंग सॉफ्टवेअर नाही. कोणतीही टेलिमेट्री नाही. जाहिराती नाहीत. SSH कनेक्शन शिवाय इतर कोणत्याही नेटवर्क विनंत्या नाहीत.

या अ‍ॅपच्या सुरक्षेचे ऑडिट झालेले नाही. तुम्हाला Android सुरक्षा किंवा SSH सुरक्षेचा अनुभव असल्यास, कृपया स्रोत कोड पहा आणि तुमचे निष्कर्ष या GitHub issue मध्ये नोंदवा:

<https://github.com/stefansundin/SSHRemote/issues/1>
