## ກ່ຽວກັບ SSH Remote

ຄໍາແປນີ້ຖືກສ້າງໂດຍ AI ໂດຍໃຊ້ GitHub Copilot (GPT-5.3-Codex) ແລະອາດມີຂໍ້ຜິດພາດໃນການແປຢູ່ບ້າງ.

SSH Remote ແມ່ນແອັບຟຣີແລະເປັນໂອເພນຊອດ ທີ່ຊ່ວຍໃຫ້ທ່ານຄວບຄຸມຄອມພິວເຕີຈາກໄລຍະໄກຜ່ານ SSH.

ທ່ານສາມາດປັບແຕ່ງຄໍາສັ່ງທີ່ຖືກດໍາເນີນໄດ້ຢ່າງເຕັມທີ່ ແລະຍັງມີຄ່າຕັ້ງສໍາເລັດຮູບສໍາລັບການຕັ້ງຄ່າທົ່ວໄປ.

ຂ້ອຍໃຊ້ແອັບນີ້ເພື່ອຄວບຄຸມຊຸດ HTPC ຂອງຂ້ອຍ ທີ່ໃຊ້ Raspberry Pi OS. ການຄວບຄຸມ HTPC ແມ່ນສະຖານະການຫຼັກທີ່ແອັບນີ້ຖືກປັບໃຫ້ເໝາະສົມ.

ແອັບນີ້ບໍ່ແມ່ນ terminal emulator, ແຕ່ມັນກໍຊ່ວຍໃຫ້ທ່ານສາມາດຮັນ `apt-get install` ໄດ້ໃນຍາມສຸກເສີນ.

## ເລີ່ມຕົ້ນ

ຖ້າທ່ານຕ້ອງການໃຊ້ກະແຈ SSH ເພື່ອເຊື່ອມຕໍ່, ໃຫ້ເປີດການຕັ້ງຄ່າຂອງແອັບກ່ອນ ແລະນໍາເຂົ້າ ຫຼື ສ້າງກະແຈໃໝ່.

ເພີ່ມໂຮສໃໝ່ໂດຍແຕະປຸ່ມ `+` ທີ່ມຸມລຸ່ມຂວາ. ປ້ອນລາຍລະອຽດການເຊື່ອມຕໍ່ແລະບັນທຶກ.

ໃນຄັ້ງທໍາອິດທີ່ທ່ານເຊື່ອມຕໍ່, ລະບົບຈະໃຫ້ທ່ານເລືອກຄ່າຕັ້ງສໍາເລັດຮູບ. ການເລືອກນີ້ຈະກໍານົດປຸ່ມຄວບຄຸມຣີໂມດໃຫ້ເຮັດວຽກໄດ້ດີກັບຄອມພິວເຕີຫຼາຍປະເພດ. ເບິ່ງຄໍາອະທິບາຍຄ່າຕັ້ງສໍາເລັດຮູບທີ່ມີຢູ່ຂ້າງລຸ່ມ. ຖ້າທ່ານຕ້ອງການ ທ່ານສາມາດເລີ່ມດ້ວຍບໍ່ມີການຕັ້ງຄ່າໄດ້ໂດຍເລືອກ `No preset`.

ຖ້າທ່ານບໍ່ຮູ້ວ່າຄອມພິວເຕີ Linux ຂອງທ່ານໃຊ້ X11 ຫຼື Wayland, ໃຫ້ຮັນຄໍາສັ່ງນີ້ໃນ terminal:

```shell
echo $XDG_SESSION_TYPE
```

ຄໍາສັ່ງນີ້ຄວນສະແດງ `x11` ຫຼື `wayland`. ທ່ານຕ້ອງຮັນມັນຈາກພາຍໃນ desktop environment.

## ການຄວບຄຸມຣີໂມດ

ເມື່ອເຊື່ອມຕໍ່ແລ້ວ, ທ່ານສາມາດໃຊ້ໜ້າຈໍຄວບຄຸມຣີໂມດເພື່ອສົ່ງຄໍາສັ່ງໄປຫາໂຮສ. ສະຫຼັບແຖບເພື່ອເຂົ້າເຖິງວິທີປ້ອນຂໍ້ມູນຫຼາຍແບບ.

ການກົດປຸ່ມແຕ່ລະຄັ້ງຈະເຮັດການຮັນຄໍາສັ່ງໃນໂຮສ. ນີ້ມີ overhead ສູງສໍາລັບສິ່ງງ່າຍໆເຊັ່ນການກົດປຸ່ມແປ້ນພິມ ແລະທ່ານອາດພົບຄວາມຫນ່ວງສູງພໍສົມຄວນເມື່ອທຽບກັບແປ້ນພິມທົ່ວໄປ. ຂ້ອຍຫວັງວ່າຈະປັບປຸງສ່ວນນີ້ໃນອະນາຄົດ.

ໃຊ້ເມນູເພື່ອເຂົ້າສູ່ໂໝດແກ້ໄຂ. ໃນຕອນນີ້ຍັງບໍ່ສາມາດແກ້ໄຂ layout ຫຼື icon ຂອງປຸ່ມໄດ້. ຂ້ອຍຫວັງວ່າຈະເຮັດໃຫ້ເປັນໄປໄດ້ໃນອະນາຄົດ.

## ຄ່າຕັ້ງສໍາເລັດຮູບ

ທ່ານຈະຕ້ອງຕິດຕັ້ງເຄື່ອງມືທີ່ຈໍາເປັນສໍາລັບ desktop environment ຂອງທ່ານ.

ຂ້ອຍແນະນໍາ `ydotool` ເພາະໃນການທົດສອບຂອງຂ້ອຍ ມັນໃຫ້ປະສິດທິພາບດີທີ່ສຸດ ແລະມັນໃຊ້ໄດ້ທັງ X11 ແລະ Wayland.

### ydotool

`ydotool` ຄວນໃຊ້ໄດ້ກັບ window manager ທຸກແບບ, ແຕ່ທ່ານຕ້ອງມີ background service ທີ່ກໍາລັງເຮັດວຽກ. ຖ້າ distribution ຂອງທ່ານມີ systemd user service ໃຫ້ເລີ່ມມັນໂດຍຮັນ:

```shell
systemctl start --user ydotool
```

ໃຫ້ service ເລີ່ມອັດຕະໂນມັດເມື່ອ login ໂດຍຮັນ:

```shell
systemctl enable --user ydotool
```

ກະລຸນາໃຫ້ແນ່ໃຈວ່າທ່ານກໍາລັງຕິດຕັ້ງ `ydotool` ເວີຊັນໃໝ່ພໍ. Ubuntu ກ່ອນເວີຊັນ 26.04 ໃຫ້ເວີຊັນທີ່ເກົ່າເກີນໄປ. ເບິ່ງ discussion thread ສໍາລັບວິທີແກ້ໄຂຊົ່ວຄາວ.

ກະລຸນາຮ່ວມສົນທະນາກ່ຽວກັບ `ydotool` ໃນ discussion thread ນີ້: <https://github.com/stefansundin/SSHRemote/discussions/5>

### xdotool

`xdotool` ສໍາລັບຄອມພິວເຕີທີ່ໃຊ້ X11. X11 ແມ່ນສິ່ງທີ່ຄອມພິວເຕີ Linux ສ່ວນໃຫຍ່ເຄີຍໃຊ້ມາຕະຫຼອດ, ແມ່ນວ່າ Wayland ກໍາລັງເປັນທີ່ນິຍົມຫຼາຍຂຶ້ນ.

ຈຸດໜຶ່ງທີ່ຄວນຮູ້ຂອງ X11 ແມ່ນ ທ່ານອາດຈະຕ້ອງອະນຸຍາດການເຂົ້າເຖິງ X server. ນີ້ແມ່ນສາເຫດຖ້າທ່ານໄດ້ຮັບຂໍ້ຜິດພາດ "Authorization required". ທ່ານມີຫຼາຍທາງເລືອກໃນການແກ້ໄຂບັນຫານີ້, ນີ້ແມ່ນສອງວິທີທີ່ເຄີຍໃຊ້ໄດ້ຜົນສໍາລັບຂ້ອຍ:

ຖ້າ `xauth list` ບໍ່ສະແດງລາຍການໃດໆ ໃຫ້ລອງສ້າງໄຟລ໌ `.Xauthority`:

```shell
xauth generate :0 . trusted
```

ຖ້າວິທີນັ້ນບໍ່ໄດ້ຜົນ ໃຫ້ລອງອະນຸຍາດການເຂົ້າເຖິງດ້ວຍ `xhost`:

```shell
xhost +local:$USER
```

ທ່ານຈະຕ້ອງຮັນຄໍາສັ່ງ `xhost` ຫຼັງຈາກບູດທຸກຄັ້ງ. ທ່ານສາມາດອັດຕະໂນມັດສິ່ງນີ້ໄດ້ໂດຍການສ້າງ bash script ແລະຕັ້ງຄ່າໃຫ້ມັນຮັນອັດຕະໂນມັດເມື່ອ login.

ກະລຸນາຮ່ວມສົນທະນາກ່ຽວກັບ `xdotool` ໃນ discussion thread ນີ້: <https://github.com/stefansundin/SSHRemote/discussions/3>

### wtype

`wtype` ຄ້າຍກັບ `xdotool`, ແຕ່ສໍາລັບ Wayland.

ໂດຍປົກກະຕິແລ້ວ ມັນບໍ່ຮອງຮັບການຄວບຄຸມເມົາສ໌, ແຕ່ຂ້ອຍໄດ້ສ້າງສະບັບທີ່ຖືກດັດແປງເພື່ອເພີ່ມການຮອງຮັບເມົາສ໌. ກະລຸນາຕິດຕັ້ງມັນຖ້າທ່ານຕ້ອງການໃຊ້ເມົາສ໌: <https://github.com/stefansundin/wtype>

ຖ້າທ່ານໄດ້ຮັບຂໍ້ຜິດພາດ `Compositor does not support the virtual keyboard protocol` ຂ້ອຍແນະນໍາໃຫ້ທ່ານລອງເຄື່ອງມືອື່ນ ເຊັ່ນ `ydotool`.

ກະລຸນາຮ່ວມສົນທະນາກ່ຽວກັບ `wtype` ໃນ discussion thread ນີ້: <https://github.com/stefansundin/SSHRemote/discussions/4>

### dotool

`dotool` ຄວນໃຊ້ໄດ້ກັບ window manager ທຸກແບບ ຄ້າຍກັບ `ydotool`. ໃນການທົດສອບຈໍາກັດຂອງຂ້ອຍ ມັນຊ້າກວ່າ `ydotool` ຫຼາຍ.

ກະລຸນາຮ່ວມສົນທະນາກ່ຽວກັບ `dotool` ໃນ discussion thread ນີ້: <https://github.com/stefansundin/SSHRemote/discussions/6>

### cec-client

ຄ່າຕັ້ງສໍາເລັດຮູບນີ້ຍັງເປັນແບບທົດລອງ ເພາະຂ້ອຍຍັງບໍ່ສາມາດຢືນຢັນໄດ້ກັບຮາດແວຂອງຂ້ອຍເອງ. ຍິນດີຮັບຄໍາຕິຊົມ.

ກະລຸນາຮ່ວມສົນທະນາກ່ຽວກັບ `cec-client` ໃນ discussion thread ນີ້: <https://github.com/stefansundin/SSHRemote/discussions/7>

### macOS

ມີຄ່າຕັ້ງສໍາເລັດຮູບສໍາລັບຄວບຄຸມ VLC ໃນ macOS ໂດຍໃຊ້ຄໍາສັ່ງ AppleScript. ຂ້ອຍບໍ່ຮູ້ຈັກເຄື່ອງມືໃດທີ່ຮອງຮັບການສົ່ງເຫດການແປ້ນພິມ ຫຼື ເມົາສ໌.

ກະລຸນາຮ່ວມສົນທະນາກ່ຽວກັບ macOS ໃນ discussion thread ນີ້: <https://github.com/stefansundin/SSHRemote/discussions/8>

### Android

ຖ້າອຸປະກອນ Android ຂອງທ່ານມີ SSH server ມາພ້ອມ, ທ່ານອາດສາມາດເຊື່ອມຕໍ່ໄປຫາມັນແລະສົ່ງ input events ໄດ້. ນີ້ມີໂອກາດເກີດຂຶ້ນຫຼາຍກວ່າຖ້າອຸປະກອນຂອງທ່ານໃຊ້ custom ROM, ເຊັ່ນ KonstaKANG ໃນ Raspberry Pi.

ຂ້ອຍຍັງບໍ່ສາມາດຫາວິທີໃຫ້ mouse support ເຮັດວຽກໄດ້.

ກະລຸນາຮ່ວມສົນທະນາກ່ຽວກັບ Android ໃນ discussion thread ນີ້: <https://github.com/stefansundin/SSHRemote/discussions/9>

## ການຄວບຄຸມລະດັບສຽງອັດສະລິຍະ

ເມື່ອແກ້ໄຂການຄວບຄຸມຣີໂມດ, ທ່ານສາມາດພົບການຕັ້ງຄ່າຄວບຄຸມລະດັບສຽງແບບ "ອັດສະລິຍະ" ໃນເມນູ. ມັນສາມາດສະແດງລະດັບສຽງປັດຈຸບັນຂອງຄອມພິວເຕີໃນແອັບ ແລະຊ່ວຍໃຫ້ທ່ານຕັ້ງຄ່າລະດັບສຽງໄດ້ໄວດ້ວຍ slider. ທ່ານຍັງສາມາດໃຊ້ປຸ່ມຮາດແວຂອງອຸປະກອນເພື່ອສົ່ງຄໍາສັ່ງເພີ່ມ/ຫຼຸດສຽງໄດ້ຢ່າງວ່ອງໄວ.

ການອ່ານລະດັບສຽງປັດຈຸບັນ ແລະການຕັ້ງຄ່າລະດັບສຽງໃໝ່ຜ່ານ slider ໃນຕອນນີ້ຖືກກໍານົດໃຫ້ໃຊ້ `pactl` ແບບຕາຍຕົວ.

package ທີ່ບັນຈຸ `pactl` ມັກຈະຊື່ `pulseaudio-utils` ຫຼື `libpulse`.

## ກະແຈ SSH

ທ່ານສາມາດນໍາເຂົ້າ ຫຼື ສ້າງກະແຈ SSH ໃນການຕັ້ງຄ່າຂອງແອັບ. ການເຊື່ອມຕໍ່ດ້ວຍກະແຈ SSH ປອດໄພກວ່າການໃຊ້ລະຫັດຜ່ານ.

ວິທີທີ່ງ່າຍທີ່ສຸດໃນການນໍາເຂົ້າກະແຈ SSH ທີ່ມີຢູ່ຈາກຄອມພິວເຕີ ແມ່ນການສະແກນ QR code. ທ່ານສາມາດໃຊ້ໂປຣແກຣມ `qrencode` ເພື່ອສ້າງຮູບ QR code ໄດ້. ຮັນຄໍາສັ່ງຄ້າຍຕໍ່ໄປນີ້ເພື່ອສ້າງ QR code:

```shell
# ໄປທີ່ໂຟນເດີ SSH keys ຂອງທ່ານ:
cd ~/.ssh

# ສະແດງ QR code ໃນ terminal:
qrencode -r id_ed25519 -t ansiutf8

# ຫຼືຈະສ້າງເປັນໄຟລ໌ຮູບພາບ:
qrencode -r id_ed25519 -o qr.png

# ກະແຈ RSA 4096-bit ມີຂະໜາດໃຫຍ່ເກີນສໍາລັບ QR code. ທ່ານສາມາດໃຊ້ gzip ເພື່ອໃຫ້ມັນພໍດີໄດ້ແບບສະບາຍໆ:
cat id_rsa_4096 | gzip | base64 | qrencode -o qr.png
```

ທ່ານສາມາດສົ່ງກະແຈ SSH ສາທາລະນະໄປຫາເຊີບເວີໄດ້ໂດຍໃຊ້ຟີເຈີ `ສົ່ງກະແຈສາທາລະນະ` ໃນເມນູ. ມັນຈະເພີ່ມກະແຈ SSH ທີ່ຖືກເລືອກເຂົ້າໄປໃນໄຟລ໌ `~/.ssh/authorized_keys`. ນີ້ຊ່ວຍໃຫ້ທ່ານຍ້າຍຈາກການ login ດ້ວຍລະຫັດຜ່ານໄປຫາການ login ດ້ວຍກະແຈ SSH ໄດ້ຢ່າງງ່າຍດາຍ.

ທ່ານສາມາດນໍາເຂົ້າແລະໃຊ້ກະແຈ SSH ທີ່ຖືກເຂົ້າລະຫັດໄດ້, ແຕ່ໃນຕອນນີ້ທ່ານຍັງບໍ່ສາມາດສ້າງມັນໃນແອັບໄດ້.

## ຄວາມປອດໄພ

ບໍ່ສາມາດສົ່ງອອກ ຫຼື ດຶງສ່ວນ private ຂອງກະແຈ SSH ຫຼື ລະຫັດຜ່ານທີ່ຖືກເກັບໄວ້ ອອກຈາກແອັບໄດ້. ຂໍ້ມູນນີ້ຖືກເຂົ້າລະຫັດດ້ວຍ AES 256-bit ແລະກະແຈເຂົ້າລະຫັດຖືກເກັບໄວ້ໃນ Android Keystore. ຂໍ້ມູນທີ່ຖືກເຂົ້າລະຫັດຈະບໍ່ຖືກລວມໃນ Android backups.

ໃນແອັບນີ້ບໍ່ມີ software ລາຍງານ crash. ບໍ່ມີ telemetry. ບໍ່ມີໂຄສະນາ. ບໍ່ມີ network request ນອກເໜືອຈາກການເຊື່ອມຕໍ່ SSH.

ຄວາມປອດໄພຂອງແອັບນີ້ຍັງບໍ່ຖືກ audit. ຖ້າທ່ານມີປະສົບການດ້ານ Android security ຫຼື SSH security, ກະລຸນາເບິ່ງ source code ແລະລາຍງານສິ່ງທີ່ທ່ານພົບໃນ GitHub issue ນີ້:

<https://github.com/stefansundin/SSHRemote/issues/1>

## ຄໍາຮ້ອງຂໍຟີເຈີ

ທ່ານສາມາດສົ່ງຄໍາຮ້ອງຂໍຟີເຈີ ແລະ ລາຍງານ bug ໄດ້ໃນ GitHub repository. ກະລຸນາໃຊ້ພາສາອັງກິດ. ກະລຸນາຮັກສາຄໍາເຫັນຂອງທ່ານໃຫ້ສຸພາບ. ຄໍາເຫັນທີ່ບໍ່ໃຫ້ຄວາມເຄົາລົບຈະຖືກລຶບ ແລະ ຜູ້ໃຊ້ອາດຖືກບລັອກຈາກ repository.

ກະລຸນາເບິ່ງ issue ແລະ discussion thread ທີ່ມີຢູ່ກ່ອນ ເພື່ອກວດວ່າຄໍາຖາມຂອງທ່ານເຄີຍຖືກຖາມ ຫຼື ຕອບໄປແລ້ວຫຼືບໍ່.

ກະລຸນາໃຫ້ຄວາມເຄົາລົບ. ຂ້ອຍສ້າງແອັບນີ້ໃນເວລາຫວ່າງຂອງຂ້ອຍ ແລະຂ້ອຍກໍາລັງແຈກມັນໃຫ້ໃຊ້ຟຣີ. ຂ້ອຍສ້າງແອັບນີ້ເພື່ອໃຊ້ເອງເປັນຫຼັກ.

ກະລຸນາຢ່າສົ່ງອີເມວມາຖາມຂ້ອຍ. ກະລຸນາພະຍາຍາມຮັກສາການສົນທະນາໄວ້ໃນ GitHub ເພາະມັນຊ່ວຍຄົນອື່ນໄດ້ເຊັ່ນກັນ! ທ່ານສາມາດຖາມຄໍາຖາມໃນສ່ວນ discussion ຂອງ GitHub ໄດ້.

ທ່ານຍິນດີສະເໝີທີ່ຈະ fork ແອັບນີ້ເພື່ອນໍາໄປພັດທະນາຟີເຈີຂອງທ່ານເອງ. ນັ້ນເປັນວິທີທີ່ດີໃນການຮຽນຮູ້. ກະລຸນາພິຈາລະນາຊ່ວຍປະກອບສ່ວນຟີເຈີທີ່ມີປະໂຫຍດ.
