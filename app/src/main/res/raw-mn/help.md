## SSH Remote-ийн тухай

Энэ орчуулгыг GitHub Copilot AI ашиглан хийсэн тул орчуулгын алдаа байж болзошгүй.

SSH Remote нь SSH ашиглан компьютеруудыг алсаас удирдах боломж олгодог, үнэгүй бөгөөд нээлттэй эхтэй апп юм.

Ажиллуулах тушаалуудыг та бүрэн өөрчилж болно, мөн түгээмэл тохиргоонуудад зориулсан урьдчилсан тохиргоонууд бий.

Би энэ аппыг Raspberry Pi OS ажиллуулж буй HTPC тохиргоогоо удирдахад ашигладаг. HTPC удирдах нь энэ аппын хамгийн их оновчлогдсон үндсэн хэрэглээний хувилбар юм.

Энэ апп нь терминал эмулятор биш, гэхдээ яаралтай үед `apt-get install` ажиллуулах боломж өгнө.

## Эхлүүлэх

Хэрэв та SSH түлхүүр ашиглан холбогдохыг хүсвэл эхлээд аппын тохиргоог нээгээд түлхүүр импортлох эсвэл үүсгээрэй.

Баруун доод буланд байгаа `+` товчийг дарж шинэ хост нэмнэ үү. Холболтын мэдээллээ оруулаад хадгална уу.

Анх удаа холбогдох үед танаас урьдчилсан тохиргоо сонгохыг хүснэ. Энэ сонголт нь алсын удирдлагын товчлууруудыг янз бүрийн төрлийн компьютерт сайн ажиллахаар тохируулдаг. Боломжтой урьдчилсан тохиргоонуудын тайлбарыг доороос үзнэ үү. Хэрэв хүсвэл `Урьдчилсан тохиргоогүй`-г сонгож ямар ч тохиргоогүйгээр эхэлж болно.

Хэрэв таны Linux компьютер X11 эсвэл Wayland ажиллуулж байгааг мэдэхгүй бол терминал дээр дараахыг ажиллуулна уу:

```shell
echo $XDG_SESSION_TYPE
```

Энэ нь `x11` эсвэл `wayland` гэж гаргах ёстой. Үүнийг та desktop environment дотроос ажиллуулах ёстой.

## Алсын удирдлага

Холбогдсоны дараа та алсын удирдлагын интерфэйсийг ашиглан тушаалууд илгээж болно. Янз бүрийн оролтын аргууд руу хандахын тулд табуудын хооронд шилжинэ үү.

Товч дарах бүрт хост дээр нэг тушаал ажиллана. Энэ нь энгийн товч даралттай харьцуулахад ихээхэн нэмэлт ачаалалтай тул энгийн гарын товчтой харьцуулахад нэлээд их саатал мэдрэгдэж магадгүй. Ирээдүйн хувилбаруудад үүнийг сайжруулна гэж найдаж байна.

Засах горимд орохын тулд цэсийг ашиглана уу. Одоогоор байршил болон товчны дүрсийг засах боломжгүй байна. Ирээдүйд үүнийг боломжтой болгоно гэж найдаж байна.

## Урьдчилсан тохиргоонууд

Та өөрийн desktop environment-д шаардлагатай хэрэгслийг суулгах хэрэгтэй.

Миний туршилтаар `ydotool` хамгийн сайн гүйцэтгэлтэй байсан бөгөөд X11 болон Wayland хоёуланд нь ажилладаг тул би үүнийг зөвлөж байна.

### ydotool

`ydotool` нь ямар ч window manager дээр ажиллах ёстой, гэхдээ арын үйлчилгээ ажиллаж байх шаардлагатай. Хэрэв таны түгээлт systemd user service өгдөг бол дараахыг ажиллуулж эхлүүлнэ үү:

```shell
systemctl start --user ydotool
```

Нэвтрэх үед үйлчилгээг автоматаар эхлүүлэхийн тулд дараахыг ажиллуулна уу:

```shell
systemctl enable --user ydotool
```

`ydotool`-ийн хангалттай шинэ хувилбар суулгаж байгаагаа шалгаарай. Ubuntu 26.04-өөс өмнөх хувилбарууд хэт хуучин хувилбар өгдөг. Түр шийдлийн талаар хэлэлцүүлгийн сэдвийг үзнэ үү.

`ydotool`-ийн талаар энэ хэлэлцүүлгийн сэдэвт ярилцана уу: <https://github.com/stefansundin/SSHRemote/discussions/5>

### xdotool

`xdotool` нь X11 ажиллуулж буй компьютерт зориулагдсан. X11 нь түүхийн хувьд ихэнх Linux компьютеруудын ашиглаж ирсэн систем боловч Wayland улам түгээмэл болж байна.

X11-ийн нэг онцлог нь танд X server-т хандалт зөвшөөрөх шаардлага гарч магадгүй юм. Хэрэв та "Authorization required" алдаа авч байвал энэ асуудал мөн. Энэ асуудлыг засах хэд хэдэн арга бий, надад ажилласан хоёр аргыг энд оруулав:

Хэрэв `xauth list` ямар ч бичлэг харуулахгүй бол `.Xauthority` файл үүсгээд үзээрэй:

```shell
xauth generate :0 . trusted
```

Хэрэв энэ нь ажиллахгүй бол `xhost` ашиглан хандалт олгоод үзээрэй:

```shell
xhost +local:$USER
```

Та `xhost` тушаалыг асаалт бүрийн дараа ажиллуулах хэрэгтэй болно. Үүнийг автоматжуулахын тулд bash script үүсгээд нэвтрэх үед автоматаар эхлүүлэхээр тохируулж болно.

`xdotool`-ийн талаар энэ хэлэлцүүлгийн сэдэвт ярилцана уу: <https://github.com/stefansundin/SSHRemote/discussions/3>

### wtype

`wtype` нь Wayland-д зориулсан `xdotool`-той төстэй.

Ердийн үед энэ нь хулганын удирдлагыг дэмждэггүй, гэхдээ би хулганын дэмжлэг нэмсэн өөрчилсөн хувилбар хийсэн. Хэрэв танд хулганын дэмжлэг хэрэгтэй бол үүнийг суулгана уу: <https://github.com/stefansundin/wtype>

Хэрэв `Compositor does not support the virtual keyboard protocol` алдаа гарвал `ydotool` зэрэг өөр хэрэгсэл туршиж үзэхийг зөвлөж байна.

`wtype`-ийн талаар энэ хэлэлцүүлгийн сэдэвт ярилцана уу: <https://github.com/stefansundin/SSHRemote/discussions/4>

### dotool

`dotool` нь `ydotool`-той адил ямар ч window manager дээр ажиллах ёстой. Миний хязгаарлагдмал туршилтаар энэ нь `ydotool`-оос хамаагүй удаан байсан.

`dotool`-ийн талаар энэ хэлэлцүүлгийн сэдэвт ярилцана уу: <https://github.com/stefansundin/SSHRemote/discussions/6>

### cec-client

Энэ урьдчилсан тохиргоо нь туршилтын шинжтэй, учир нь би үүнийг өөрийн төхөөрөмж дээр баталгаажуулж чадаагүй. Санал хүсэлт тавтай морилно уу.

`cec-client`-ийн талаар энэ хэлэлцүүлгийн сэдэвт ярилцана уу: <https://github.com/stefansundin/SSHRemote/discussions/7>

### macOS

macOS дээр VLC-г AppleScript тушаалаар удирдах урьдчилсан тохиргоо бий. Гарын эсвэл хулганы үйлдэл илгээхийг дэмждэг хэрэгсэл миний мэдэхээр байхгүй.

macOS-ийн талаар энэ хэлэлцүүлгийн сэдэвт ярилцана уу: <https://github.com/stefansundin/SSHRemote/discussions/8>

### Android

Хэрэв таны Android төхөөрөмж SSH сервертэй ирдэг бол түүн рүү холбогдож оролтын үйлдэл илгээх боломжтой байж магадгүй. Энэ нь Raspberry Pi дээрх KonstaKANG зэрэг custom ROM ажиллуулж байвал илүү магадлалтай.

Хулганын дэмжлэгийг хэрхэн ажиллуулахыг би одоогоор олж чадаагүй байна.

Android-ийн талаар энэ хэлэлцүүлгийн сэдэвт ярилцана уу: <https://github.com/stefansundin/SSHRemote/discussions/9>

## Ухаалаг дууны удирдлага

Алсын удирдлагыг засах үед цэснээс "ухаалаг" дууны удирдлагын тохиргоонуудыг олж болно. Энэ нь компьютерийн одоогийн дууны түвшнийг апп дотор харуулах, мөн гулсуур ашиглан дууг хурдан тохируулах боломж олгоно. Мөн төхөөрөмжийнхөө физик товчлууруудыг ашиглан дууг нэмэх/багасгах тушаалуудыг хурдан илгээж болно.

Одоогийн дууны түвшинг унших болон гулсуураар шинэ түвшин тохируулах нь одоогоор `pactl` ашиглахаар хатуу тохируулагдсан.

`pactl` агуулдаг багцыг ихэвчлэн `pulseaudio-utils` эсвэл `libpulse` гэж нэрлэдэг.

## SSH түлхүүрүүд

Та аппын тохиргооноос SSH түлхүүр импортлох эсвэл үүсгэж болно. SSH түлхүүрээр холбогдох нь нууц үг ашиглахаас илүү аюулгүй.

Компьютероос байгаа SSH түлхүүр импортлох хамгийн хялбар арга бол QR код уншуулах явдал юм. Та QR кодын зургийг үүсгэхийн тулд `qrencode` програмыг ашиглаж болно. Дараахтай төстэй тушаал ажиллуулж QR код үүсгэнэ үү:

```shell
# Өөрийн SSH түлхүүрүүд рүү очих:
cd ~/.ssh

# QR кодыг терминал дээр харуулах:
qrencode -r id_ed25519 -t ansiutf8

# Эсвэл зургийн файл үүсгэх:
qrencode -r id_ed25519 -o qr.png

# 4096-бит RSA түлхүүрүүд QR кодод хэт том байдаг. gzip ашиглавал арай гэж багтана:
cat id_rsa_4096 | gzip | base64 | qrencode -o qr.png
```

Та цэсний `Нийтийн түлхүүр илгээх` боломжийг ашиглан нийтийн SSH түлхүүрийг сервер рүү илгээж болно. Энэ нь сонгосон SSH түлхүүрийг `~/.ssh/authorized_keys` файлд нэмнэ. Ингэснээр нууц үгээр нэвтрэхээс SSH түлхүүрээр нэвтрэх рүү амархан шилжих боломжтой.

Та шифрлэгдсэн SSH түлхүүрүүдийг импортолж ашиглаж болно, гэхдээ одоогоор апп дотор ийм түлхүүр үүсгэх боломжгүй.

## Аюулгүй байдал

Аппаас SSH түлхүүрийн хувийн хэсгийг эсвэл хадгалсан нууц үгсийг экспортлох, гаргаж авах боломжгүй. Энэ өгөгдөл нь 256-бит AES ашиглан шифрлэгддэг бөгөөд шифрлэлтийн түлхүүр нь Android Keystore-д хадгалагддаг. Шифрлэгдсэн өгөгдөл Android нөөцөөс хасагддаг.

Энэ аппад crash reporting програм хангамж байхгүй. Телеметр байхгүй. Зар сурталчилгаа байхгүй. SSH холболтоос өөр сүлжээний хүсэлт байхгүй.

Энэ аппын аюулгүй байдалд аудит хийгдээгүй. Хэрэв та Android аюулгүй байдал эсвэл SSH аюулгүй байдлын туршлагатай бол эх кодыг үзэж, олсон зүйлээ энэ GitHub issue-д мэдэгдэнэ үү:

<https://github.com/stefansundin/SSHRemote/issues/1>
