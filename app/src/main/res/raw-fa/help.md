## درباره SSH Remote

این ترجمه فارسی با GitHub Copilot (مدل مورد استفاده در این محیط به‌صورت دقیق در دسترس نیست) تولید شده است و ممکن است خطاهای ترجمه وجود داشته باشد.

SSH Remote یک برنامه رایگان و متن‌باز است که به شما اجازه می‌دهد با استفاده از SSH رایانه‌ها را از راه دور کنترل کنید.

شما می‌توانید دستورهایی را که اجرا می‌شوند به‌طور کامل سفارشی کنید، و برای تنظیمات رایج نیز پیش‌تنظیم‌هایی وجود دارد.

من از این برنامه برای کنترل راه‌اندازی HTPC خودم که روی Raspberry Pi OS اجرا می‌شود استفاده می‌کنم. کنترل HTPC سناریوی اصلی‌ای است که برنامه برای آن بهینه شده است.

این برنامه یک شبیه‌ساز ترمینال نیست، اما در مواقع اضطراری به شما اجازه می‌دهد `apt-get install` را اجرا کنید.

## شروع کار

اگر می‌خواهید برای اتصال از یک کلید SSH استفاده کنید، ابتدا تنظیمات برنامه را باز کنید و یک کلید را وارد یا ایجاد کنید.

با زدن دکمه `+` در پایین سمت راست، یک میزبان جدید اضافه کنید. جزئیات اتصال را وارد کرده و ذخیره کنید.

اولین باری که متصل می‌شوید، از شما خواسته می‌شود یک پیش‌تنظیم انتخاب کنید. این انتخاب دکمه‌های کنترل از راه دور را طوری پیکربندی می‌کند که روی انواع مختلف رایانه‌ها به‌خوبی کار کنند. برای توضیح پیش‌تنظیم‌های موجود، بخش پایین را ببینید. اگر ترجیح می‌دهید، می‌توانید با انتخاب `بدون پیش‌تنظیم` بدون هیچ پیکربندی‌ای شروع کنید.

اگر نمی‌دانید رایانه لینوکسی شما از X11 استفاده می‌کند یا Wayland، این دستور را در یک ترمینال اجرا کنید:

```shell
echo $XDG_SESSION_TYPE
```

باید `x11` یا `wayland` را چاپ کند. این دستور را باید درون محیط دسکتاپ اجرا کنید.

## کنترل از راه دور

پس از اتصال، می‌توانید از رابط کنترل از راه دور برای فرستادن دستورها استفاده کنید. برای دسترسی به روش‌های مختلف ورودی، بین زبانه‌ها جابه‌جا شوید.

هر بار فشردن یک دکمه، یک دستور را روی میزبان اجرا می‌کند. این کار برای چیزی به سادگی یک فشردن کلید، سربار زیادی دارد و ممکن است در مقایسه با یک صفحه‌کلید معمولی، تأخیر نسبتاً زیادی را تجربه کنید. امیدوارم در نسخه‌های آینده این مورد را بهبود بدهم.

از منو برای ورود به حالت ویرایش استفاده کنید. در حال حاضر امکان ویرایش چیدمان یا آیکون‌های دکمه‌ها وجود ندارد. امیدوارم در نسخه‌های آینده این امکان را اضافه کنم.

## پیش‌تنظیم‌ها

باید ابزار موردنیاز محیط دسکتاپ خود را نصب کنید.

من `ydotool` را پیشنهاد می‌کنم، چون در آزمایش‌های من بهترین عملکرد را داشته و هم روی X11 و هم Wayland کار می‌کند.

### ydotool

`ydotool` باید با هر مدیر پنجره‌ای کار کند، اما نیاز به یک سرویس پس‌زمینه در حال اجرا دارد. اگر توزیع شما یک سرویس کاربری systemd ارائه می‌دهد، آن را با این دستور اجرا کنید:

```shell
systemctl start --user ydotool
```

برای اینکه سرویس هنگام ورود به سیستم به‌طور خودکار شروع شود، این دستور را اجرا کنید:

```shell
systemctl enable --user ydotool
```

لطفاً مطمئن شوید نسخه‌ای به‌اندازه کافی جدید از `ydotool` را نصب می‌کنید. نسخه‌های Ubuntu پیش از 26.04 نسخه‌هایی ارائه می‌دهند که بیش از حد قدیمی هستند. برای یک راه‌حل جایگزین، موضوع بحث را ببینید.

لطفاً درباره `ydotool` در این موضوع بحث گفتگو کنید: <https://github.com/stefansundin/SSHRemote/discussions/5>

### xdotool

`xdotool` برای رایانه‌هایی است که از X11 استفاده می‌کنند. X11 همان چیزی است که بیشتر رایانه‌های لینوکسی در گذشته از آن استفاده کرده‌اند، هرچند Wayland در حال محبوب‌تر شدن است.

یکی از نکته‌های خاص X11 این است که ممکن است لازم باشد دسترسی به X server را مجاز کنید. اگر با خطاهای «Authorization required» روبه‌رو می‌شوید، احتمالاً مشکل همین است. چند راه برای رفع این مشکل وجود دارد؛ در اینجا دو روشی را که برای من جواب داده‌اند آورده‌ام:

اگر `xauth list` هیچ ورودی‌ای نشان نمی‌دهد، این را امتحان کنید تا یک فایل `.Xauthority` ایجاد شود:

```shell
xauth generate :0 . trusted
```

اگر این کار جواب نداد، این را امتحان کنید تا با استفاده از `xhost` دسترسی بدهید:

```shell
xhost +local:$USER
```

باید دستور `xhost` را پس از هر بار راه‌اندازی سیستم اجرا کنید. می‌توانید این کار را با ساختن یک اسکریپت bash و پیکربندی آن برای اجرای خودکار هنگام ورود به سیستم، خودکار کنید.

لطفاً درباره `xdotool` در این موضوع بحث گفتگو کنید: <https://github.com/stefansundin/SSHRemote/discussions/3>

### wtype

`wtype` شبیه `xdotool` است، اما برای Wayland.

به‌طور معمول از کنترل ماوس پشتیبانی نمی‌کند، اما من یک نسخه دست‌کاری‌شده ساخته‌ام که پشتیبانی از ماوس را اضافه می‌کند. اگر به پشتیبانی از ماوس نیاز دارید، لطفاً آن را نصب کنید: <https://github.com/stefansundin/wtype>

اگر با خطای `Compositor does not support the virtual keyboard protocol` مواجه شدید، پیشنهاد می‌کنم ابزار دیگری مانند `ydotool` را امتحان کنید.

لطفاً درباره `wtype` در این موضوع بحث گفتگو کنید: <https://github.com/stefansundin/SSHRemote/discussions/4>

### dotool

`dotool` مشابه `ydotool` باید با هر مدیر پنجره‌ای کار کند. در آزمایش محدود من، خیلی کندتر از `ydotool` بود.

لطفاً درباره `dotool` در این موضوع بحث گفتگو کنید: <https://github.com/stefansundin/SSHRemote/discussions/6>

### cec-client

این پیش‌تنظیم آزمایشی است، چون نتوانسته‌ام آن را روی سخت‌افزار خودم بررسی کنم. بازخورد شما خوش‌آمد است.

لطفاً درباره `cec-client` در این موضوع بحث گفتگو کنید: <https://github.com/stefansundin/SSHRemote/discussions/7>

### macOS

یک پیش‌تنظیم برای کنترل VLC در macOS با استفاده از دستورهای AppleScript وجود دارد. من از ابزاری که از ارسال رویدادهای صفحه‌کلید یا ماوس پشتیبانی کند اطلاع ندارم.

لطفاً درباره macOS در این موضوع بحث گفتگو کنید: <https://github.com/stefansundin/SSHRemote/discussions/8>

### Android

اگر دستگاه Android شما همراه با یک SSH server ارائه می‌شود، شاید بتوانید به آن متصل شوید و رویدادهای ورودی بفرستید. اگر دستگاه شما از یک ROM سفارشی مثل KonstaKANG روی Raspberry Pi استفاده می‌کند، این احتمال بیشتر است.

من هنوز نتوانسته‌ام راهی برای کار کردن پشتیبانی از ماوس پیدا کنم.

لطفاً درباره Android در این موضوع بحث گفتگو کنید: <https://github.com/stefansundin/SSHRemote/discussions/9>

## کنترل هوشمند صدا

هنگام ویرایش کنترل از راه دور، می‌توانید تنظیمات کنترل صدای «هوشمند» را در منو پیدا کنید. این قابلیت می‌تواند صدای فعلی رایانه را در برنامه نمایش دهد و به شما اجازه دهد با یک لغزنده، صدا را سریع تنظیم کنید. همچنین می‌توانید از دکمه‌های سخت‌افزاری دستگاه خود برای فرستادن سریع دستورهای افزایش/کاهش صدا استفاده کنید.

خواندن صدای فعلی و تنظیم صدای جدید با استفاده از لغزنده در حال حاضر به‌صورت ثابت برای استفاده از `pactl` نوشته شده است.

بسته‌ای که `pactl` را در خود دارد معمولاً `pulseaudio-utils` یا `libpulse` نام دارد.

## کلیدهای SSH

شما می‌توانید در تنظیمات برنامه کلیدهای SSH را وارد یا ایجاد کنید. اتصال با کلید SSH از استفاده از گذرواژه امن‌تر است.

ساده‌ترین راه برای وارد کردن یک کلید SSH موجود از یک رایانه، اسکن کردن یک کد QR است. می‌توانید از برنامه `qrencode` برای ساخت تصویر کد QR استفاده کنید. دستوری مانند نمونه زیر را اجرا کنید تا کد QR ساخته شود:

```shell
# به پوشه کلیدهای SSH خود بروید:
cd ~/.ssh

# کد QR را در ترمینال نمایش دهید:
qrencode -r id_ed25519 -t ansiutf8

# یا به‌جای آن یک فایل تصویری بسازید:
qrencode -r id_ed25519 -o qr.png

# کلیدهای RSA با اندازه 4096 بیت برای یک کد QR بیش از حد بزرگ هستند. می‌توانید با gzip آن را به‌سختی جا بدهید:
cat id_rsa_4096 | gzip | base64 | qrencode -o qr.png
```

می‌توانید با استفاده از قابلیت `ارسال کلید عمومی` در منو، کلیدهای عمومی SSH را به یک سرور بفرستید. این کار کلید SSH انتخاب‌شده را به فایل `~/.ssh/authorized_keys` اضافه می‌کند. این قابلیت به شما اجازه می‌دهد به‌راحتی از ورود با گذرواژه به ورود با کلید SSH مهاجرت کنید.

می‌توانید کلیدهای SSH رمزگذاری‌شده را وارد و استفاده کنید، اما در حال حاضر نمی‌توانید آن‌ها را در برنامه ایجاد کنید.

## امنیت

خروجی گرفتن یا استخراج بخش خصوصی کلیدهای SSH یا گذرواژه‌های ذخیره‌شده از برنامه ممکن نیست. این داده‌ها با AES 256 بیتی رمزگذاری می‌شوند و کلید رمزگذاری در Android Keystore ذخیره می‌شود. داده‌های رمزگذاری‌شده از نسخه‌های پشتیبان Android حذف می‌شوند.

در این برنامه هیچ نرم‌افزار گزارش خرابی وجود ندارد. هیچ تله‌متری‌ای وجود ندارد. هیچ تبلیغی وجود ندارد. هیچ درخواست شبکه‌ای وجود ندارد، به‌جز اتصال SSH.

امنیت این برنامه ممیزی نشده است. اگر در زمینه امنیت Android یا امنیت SSH تجربه دارید، لطفاً کد منبع را بررسی کنید و یافته‌های خود را در این issue در GitHub گزارش کنید:

<https://github.com/stefansundin/SSHRemote/issues/1>
