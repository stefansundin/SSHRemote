## SSH Remote ගැන

මෙම පරිවර්තනය GitHub Copilot (GPT-5.3-Codex) විසින් සාදන ලදී. පරිවර්තන දෝෂ හෝ ස්වභාවික නොවන වාක්‍ය රීති තිබිය හැක.

SSH Remote යනු SSH භාවිතයෙන් දුරස්ථව පරිගණක පාලනය කිරීමට ඔබට ඉඩ දෙන නොමිලේ හා විවෘත මූලාශ්‍ර යෙදුමකි.

ක්‍රියාත්මක වන විධාන සම්පූර්ණයෙන්ම අභිරුචිකරණය කළ හැකි අතර, සාමාන්‍ය සැකසුම් සඳහා පෙරසිටුවීම්ද ඇත.

මම මෙම යෙදුම භාවිත කරන්නේ Raspberry Pi OS ධාවනය වන මගේ HTPC සැකසුම පාලනය කිරීමටය. යෙදුම මූලිකවම ප්‍රශස්ත කර ඇත්තේ HTPC පාලනය කිරීමේ මෙම මූලික භාවිත අවස්ථාව සඳහාය.

මෙම යෙදුම ටර්මිනල් ඉමුලේටරයක් නොවේ, නමුත් හදිසි අවස්ථාවකදී `apt-get install` ධාවනය කිරීමට එය ඔබට ඉඩ දේ.

## ආරම්භ කිරීම

සම්බන්ධ වීමට SSH යතුරක් භාවිත කිරීමට ඔබට අවශ්‍ය නම්, මුලින්ම යෙදුම් සැකසුම් විවෘත කර යතුරක් ආයාත කරන්න හෝ ජනනය කරන්න.

පහළ දකුණු කෙළවරේ ඇති `+` බොත්තම තට්ටු කර නව ධාරකයක් එක් කරන්න. සම්බන්ධතා විස්තර ඇතුළත් කර සුරකින්න.

ඔබ පළමු වරට සම්බන්ධ වන විට, පෙරසිටුවීමක් තෝරා ගැනීමට ඔබෙන් අසනු ඇත. මෙම තේරීම මඟින් විවිධ වර්ගයේ පරිගණකවලට හොඳින් ගැලපෙන ලෙස දුරස්ථ පාලක බොත්තම් සකස් කරයි. පවතින පෙරසිටුවීම් පිළිබඳ විස්තරයක් සඳහා පහත බලන්න. ඔබට කැමති නම්, `No preset` තෝරා කිසිදු සැකසුමකින් තොරව ආරම්භ කළ හැක.

ඔබගේ Linux පරිගණකය X11 ද Wayland ද ධාවනය වන්නේ දැයි ඔබ නොදන්නා නම්, ටර්මිනලයක මෙය ධාවනය කරන්න:

```shell
echo $XDG_SESSION_TYPE
```

එය `x11` හෝ `wayland` ලෙස ප්‍රතිදානය විය යුතුය. මෙය ඩෙස්ක්ටොප් පරිසරය ඇතුළත ධාවනය කළ යුතුය.

## දුරස්ථ පාලකය

සම්බන්ධ වූ පසු, විධාන යැවීමට ඔබට දුරස්ථ පාලක අතුරුමුහුණත භාවිත කළ හැක. විවිධ ආදාන ක්‍රම වෙත ප්‍රවේශ වීමට ටැබ් මාරු කරන්න.

සෑම බොත්තමක් ඔබන විටම ධාරකය මත විධානයක් ක්‍රියාත්මක වේ. යතුරක් ඔබීම වැනි සරල දෙයක් සඳහා මෙය සැලකිය යුතු පරිමාවේ overhead එකක් වන බැවින්, සාමාන්‍ය යතුරුපුවරුවකට සාපේක්ෂව ඔබට තරමක් ඉහළ latency එකක් දැනෙන්න පුළුවන්. අනාගත අනුවාද වලදී මෙය වැඩිදියුණු කිරීමට මම බලාපොරොත්තු වෙමි.

සංස්කරණ ප්‍රකාරයට පිවිසීමට මෙනුව භාවිත කරන්න. දැනට layout එක හෝ button icons සංස්කරණය කළ නොහැක. අනාගත අනුවාදයකදී මෙය කළ හැකි බවට මම බලාපොරොත්තු වෙමි.

## පෙරසිටුවීම්

ඔබගේ ඩෙස්ක්ටොප් පරිසරයට අවශ්‍ය මෙවලම ස්ථාපනය කළ යුතුය.

මම `ydotool` නිර්දේශ කරමි, මගේ පරීක්ෂණ අනුව එයට හොඳම කාර්ය සාධනය තිබෙන අතර එය X11 සහ Wayland දෙකටම ක්‍රියා කරයි.

### ydotool

`ydotool` ඕනෑම window manager එකක් සමඟ ක්‍රියා කළ යුතුය, නමුත් ඔබට පසුබිම් සේවාවක් ධාවනය වීම අවශ්‍ය වේ. ඔබගේ distribution එක systemd user service එකක් සපයන්නේ නම්, මෙය ධාවනය කර එය ආරම්භ කරන්න:

```shell
systemctl start --user ydotool
```

login වෙලාවේදී සේවාව ස්වයංක්‍රීයව ආරම්භ වීමට මෙය ධාවනය කරන්න:

```shell
systemctl enable --user ydotool
```

කරුණාකර ඔබ `ydotool` හි ප්‍රමාණවත් තරම් නව අනුවාදයක් ස්ථාපනය කර ඇති බව තහවුරු කරගන්න. Ubuntu 26.04 ට පෙර අනුවාදවල පවතින සංස්කරණ ඉතා පැරණි වන අතර ප්‍රමාණවත් නොවේ. විසඳුමක් සඳහා discussion thread එක බලන්න.

කරුණාකර `ydotool` පිළිබඳව මෙම discussion thread එකේ සාකච්ඡා කරන්න: <https://github.com/stefansundin/SSHRemote/discussions/5>

### xdotool

`xdotool` යනු X11 ධාවනය වන පරිගණක සඳහාය. X11 යනු ඉතිහාසපුරා බොහෝ Linux පරිගණක භාවිත කළ පද්ධතියකි, නමුත් Wayland දැන් වැඩි වශයෙන් ජනප්‍රිය වෙමින් පවතී.

X11 හි එක් විශේෂතාවයක් වන්නේ ඔබට X server වෙත ප්‍රවේශය ඉඩ දිය යුතු වීමයි. ඔබට "Authorization required" දෝෂ ලැබේ නම් ප්‍රශ්නය එයයි. මෙම ගැටලුව විසඳීමට විකල්ප කිහිපයක් ඇත; මට ක්‍රියා කළ විකල්ප දෙකක් මෙන්න:

`xauth list` කිසිදු ඇතුළත් කිරීමක් නොපෙන්වන්නේ නම්, `.Xauthority` ගොනුවක් ජනනය කිරීමට මෙය උත්සාහ කරන්න:

```shell
xauth generate :0 . trusted
```

එය ක්‍රියා නොකළේ නම්, `xhost` භාවිතයෙන් ප්‍රවේශය ලබා දීමට මෙය උත්සාහ කරන්න:

```shell
xhost +local:$USER
```

ඔබට මෙම `xhost` විධානය සෑම boot එකකට පසුවම ධාවනය කළ යුතුය. ඔබට මෙය ස්වයංක්‍රීය කිරීමට bash script එකක් සාදා login වෙලාවේදී autostart ලෙස සකස් කළ හැක.

කරුණාකර `xdotool` පිළිබඳව මෙම discussion thread එකේ සාකච්ඡා කරන්න: <https://github.com/stefansundin/SSHRemote/discussions/3>

### wtype

`wtype` යනු Wayland සඳහා වන `xdotool` වැනිය.

සාමාන්‍යයෙන් එය මූසික පාලනයට සහය නොදක්වයි, නමුත් මම මූසික සහය එක් කරන ලද වෙනස් කළ සංස්කරණයක් සාදා ඇත. ඔබට මූසික සහය අවශ්‍ය නම් කරුණාකර එය ස්ථාපනය කරන්න: <https://github.com/stefansundin/wtype>

ඔබට `Compositor does not support the virtual keyboard protocol` දෝෂය ලැබේ නම්, `ydotool` වැනි වෙනත් මෙවලමක් උත්සාහ කරන ලෙස මම යෝජනා කරමි.

කරුණාකර `wtype` පිළිබඳව මෙම discussion thread එකේ සාකච්ඡා කරන්න: <https://github.com/stefansundin/SSHRemote/discussions/4>

## ස්මාර්ට් ශබ්ද පාලනය

දුරස්ථ පාලකය සංස්කරණය කරන විට, මෙනුව තුළ "smart" volume control settings ඔබට හමුවේ. මෙය යෙදුම තුළ පරිගණකයේ වත්මන් ශබ්ද මට්ටම පෙන්විය හැකි අතර slider එකක් භාවිතයෙන් එය ඉක්මනින් සකස් කිරීමට ඉඩ සලසයි. ඔබගේ උපාංගයේ hardware buttons භාවිතයෙන් ඉක්මනින් ශබ්දය වැඩි/අඩු කිරීමේ විධාන යැවිය හැක.

වත්මන් ශබ්ද මට්ටම කියවීම සහ slider එක භාවිතයෙන් නව අගයක් සකස් කිරීම දැනට hard-code කර ඇත්තේ `pactl` භාවිත කිරීමටය.

`pactl` අඩංගු package එක සාමාන්‍යයෙන් `pulseaudio-utils` හෝ `libpulse` ලෙස හැඳින්වේ.

## SSH යතුරු

ඔබට යෙදුම් සැකසුම් තුළ SSH යතුරු ආයාත හෝ ජනනය කළ හැක. SSH යතුරක් භාවිතයෙන් සම්බන්ධ වීම මුරපද භාවිතයට වඩා ආරක්ෂිතය.

පරිගණකයකින් පවතින SSH යතුරක් ආයාත කිරීමට පහසුම ක්‍රමය වන්නේ QR කේතයක් ස්කෑන් කිරීමයි. QR කේත රූපය ජනනය කිරීමට ඔබට `qrencode` වැඩසටහන භාවිත කළ හැක. QR කේතය ජනනය කිරීමට පහත පරිදි විධානයක් ධාවනය කරන්න:

```shell
# ඔබගේ SSH යතුරු වෙත යන්න:
cd ~/.ssh

# QR කේතය ටර්මිනලයේ පෙන්වන්න:
qrencode -r id_ed25519 -t ansiutf8

# විකල්ප ලෙස, රූප ගොනුවක් සාදන්න:
qrencode -r id_ed25519 -o qr.png

# 4096-bit RSA යතුරු QR කේතයකට විශාලය. gzip භාවිතයෙන් එය අමාරුවෙන් ගැලපෙන තරම් කුඩා කළ හැක:
cat id_rsa_4096 | gzip | base64 | qrencode -o qr.png
```

මෙනුවේ ඇති `පොදු යතුර යවන්න` විශේෂාංගය භාවිතයෙන් ඔබට public SSH keys server එකකට යැවිය හැක. මෙය තෝරාගත් SSH යතුර `~/.ssh/authorized_keys` ගොනුවට append කරයි. මෙය මුරපදයෙන් login වීමෙන් SSH යතුරකින් login වීමට පහසුවෙන් මාරු වීමට උපකාරී වේ.

ඔබට සංකේතනය කළ SSH යතුරු ආයාත කර භාවිත කළ හැක, නමුත් දැනට යෙදුම තුළ එවැනි යතුරු ජනනය කළ නොහැක.

## ආරක්ෂාව

යෙදුමෙන් SSH යතුරු වල private කොටස හෝ ගබඩා කළ මුරපද export කිරීමට හෝ extract කිරීමට නොහැක. මෙම දත්ත 256-bit AES භාවිතයෙන් සංකේතනය කර ඇති අතර encryption key එක Android Keystore තුළ ගබඩා කර ඇත. සංකේතනය කළ දත්ත Android backups වලින් බැහැර කර ඇත.

මෙම යෙදුම තුළ crash reporting software නොමැත. telemetry නොමැත. ads නොමැත. SSH සම්බන්ධතාවය හැර වෙනත් network requests කිසිවක් නොමැත.

මෙම යෙදුමේ ආරක්ෂාව audit කර නැත. ඔබ Android security හෝ SSH security පිළිබඳ පළපුරුදු නම්, කරුණාකර source code එක බලලා ඔබගේ සොයාගැනීම් මෙම GitHub issue එකේ වාර්තා කරන්න:

<https://github.com/stefansundin/SSHRemote/issues/1>
