## SSH Remote વિશે

આ અનુવાદ GitHub Copilot દ્વારા AI ની મદદથી બનાવવામાં આવ્યો છે. આ પર્યાવરણમાં મોડેલનું નામ ઉપલબ્ધ નથી, તેથી અહીં તેને ચોક્કસ રીતે દર્શાવી શકાયું નથી. અનુવાદમાં ભૂલો હોઈ શકે છે.

SSH Remote એક મફત અને ઓપન સોર્સ એપ છે જે તમને SSH નો ઉપયોગ કરીને કમ્પ્યુટર્સને દૂરસ્થ રીતે નિયંત્રિત કરવાની મંજૂરી આપે છે.

તમે ચલાવવામાં આવતા આદેશોને સંપૂર્ણપણે કસ્ટમાઇઝ કરી શકો છો, અને સામાન્ય સેટઅપ માટે પ્રિસેટ્સ ઉપલબ્ધ છે.

હું આ એપનો ઉપયોગ મારા HTPC સેટઅપને નિયંત્રિત કરવા માટે કરું છું, જે Raspberry Pi OS પર ચાલે છે. HTPC ને નિયંત્રિત કરવું એ મૂળભૂત પરિસ્થિતિ છે જેના માટે આ એપ ઑપ્ટિમાઇઝ કરવામાં આવી છે.

આ એપ ટર્મિનલ ઇમ્યુલેટર નથી, પરંતુ જરૂર પડે તો તમને `apt-get install` ચલાવવા દેશે.

## શરૂઆત

જો તમે જોડાવા માટે SSH કી વાપરવા માંગો છો, તો પહેલા એપ સેટિંગ્સ ખોલો અને કી આયાત કરો અથવા બનાવો.

નીચે જમણી બાજુના `+` બટન પર ટૅપ કરીને નવો હોસ્ટ ઉમેરો. કનેક્શન વિગતો દાખલ કરો અને સાચવો.

જ્યારે તમે પહેલી વાર જોડાશો, ત્યારે તમને એક પ્રિસેટ પસંદ કરવા કહેવામાં આવશે. આ પસંદગી રિમોટ કંટ્રોલના બટનોને વિવિધ પ્રકારના કમ્પ્યુટરો પર સારી રીતે કામ કરે એવી રીતે ગોઠવે છે. ઉપલબ્ધ પ્રિસેટ્સનું વર્ણન નીચે જુઓ. જો તમને ગમતું હોય, તો તમે `No preset` પસંદ કરીને કોઈ રૂપરેખાંકન વિના પણ શરૂઆત કરી શકો છો.

જો તમને ખબર ન હોય કે તમારું Linux કમ્પ્યુટર X11 પર ચાલી રહ્યું છે કે Wayland પર, તો ટર્મિનલમાં આ ચલાવો:

```shell
echo $XDG_SESSION_TYPE
```

આ `x11` અથવા `wayland` આઉટપુટ કરવું જોઈએ. તમારે આ command desktop environment ની અંદર ચલાવવી પડશે.

## રિમોટ કંટ્રોલ

એકવાર જોડાઈ ગયા પછી, તમે આદેશો મોકલવા માટે રિમોટ કંટ્રોલ ઇન્ટરફેસનો ઉપયોગ કરી શકો છો. વિવિધ ઇનપુટ પદ્ધતિઓ માટે tabs બદલો.

દરેક બટન દબાવવાથી હોસ્ટ પર એક આદેશ ચલાવવામાં આવશે. એક સામાન્ય કી દબાવવા જેવી ક્રિયા માટે આ ઘણું overhead છે, તેથી સામાન્ય કીબોર્ડની તુલનામાં તમને થોડું વધારે latency અનુભવી શકાય છે. હું ભવિષ્યની આવૃત્તિઓમાં આ સુધારવાની આશા રાખું છું.

edit mode માં જવા માટે menu નો ઉપયોગ કરો. હાલમાં layout અથવા button icons સંપાદિત કરવી શક્ય નથી. હું આશા રાખું છું કે ભવિષ્યની આવૃત્તિમાં આ શક્ય બનાવી શકીશ.

## પ્રિસેટ્સ

તમારે તમારા desktop environment માટે જરૂરી tool ઇન્સ્ટોલ કરવું પડશે.

હું `ydotool` ની ભલામણ કરું છું કારણ કે મારા પરીક્ષણમાં તેની કામગીરી સર્વોત્તમ રહી છે, અને તે X11 તથા Wayland બંનેમાં કામ કરે છે.

### ydotool

`ydotool` કોઈપણ window manager સાથે કામ કરવું જોઈએ, પરંતુ તેની માટે background service ચાલુ હોવી જરૂરી છે. જો તમારી distribution systemd user service પ્રદાન કરે છે, તો તેને આ રીતે શરૂ કરો:

```shell
systemctl start --user ydotool
```

લૉગિન સમયે service આપમેળે શરૂ થાય તે માટે આ ચલાવો:

```shell
systemctl enable --user ydotool
```

કૃપા કરીને ખાતરી કરો કે તમે `ydotool` નું પૂરતું નવું version ઇન્સ્ટોલ કરી રહ્યા છો. Ubuntu 26.04 પહેલાંની આવૃત્તિઓમાં મળતા versions બહુ જૂના છે. workaround માટે ચર્ચા થ્રેડ જુઓ.

કૃપા કરીને `ydotool` અંગે આ ચર્ચા થ્રેડમાં ચર્ચા કરો: <https://github.com/stefansundin/SSHRemote/discussions/5>

### xdotool

`xdotool` X11 પર ચાલતા કમ્પ્યુટરો માટે છે. ઐતિહાસિક રીતે મોટાભાગના Linux કમ્પ્યુટરો X11 વાપરતા આવ્યા છે, જો કે Wayland વધુ લોકપ્રિય બની રહ્યું છે.

X11 ની એક ખાસ બાબત એ છે કે તમને કદાચ X server માટે ઍક્સેસની પરવાનગી આપવી પડી શકે. જો તમને "Authorization required" જેવી ભૂલો મળે છે, તો કદાચ આ જ સમસ્યા છે. આ સમસ્યાને ઠીક કરવા માટે તમારી પાસે કેટલાક વિકલ્પો છે; અહીં બે એવા વિકલ્પો છે જેઓ મારા માટે કામ કર્યા છે:

જો `xauth list` કોઈ entries બતાવતું ન હોય, તો `.Xauthority` ફાઇલ બનાવવા પ્રયત્ન કરો:

```shell
xauth generate :0 . trusted
```

જો તે કામ ન કરે, તો `xhost` વડે ઍક્સેસ આપવાનો પ્રયત્ન કરો:

```shell
xhost +local:$USER
```

તમારે દરેક boot પછી `xhost` command ચલાવવી પડશે. તમે bash script બનાવી અને login સમયે autostart ગોઠવીને આ automate કરી શકો છો.

કૃપા કરીને `xdotool` અંગે આ ચર્ચા થ્રેડમાં ચર્ચા કરો: <https://github.com/stefansundin/SSHRemote/discussions/3>

### wtype

`wtype` `xdotool` જેવું છે, પરંતુ Wayland માટે.

સામાન્ય રીતે તેમાં mouse control નો સપોર્ટ નથી, પરંતુ મેં mouse support ઉમેરતી એક સુધારેલી આવૃત્તિ બનાવી છે. જો તમને mouse support જોઈએ, તો કૃપા કરીને તેને ઇન્સ્ટોલ કરો: <https://github.com/stefansundin/wtype>

જો તમને `Compositor does not support the virtual keyboard protocol` જેવી ભૂલ મળે, તો હું સૂચવું છું કે તમે `ydotool` જેવા બીજા tool નો પ્રયાસ કરો.

કૃપા કરીને `wtype` અંગે આ ચર્ચા થ્રેડમાં ચર્ચા કરો: <https://github.com/stefansundin/SSHRemote/discussions/4>

### dotool

`dotool` પણ `ydotool` જેવી રીતે કોઈપણ window manager સાથે કામ કરવું જોઈએ. મારા મર્યાદિત પરીક્ષણમાં તે `ydotool` કરતાં ઘણું ધીમું હતું.

કૃપા કરીને `dotool` અંગે આ ચર્ચા થ્રેડમાં ચર્ચા કરો: <https://github.com/stefansundin/SSHRemote/discussions/6>

### cec-client

આ પ્રિસેટ પ્રયોગાત્મક છે કારણ કે હું તેને મારા પોતાના hardware પર ચકાસી શક્યો નથી. પ્રતિસાદનું સ્વાગત છે.

કૃપા કરીને `cec-client` અંગે આ ચર્ચા થ્રેડમાં ચર્ચા કરો: <https://github.com/stefansundin/SSHRemote/discussions/7>

### macOS

macOS પર VLC ને નિયંત્રિત કરવા માટે એક પ્રિસેટ છે, જેમાં AppleScript commands વપરાય છે. કીબોર્ડ અથવા mouse events મોકલવાનું સમર્થન આપતું કોઈ tool મને જાણીતું નથી.

કૃપા કરીને macOS અંગે આ ચર્ચા થ્રેડમાં ચર્ચા કરો: <https://github.com/stefansundin/SSHRemote/discussions/8>

### Android

જો તમારા Android ડિવાઇસમાં SSH server હોય, તો તમે શક્ય છે કે તેમાં જોડાઈ શકો અને input events મોકલી શકો. જો તમારું ડિવાઇસ Raspberry Pi પર KonstaKANG જેવી custom ROM ચલાવે છે, તો તેની શક્યતા વધારે છે.

mouse support કેવી રીતે કાર્યરત કરવું તે હું હજી સમજી શક્યો નથી.

કૃપા કરીને Android અંગે આ ચર્ચા થ્રેડમાં ચર્ચા કરો: <https://github.com/stefansundin/SSHRemote/discussions/9>

## સ્માર્ટ વોલ્યુમ નિયંત્રણ

જ્યારે તમે રિમોટ કંટ્રોલ સંપાદિત કરો છો, ત્યારે menu માં તમને “smart” volume control settings મળશે. આ એપમાં કમ્પ્યુટરનો હાલનો વોલ્યુમ બતાવી શકે છે અને slider વડે ઝડપથી વોલ્યુમ સેટ કરવાની સુવિધા આપે છે. તમે તમારા ડિવાઇસના hardware buttons નો ઉપયોગ કરીને ઝડપથી volume up/down commands મોકલી શકો છો.

હાલમાં વર્તમાન વોલ્યુમ વાંચવું અને slider વડે નવો વોલ્યુમ સેટ કરવું `pactl` નો ઉપયોગ કરવા માટે hard-coded છે.

`pactl` ધરાવતો package સામાન્ય રીતે `pulseaudio-utils` અથવા `libpulse` કહેવાય છે.

## SSH કીઓ

તમે એપ સેટિંગ્સમાં SSH કીઓ આયાત અથવા બનાવી શકો છો. SSH કી સાથે જોડાવું પાસવર્ડ વાપરવા કરતાં વધુ સુરક્ષિત છે.

કમ્પ્યુટરમાંથી હાલની SSH કી આયાત કરવાની સૌથી સરળ રીત QR કોડ સ્કેન કરવી છે. તમે QR કોડ image બનાવવા માટે `qrencode` પ્રોગ્રામનો ઉપયોગ કરી શકો છો. QR કોડ જનરેટ કરવા માટે નીચે જેવી command ચલાવો:

```shell
# તમારી SSH કીઓ સુધી જાઓ:
cd ~/.ssh

# QR કોડ ટર્મિનલમાં બતાવો:
qrencode -r id_ed25519 -t ansiutf8

# વૈકલ્પિક રીતે, image file બનાવો:
qrencode -r id_ed25519 -o qr.png

# 4096-bit RSA કીઓ QR કોડ માટે બહુ મોટી થાય છે. તમે gzip નો ઉપયોગ કરીને એકને કઠિનતાથી ફિટ કરી શકો છો:
cat id_rsa_4096 | gzip | base64 | qrencode -o qr.png
```

menu માં આવેલા `જાહેર કી મોકલો` feature નો ઉપયોગ કરીને તમે public SSH keys ને server પર મોકલી શકો છો. આ પસંદ કરેલી SSH કીને `~/.ssh/authorized_keys` ફાઇલમાં append કરશે. આ રીતે તમે પાસવર્ડથી લૉગિન કરવાથી SSH કીથી લૉગિન કરવા સરળતાથી સ્થળાંતર કરી શકો છો.

તમે encrypted SSH keys આયાત કરી અને વાપરી શકો છો, પરંતુ હાલમાં એપમાં આવી કીઓ બનાવવી શક્ય નથી.

## સુરક્ષા

એપમાંથી SSH કીઓનો private ભાગ અથવા સાચવેલા passwords નિકાસ કરવો કે બહાર કાઢવો શક્ય નથી. આ ડેટા 256-bit AES વડે encrypted છે, અને encryption key Android Keystore માં સંગ્રહિત છે. encrypted data ને Android backups માંથી બહાર રાખવામાં આવે છે.

આ એપમાં કોઈ crash reporting software નથી. કોઈ telemetry નથી. કોઈ ads નથી. SSH connection સિવાય કોઈ network requests નથી.

આ એપની સુરક્ષાનું audit થયું નથી. જો તમને Android security અથવા SSH security નો અનુભવ હોય, તો કૃપા કરીને source code જુઓ અને તમારા નિષ્કર્ષો આ GitHub issue માં જણાવો:

<https://github.com/stefansundin/SSHRemote/issues/1>

## ફીચર વિનંતીઓ

GitHub repository માં feature requests અને bug reports મોકલવા માટે નિરાંકે આવો. કૃપા કરીને English નો ઉપયોગ કરો. તમારી ટિપ્પણીઓ સૌજન્યપૂર્ણ રાખો. અશિષ્ટ ટિપ્પણીઓ દૂર કરવામાં આવશે અને વપરાશકર્તાઓને repository માંથી block પણ કરી શકાય છે.

તમારો પ્રશ્ન પહેલેથી પૂછાયો છે કે તેનો જવાબ આપવામાં આવ્યો છે કે નહીં તે જોવા માટે હાલની issues અને discussion threads જરૂર તપાસો.

કૃપા કરીને સન્માનપૂર્ણ રહો. મેં આ એપ મારા ફુરસદના સમયમાં બનાવી છે અને હું તેને મફતમાં આપી રહ્યો છું. હું આ એપ સૌથી પહેલા મારા પોતાના ઉપયોગ માટે બનાવી રહ્યો છું.

કૃપા કરીને મને ઇમેઇલ દ્વારા પ્રશ્નો ન મોકલો. શક્ય હોય ત્યાં સુધી ચર્ચાઓ GitHub પર જ રાખવાનો પ્રયત્ન કરો, કારણ કે એથી બીજા લોકોને પણ મદદ મળે છે! તમે GitHub પર discussion section માં પ્રશ્નો પૂછી શકો છો.

તમને હંમેશા એપને fork કરીને તમારી પોતાની સુવિધાઓ અમલમાં મૂકવા માટે આવકાર છે. શીખવા માટે આ ઉત્તમ રીત છે. ઉપયોગી સુવિધાઓનું યોગદાન આપવાનો વિચાર કરો.

સોર્સ કોડ GNU GPLv3 હેઠળ લાઇસન્સ્ડ છે. જો તમે આ એપની ફેરફાર કરેલી આવૃત્તિઓ વિતરિત કરો, તો તમારે સોર્સ કોડ પણ ઉપલબ્ધ કરાવવો પડશે.

<https://github.com/stefansundin/SSHRemote>
