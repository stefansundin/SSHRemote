## SSH Remoteയെ കുറിച്ച്

ഈ വിവർത്തനം GitHub Copilot AI സഹായത്തോടെ തയ്യാറാക്കിയതാണ്, അതിനാൽ വിവർത്തന പിശകുകൾ ഉണ്ടായേക്കാം.

SSH ഉപയോഗിച്ച് കമ്പ്യൂട്ടറുകളെ ദൂരസ്ഥമായി നിയന്ത്രിക്കാൻ നിങ്ങളെ അനുവദിക്കുന്ന ഒരു സൗജന്യവും ഓപ്പൺ സോഴ്‌സുമായ ആപ്പാണ് SSH Remote.

നടപ്പിലാക്കുന്ന കമാൻഡുകൾ നിങ്ങൾക്ക് പൂർണ്ണമായി ഇഷ്ടാനുസൃതമാക്കാം, കൂടാതെ സാധാരണ ക്രമീകരണങ്ങൾക്കായി പ്രീസെറ്റുകളും ലഭ്യമാണ്.

Raspberry Pi OS പ്രവർത്തിക്കുന്ന എന്റെ HTPC ക്രമീകരണം നിയന്ത്രിക്കാൻ ഞാൻ ഈ ആപ്പ് ഉപയോഗിക്കുന്നു. HTPC നിയന്ത്രിക്കലാണ് ഈ ആപ്പ് പ്രത്യേകിച്ച് ഒപ്റ്റിമൈസ് ചെയ്തിരിക്കുന്ന അടിസ്ഥാന സാഹചര്യം.

ഈ ആപ്പ് ഒരു ടെർമിനൽ എമുലേറ്റർ അല്ല, പക്ഷേ അടിയന്തര സാഹചര്യത്തിൽ `apt-get install` പ്രവർത്തിപ്പിക്കാൻ ഇത് നിങ്ങളെ അനുവദിക്കും.

## ആരംഭിക്കുന്നത്

കണക്റ്റ് ചെയ്യാൻ SSH കീ ഉപയോഗിക്കണമെങ്കിൽ, ആദ്യം ആപ്പിന്റെ ക്രമീകരണങ്ങൾ തുറന്ന് ഒരു കീ ഇറക്കുമതി ചെയ്യുകയോ സൃഷ്ടിക്കുകയോ ചെയ്യുക.

താഴെ വലത് വശത്തുള്ള `+` ബട്ടൺ തട്ടി ഒരു പുതിയ ഹോസ്റ്റ് ചേർക്കുക. കണക്ഷൻ വിവരങ്ങൾ നൽകുകയും സംരക്ഷിക്കുകയും ചെയ്യുക.

നിങ്ങൾ ആദ്യമായി കണക്റ്റ് ചെയ്യുമ്പോൾ, ഒരു പ്രീസെറ്റ് തിരഞ്ഞെടുക്കാൻ നിങ്ങളോട് ആവശ്യപ്പെടും. ഈ തിരഞ്ഞെടുപ്പ് വിവിധ തരത്തിലുള്ള കമ്പ്യൂട്ടറുകളിൽ റിമോട്ട് നിയന്ത്രണ ബട്ടണുകൾ നന്നായി പ്രവർത്തിക്കുന്ന തരത്തിൽ ക്രമീകരിക്കുന്നു. ലഭ്യമായ പ്രീസെറ്റുകളുടെ വിവരണത്തിന് താഴെ നോക്കുക. നിങ്ങൾക്ക് ഇഷ്ടമെങ്കിൽ, `No preset` തിരഞ്ഞെടുക്കുന്നതിലൂടെ യാതൊരു ക്രമീകരണവും ഇല്ലാതെ തുടങ്ങാം.

നിങ്ങളുടെ Linux കമ്പ്യൂട്ടർ X11 ആണോ Wayland ആണോ പ്രവർത്തിപ്പിക്കുന്നത് എന്ന് അറിയില്ലെങ്കിൽ, ടെർമിനലിൽ ഇത് പ്രവർത്തിപ്പിക്കുക:

```shell
echo $XDG_SESSION_TYPE
```

ഇത് `x11` അല്ലെങ്കിൽ `wayland` എന്നായി ഔട്ട്‌പുട്ട് നൽകണം. നിങ്ങൾ ഇത് ഡെസ്ക്ടോപ്പ് പരിസരത്തിനുള്ളിൽ തന്നെ പ്രവർത്തിപ്പിക്കണം.

## റിമോട്ട് നിയന്ത്രണം

കണക്റ്റ് ചെയ്ത ശേഷം, കമാൻഡുകൾ അയയ്ക്കാൻ റിമോട്ട് നിയന്ത്രണ ഇന്റർഫേസ് ഉപയോഗിക്കാം. വിവിധ ഇൻപുട്ട് രീതികൾ ആക്‌സസ് ചെയ്യാൻ ടാബുകൾ മാറ്റുക.

ഓരോ ബട്ടൺ അമർത്തലും ഹോസ്റ്റിൽ ഒരു കമാൻഡ് നടപ്പിലാക്കും. ഒരു കീ അമർത്തൽ പോലെ ലളിതമായ കാര്യത്തിനുപോലും ഇത് വളരെ അധികം ഓവർഹെഡാണ്, അതിനാൽ സാധാരണ കീബോർഡുമായി താരതമ്യം ചെയ്യുമ്പോൾ നിങ്ങൾക്ക് ഗണ്യമായ ലേറ്റൻസി അനുഭവപ്പെടാം. ഭാവി പതിപ്പുകളിൽ ഇത് മെച്ചപ്പെടുത്താൻ ഞാൻ പ്രതീക്ഷിക്കുന്നു.

എഡിറ്റ് മോഡിൽ പ്രവേശിക്കാൻ മെനു ഉപയോഗിക്കുക. നിലവിൽ ലേഔട്ട് അല്ലെങ്കിൽ ബട്ടൺ ഐക്കണുകൾ തിരുത്താൻ കഴിയില്ല. ഭാവി പതിപ്പിൽ ഇത് സാധ്യമാക്കാൻ ഞാൻ പ്രതീക്ഷിക്കുന്നു.

## പ്രീസെറ്റുകൾ

നിങ്ങളുടെ ഡെസ്ക്ടോപ്പ് പരിസരത്തിന് ആവശ്യമായ ടൂൾ ഇൻസ്റ്റാൾ ചെയ്യണം.

എന്റെ പരീക്ഷണത്തിൽ ഏറ്റവും നല്ല പ്രകടനം നൽകിയതും X11-ലും Wayland-ലും പ്രവർത്തിക്കുന്നതുമായതിനാൽ ഞാൻ `ydotool` ശുപാർശ ചെയ്യുന്നു.

### ydotool

`ydotool` ഏത് വിൻഡോ മാനേജറിലും പ്രവർത്തിക്കേണ്ടതാണ്, പക്ഷേ പശ്ചാത്തലത്തിൽ ഒരു സേവനം പ്രവർത്തിക്കണം. നിങ്ങളുടെ ഡിസ്ട്രിബ്യൂഷൻ systemd യൂസർ സർവീസ് നൽകുന്നുവെങ്കിൽ, ഇത് പ്രവർത്തിപ്പിച്ച് അത് ആരംഭിക്കുക:

```shell
systemctl start --user ydotool
```

ലോഗിൻ ചെയ്യുമ്പോൾ സർവീസ് സ്വയം ആരംഭിക്കാൻ ഇത് പ്രവർത്തിപ്പിക്കുക:

```shell
systemctl enable --user ydotool
```

നിങ്ങൾ മതിയായ പുതിയ പതിപ്പിലുള്ള `ydotool` ഇൻസ്റ്റാൾ ചെയ്യുന്നുണ്ടെന്ന് ഉറപ്പാക്കുക. 26.04-നു മുമ്പുള്ള Ubuntu പതിപ്പുകളിൽ ലഭ്യമാകുന്ന പതിപ്പുകൾ വളരെ പഴയതാണ്. പരിഹാര മാർഗത്തിനായി discussion thread കാണുക.

`ydotool`-നെ കുറിച്ചുള്ള ചർച്ചകൾ ഈ discussion thread-ൽ നടത്തുക: <https://github.com/stefansundin/SSHRemote/discussions/5>

### xdotool

X11 പ്രവർത്തിക്കുന്ന കമ്പ്യൂട്ടറുകൾക്കായുള്ളതാണ് `xdotool`. ചരിത്രപരമായി മിക്ക Linux കമ്പ്യൂട്ടറുകളും X11 ആണ് ഉപയോഗിച്ചിരുന്നത്, എങ്കിലും Wayland കൂടുതൽ ജനപ്രിയമാകുന്നു.

X11-നോടൊപ്പം ഒരു പ്രത്യേക പ്രശ്നമുണ്ട്: X server-ലേക്കുള്ള ആക്‌സസ് അനുവദിക്കേണ്ടി വരാം. നിങ്ങൾക്ക് "Authorization required" പിശകുകൾ ലഭിക്കുന്നുണ്ടെങ്കിൽ പ്രശ്നം ഇതാണ്. ഇത് പരിഹരിക്കാൻ ചില ഓപ്ഷനുകളുണ്ട്; എനിക്ക് പ്രവർത്തിച്ച രണ്ടെണ്ണം ഇതാ:

`xauth list` യാതൊരു എൻട്രിയും കാണിക്കാത്ത പക്ഷം, ഒരു `.Xauthority` ഫയൽ സൃഷ്ടിക്കാൻ ശ്രമിക്കുക:

```shell
xauth generate :0 . trusted
```

അതും പ്രവർത്തിക്കാത്ത പക്ഷം, `xhost` ഉപയോഗിച്ച് ആക്‌സസ് അനുവദിക്കാൻ ശ്രമിക്കുക:

```shell
xhost +local:$USER
```

ഓരോ ബൂട്ട് കഴിഞ്ഞും നിങ്ങൾ `xhost` കമാൻഡ് പ്രവർത്തിപ്പിക്കേണ്ടിവരും. ഒരു bash സ്ക്രിപ്റ്റ് ഉണ്ടാക്കി ലോഗിനിൽ അത് സ്വയം ആരംഭിക്കുന്ന രീതിയിൽ ക്രമീകരിച്ച് ഇത് ഓട്ടോമേറ്റ് ചെയ്യാം.

`xdotool`-നെ കുറിച്ചുള്ള ചർച്ചകൾ ഈ discussion thread-ൽ നടത്തുക: <https://github.com/stefansundin/SSHRemote/discussions/3>

### wtype

`wtype` `xdotool` പോലെയാണ്, പക്ഷേ Wayland-നായി.

സാധാരണയായി ഇത് മൗസ് നിയന്ത്രണം പിന്തുണയ്ക്കുന്നില്ല, പക്ഷേ മൗസ് പിന്തുണ ചേർക്കുന്ന ഒരു പരിഷ്‌കരിച്ച പതിപ്പ് ഞാൻ സൃഷ്ടിച്ചിട്ടുണ്ട്. നിങ്ങൾക്ക് മൗസ് പിന്തുണ വേണമെങ്കിൽ അത് ഇൻസ്റ്റാൾ ചെയ്യുക: <https://github.com/stefansundin/wtype>

നിങ്ങൾക്ക് `Compositor does not support the virtual keyboard protocol` എന്ന പിശക് ലഭിക്കുന്നുണ്ടെങ്കിൽ, `ydotool` പോലുള്ള മറ്റൊരു ടൂൾ പരീക്ഷിക്കാൻ ഞാൻ നിർദേശിക്കുന്നു.

`wtype`-നെ കുറിച്ചുള്ള ചർച്ചകൾ ഈ discussion thread-ൽ നടത്തുക: <https://github.com/stefansundin/SSHRemote/discussions/4>

### dotool

`dotool` `ydotool` പോലെ ഏത് വിൻഡോ മാനേജറിലും പ്രവർത്തിക്കേണ്ടതാണ്. എന്റെ പരിമിതമായ പരീക്ഷണത്തിൽ ഇത് `ydotool`-നെക്കാൾ വളരെ മന്ദഗതിയിലായിരുന്നു.

`dotool`-നെ കുറിച്ചുള്ള ചർച്ചകൾ ഈ discussion thread-ൽ നടത്തുക: <https://github.com/stefansundin/SSHRemote/discussions/6>

### cec-client

ഈ പ്രീസെറ്റ് പരീക്ഷണാത്മകമാണ്, കാരണം എന്റെ സ്വന്തം ഹാർഡ്‌വെയറിൽ ഇത് പരിശോധിക്കാൻ എനിക്ക് കഴിഞ്ഞിട്ടില്ല. അഭിപ്രായങ്ങൾ സ്വാഗതം ചെയ്യുന്നു.

`cec-client`-നെ കുറിച്ചുള്ള ചർച്ചകൾ ഈ discussion thread-ൽ നടത്തുക: <https://github.com/stefansundin/SSHRemote/discussions/7>

### macOS

AppleScript കമാൻഡുകൾ ഉപയോഗിച്ച് macOS-ൽ VLC നിയന്ത്രിക്കാൻ ഒരു പ്രീസെറ്റ് ഉണ്ട്. കീബോർഡ് അല്ലെങ്കിൽ മൗസ് ഇവന്റുകൾ അയയ്ക്കുന്നത് പിന്തുണയ്ക്കുന്ന ഒരു ടൂളിനെക്കുറിച്ച് എനിക്ക് അറിവില്ല.

macOS-നെ കുറിച്ചുള്ള ചർച്ചകൾ ഈ discussion thread-ൽ നടത്തുക: <https://github.com/stefansundin/SSHRemote/discussions/8>

### Android

നിങ്ങളുടെ Android ഉപകരണത്തിൽ SSH സർവർ ഉണ്ടെങ്കിൽ, അതിലേക്ക് കണക്റ്റ് ചെയ്ത് ഇൻപുട്ട് ഇവന്റുകൾ അയയ്ക്കാൻ നിങ്ങൾക്ക് കഴിയാമെന്നു തോന്നുന്നു. KonstaKANG on the Raspberry Pi പോലുള്ള ഒരു custom ROM ആണ് ഉപകരണത്തിൽ പ്രവർത്തിക്കുന്നതെങ്കിൽ ഇത് കൂടുതൽ സാധ്യതയുള്ളതാണ്.

മൗസ് പിന്തുണ പ്രവർത്തിപ്പിക്കാൻ എനിക്ക് ഇതുവരെ മാർഗം കണ്ടെത്താനായിട്ടില്ല.

Android-നെ കുറിച്ചുള്ള ചർച്ചകൾ ഈ discussion thread-ൽ നടത്തുക: <https://github.com/stefansundin/SSHRemote/discussions/9>

## സ്മാർട്ട് വോള്യം നിയന്ത്രണം

റിമോട്ട് നിയന്ത്രണം എഡിറ്റ് ചെയ്യുമ്പോൾ, മെനുവിൽ "smart" volume control ക്രമീകരണങ്ങൾ കണ്ടെത്താം. ഇത് ആപ്പിൽ കമ്പ്യൂട്ടറിന്റെ നിലവിലെ വോള്യം പ്രദർശിപ്പിക്കാനും ഒരു സ്ലൈഡർ ഉപയോഗിച്ച് വേഗത്തിൽ വോള്യം ക്രമീകരിക്കാനും അനുവദിക്കുന്നു. നിങ്ങളുടെ ഉപകരണത്തിലെ ഹാർഡ്‌വെയർ ബട്ടണുകൾ ഉപയോഗിച്ച് volume up/down കമാൻഡുകളും വേഗത്തിൽ അയയ്ക്കാം.

നിലവിലെ വോള്യം വായിക്കലും സ്ലൈഡർ ഉപയോഗിച്ച് പുതിയ വോള്യം സജ്ജമാക്കലും ഇപ്പോൾ `pactl` ഉപയോഗിക്കാൻ hard-coded ചെയ്തിരിക്കുന്നു.

`pactl` ഉൾപ്പെടുന്ന പാക്കേജ് സാധാരണയായി `pulseaudio-utils` അല്ലെങ്കിൽ `libpulse` എന്നാണ് വിളിക്കപ്പെടുന്നത്.

## SSH കീകൾ

നിങ്ങൾക്ക് ആപ്പിന്റെ ക്രമീകരണങ്ങളിൽ SSH കീകൾ ഇറക്കുമതി ചെയ്യുകയോ സൃഷ്ടിക്കുകയോ ചെയ്യാം. പാസ്‌വേഡുകൾ ഉപയോഗിക്കുന്നതിനെക്കാൾ SSH കീ ഉപയോഗിച്ച് കണക്റ്റ് ചെയ്യുന്നത് കൂടുതൽ സുരക്ഷിതമാണ്.

ഒരു കമ്പ്യൂട്ടറിൽ നിന്ന് നിലവിലുള്ള SSH കീ ഇറക്കുമതി ചെയ്യാനുള്ള ഏറ്റവും എളുപ്പമുള്ള മാർഗം QR കോഡ് സ്കാൻ ചെയ്യുന്നതാണ്. QR കോഡ് ചിത്രം സൃഷ്ടിക്കാൻ നിങ്ങൾക്ക് `qrencode` പ്രോഗ്രാം ഉപയോഗിക്കാം. QR കോഡ് സൃഷ്ടിക്കാൻ താഴെ കാണുന്നതുപോലുള്ള ഒരു കമാൻഡ് പ്രവർത്തിപ്പിക്കുക:

```shell
# നിങ്ങളുടെ SSH കീകളിലേക്ക് നാവിഗേറ്റ് ചെയ്യുക:
cd ~/.ssh

# ടെർമിനലിൽ QR കോഡ് പ്രദർശിപ്പിക്കുക:
qrencode -r id_ed25519 -t ansiutf8

# അല്ലെങ്കിൽ, ഒരു ഇമേജ് ഫയൽ സൃഷ്ടിക്കുക:
qrencode -r id_ed25519 -o qr.png

# 4096-bit RSA കീകൾ QR കോഡിനായി വളരെ വലുതാണ്. ഒരൊന്ന് കൂടി ചേർക്കാൻ gzip ഉപയോഗിക്കാം:
cat id_rsa_4096 | gzip | base64 | qrencode -o qr.png
```

മെനുവിലെ `പബ്ലിക് കീ അയയ്ക്കുക` സൗകര്യം ഉപയോഗിച്ച് നിങ്ങൾക്ക് ഒരു സർവറിലേക്ക് public SSH കീകൾ അയക്കാം. ഇത് തിരഞ്ഞെടുക്കുന്ന SSH കീ `~/.ssh/authorized_keys` ഫയലിന്റെ അവസാനം ചേർക്കും. പാസ്‌വേഡ് ഉപയോഗിച്ചുള്ള ലോഗിനിൽ നിന്ന് SSH കീ ഉപയോഗിച്ചുള്ള ലോഗിനിലേക്ക് എളുപ്പത്തിൽ മാറാൻ ഇത് സഹായിക്കുന്നു.

നിങ്ങൾക്ക് എൻക്രിപ്റ്റ് ചെയ്ത SSH കീകൾ ഇറക്കുമതി ചെയ്യുകയും ഉപയോഗിക്കുകയും ചെയ്യാം, പക്ഷേ നിലവിൽ ആപ്പിനുള്ളിൽ അവ സൃഷ്ടിക്കാൻ കഴിയില്ല.

## സുരക്ഷ

ആപ്പിൽ നിന്ന് SSH കീകളുടെ സ്വകാര്യ ഭാഗമോ സംഭരിച്ച പാസ്‌വേഡുകളോ എക്സ്പോർട്ട് ചെയ്യുകയോ പുറത്തെടുക്കുകയോ ചെയ്യാൻ കഴിയില്ല. ഈ ഡാറ്റ 256-bit AES ഉപയോഗിച്ച് എൻക്രിപ്റ്റ് ചെയ്തതാണ്, കൂടാതെ എൻക്രിപ്ഷൻ കീ Android Keystore-ൽ സംഭരിച്ചിരിക്കുന്നു. എൻക്രിപ്റ്റ് ചെയ്ത ഡാറ്റ Android ബാക്ക്‌അപ്പുകളിൽ നിന്ന് ഒഴിവാക്കപ്പെടുന്നു.

ഈ ആപ്പിൽ crash reporting software ഇല്ല. ടെലിമെട്രി ഇല്ല. പരസ്യങ്ങളില്ല. SSH കണക്ഷൻ ഒഴികെ നെറ്റ്‌വർക്ക് അഭ്യർത്ഥനകളൊന്നുമില്ല.

ഈ ആപ്പിന്റെ സുരക്ഷ ഓഡിറ്റ് ചെയ്തിട്ടില്ല. Android സുരക്ഷയിലോ SSH സുരക്ഷയിലോ നിങ്ങൾക്ക് പരിചയം ഉണ്ടെങ്കിൽ, ദയവായി സോഴ്‌സ് കോഡ് പരിശോധിച്ച് നിങ്ങളുടെ കണ്ടെത്തലുകൾ ഈ GitHub issue-ൽ റിപ്പോർട്ട് ചെയ്യുക:

<https://github.com/stefansundin/SSHRemote/issues/1>

## ഫീച്ചർ അഭ്യർത്ഥനകൾ

GitHub repository-ൽ ഫീച്ചർ അഭ്യർത്ഥനകളും ബഗ് റിപ്പോർട്ടുകളും സമർപ്പിക്കാൻ മടിക്കേണ്ട. ദയവായി ഇംഗ്ലീഷ് ഉപയോഗിക്കുക. നിങ്ങളുടെ അഭിപ്രായങ്ങൾ മാന്യമായി സൂക്ഷിക്കുക. അനാദരവുള്ള അഭിപ്രായങ്ങൾ നീക്കം ചെയ്യപ്പെടുകയും ഉപയോക്താക്കളെ repository-യിൽ നിന്ന് ബ്ലോക്ക് ചെയ്യുകയും ചെയ്യാം.

നിങ്ങളുടെ ചോദ്യം ഇതിനകം ചോദിച്ചതോ മറുപടി നൽകിയതോ ആണോ എന്ന് അറിയാൻ നിലവിലുള്ള issues-ഉം discussion threads-ഉം പരിശോധിക്കുക.

ദയവായി മാന്യത പുലർത്തുക. ഞാൻ ഈ ആപ്പ് എന്റെ ഒഴിവുസമയത്ത് നിർമിച്ചതും ഇത് സൗജന്യമായി നിങ്ങൾക്ക് നൽകുന്നതുമാണ്. ആദ്യമായും പ്രധാനമായും, ഈ ആപ്പ് ഞാൻ എന്റെ സ്വന്തം ഉപയോഗത്തിനായാണ് നിർമ്മിക്കുന്നത്.

ദയവായി ചോദ്യങ്ങൾ ഇമെയിൽ ചെയ്യരുത്. സംഭാഷണങ്ങൾ GitHub-ലുതന്നെ തുടരാൻ ശ്രമിക്കുക, കാരണം അത് മറ്റ് ആളുകൾക്കും സഹായകരമാണ്! GitHub-യിലെ discussion വിഭാഗത്തിൽ നിങ്ങൾക്ക് ചോദ്യങ്ങൾ ചോദിക്കാം.

നിങ്ങളുടെ സ്വന്തം ഫീച്ചറുകൾ നടപ്പാക്കാൻ ആപ്പ് fork ചെയ്യാൻ നിങ്ങൾക്ക് എപ്പോഴും സ്വാഗതം. പഠിക്കാൻ അതൊരു മികച്ച മാർഗമാണ്. ഉപകാരപ്രദമായ ഫീച്ചറുകൾ സംഭാവന ചെയ്യുന്നതും പരിഗണിക്കുക.
