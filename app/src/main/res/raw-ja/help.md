## SSH Remote について

この翻訳は GitHub Copilot (GPT-5.3-Codex) によって作成されており、翻訳に誤りが含まれる可能性があります。

SSH Remote は、SSH を使用してコンピューターをリモート操作できる、無料のオープンソースアプリです。

実行するコマンドは完全にカスタマイズでき、一般的な構成向けのプリセットも用意されています。

私は Raspberry Pi OS を実行している HTPC 環境を操作するためにこのアプリを使っています。HTPC の操作は、このアプリが主に最適化している基本的な利用シナリオです。

このアプリはターミナルエミュレーターではありませんが、緊急時には `apt-get install` を実行できます。

## はじめに

SSH 鍵を使って接続したい場合は、まずアプリの設定を開いて鍵をインポートするか生成してください。

右下の `+` ボタンをタップして新しいホストを追加します。接続情報を入力して保存してください。

初めて接続すると、プリセットを選択するよう求められます。この選択により、さまざまな種類のコンピューターでリモートコントロールのボタンがうまく動作するように設定されます。利用可能なプリセットの説明は以下を参照してください。必要であれば、`No preset` を選んで設定なしで始めることもできます。

Linux コンピューターが X11 と Wayland のどちらで動いているかわからない場合は、ターミナルで次を実行してください。

```shell
echo $XDG_SESSION_TYPE
```

`x11` または `wayland` と表示されるはずです。これはデスクトップ環境内で実行する必要があります。

## リモートコントロール

接続後は、リモートコントロール画面を使ってコマンドを送信できます。タブを切り替えることで、さまざまな入力方法にアクセスできます。

各ボタン操作はホスト上でコマンドを実行します。単純なキー入力に対してもかなり大きなオーバーヘッドがあるため、通常のキーボードと比べると遅延がかなり大きく感じられることがあります。今後のバージョンで改善したいと考えています。

メニューから編集モードに入ってください。現在のところ、レイアウトやボタンアイコンを編集することはできません。将来のバージョンで可能にしたいと思っています。

## プリセット

デスクトップ環境に必要なツールをインストールする必要があります。

私のおすすめは `ydotool` です。テストした限りでは最もパフォーマンスが良く、X11 と Wayland の両方で動作します。

### ydotool

`ydotool` はどのウィンドウマネージャーでも動作するはずですが、バックグラウンドサービスを実行する必要があります。ディストリビューションが systemd のユーザーサービスを提供している場合は、次を実行して開始してください。

```shell
systemctl start --user ydotool
```

ログイン時に自動でサービスを開始するには、次を実行してください。

```shell
systemctl enable --user ydotool
```

十分に新しいバージョンの `ydotool` をインストールしていることを確認してください。Ubuntu 26.04 より前のバージョンに含まれるものは古すぎます。回避策についてはディスカッションスレッドを参照してください。

`ydotool` に関する話題は、このディスカッションスレッドでお願いします: <https://github.com/stefansundin/SSHRemote/discussions/5>

### xdotool

`xdotool` は X11 で動作しているコンピューター向けです。X11 は歴史的に多くの Linux コンピューターで使われてきましたが、最近は Wayland がより普及しつつあります。

X11 には少し癖があり、X サーバーへのアクセスを許可する必要がある場合があります。`Authorization required` エラーが出る場合は、これが原因です。解決方法はいくつかありますが、私の環境で有効だった 2 つの方法を紹介します。

`xauth list` を実行しても何も表示されない場合は、`.Xauthority` ファイルの生成を試してください。

```shell
xauth generate :0 . trusted
```

それでもだめなら、`xhost` を使ってアクセスを許可してみてください。

```shell
xhost +local:$USER
```

`xhost` コマンドは起動のたびに実行する必要があります。bash スクリプトを作成し、ログイン時に自動起動するよう設定すると自動化できます。

`xdotool` に関する話題は、このディスカッションスレッドでお願いします: <https://github.com/stefansundin/SSHRemote/discussions/3>

### wtype

`wtype` は Wayland 向けの `xdotool` のようなものです。

通常はマウス操作をサポートしていませんが、私はマウス対応を追加した改造版を作成しました。マウス操作が必要な場合は、そちらをインストールしてください: <https://github.com/stefansundin/wtype>

`Compositor does not support the virtual keyboard protocol` というエラーが出る場合は、`ydotool` など別のツールを試すことをおすすめします。

`wtype` に関する話題は、このディスカッションスレッドでお願いします: <https://github.com/stefansundin/SSHRemote/discussions/4>

### dotool

`dotool` は `ydotool` と同様に、どのウィンドウマネージャーでも動作するはずです。限られたテストでは、`ydotool` よりかなり遅く感じました。

`dotool` に関する話題は、このディスカッションスレッドでお願いします: <https://github.com/stefansundin/SSHRemote/discussions/6>

### cec-client

このプリセットは、私自身のハードウェアで確認できていないため実験的です。フィードバックを歓迎します。

`cec-client` に関する話題は、このディスカッションスレッドでお願いします: <https://github.com/stefansundin/SSHRemote/discussions/7>

### macOS

macOS で VLC を操作するためのプリセットがあり、AppleScript コマンドを使用します。キーボードやマウスイベントの送信をサポートするツールは、私は把握していません。

macOS に関する話題は、このディスカッションスレッドでお願いします: <https://github.com/stefansundin/SSHRemote/discussions/8>

### Android

Android デバイスに SSH サーバーが搭載されている場合、接続して入力イベントを送信できる可能性があります。Raspberry Pi 上の KonstaKANG のようなカスタム ROM を使用している場合は、その可能性がより高いです。

マウス対応をどのように実現するかは、まだわかっていません。

Android に関する話題は、このディスカッションスレッドでお願いします: <https://github.com/stefansundin/SSHRemote/discussions/9>

## スマート音量制御

リモートコントロールの編集中、メニュー内に「スマート」音量制御の設定があります。これにより、アプリ内にコンピューターの現在の音量を表示したり、スライダーを使ってすばやく音量を設定したりできます。また、端末のハードウェアボタンを使って、音量アップ/ダウンのコマンドをすばやく送信することもできます。

現在の音量の読み取りと、スライダーによる新しい音量の設定は、現時点では `pactl` を使うように固定されています。

`pactl` を含むパッケージ名は、通常 `pulseaudio-utils` または `libpulse` です。

## SSH 鍵

アプリの設定で SSH 鍵をインポートまたは生成できます。SSH 鍵で接続する方が、パスワードを使うより安全です。

コンピューターにある既存の SSH 鍵をインポートする最も簡単な方法は、QR コードをスキャンすることです。`qrencode` プログラムを使って QR コード画像を生成できます。次のようなコマンドを実行して QR コードを生成してください。

```shell
# SSH 鍵のある場所へ移動:
cd ~/.ssh

# ターミナルに QR コードを表示:
qrencode -r id_ed25519 -t ansiutf8

# または画像ファイルを作成:
qrencode -r id_ed25519 -o qr.png

# 4096 ビット RSA 鍵は QR コードには大きすぎます。gzip を使えばぎりぎり収まる場合があります:
cat id_rsa_4096 | gzip | base64 | qrencode -o qr.png
```

メニューの `公開鍵を送信` 機能を使うと、公開 SSH 鍵をサーバーに送信できます。これにより、選択した SSH 鍵が `~/.ssh/authorized_keys` ファイルに追記されます。パスワードログインから SSH 鍵ログインへ簡単に移行できます。

暗号化された SSH 鍵をインポートして使用することはできますが、現時点ではアプリ内でそれらを生成することはできません。

## セキュリティ

アプリから SSH 鍵の秘密部分や保存されたパスワードをエクスポートまたは抽出することはできません。これらのデータは 256 ビット AES で暗号化され、暗号鍵は Android Keystore に保存されます。暗号化されたデータは Android のバックアップ対象から除外されます。

このアプリにはクラッシュレポート機能はありません。テレメトリーもありません。広告もありません。ネットワークリクエストは SSH 接続以外には発生しません。

このアプリのセキュリティは監査を受けていません。Android セキュリティや SSH セキュリティに詳しい方は、ソースコードを確認し、この GitHub issue で所見を報告してください。

<https://github.com/stefansundin/SSHRemote/issues/1>
