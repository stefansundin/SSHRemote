## Σχετικά με το SSH Remote

Αυτή η μετάφραση δημιουργήθηκε με τη βοήθεια του GitHub Copilot (GPT-5.3-Codex) και ενδέχεται να περιέχει μεταφραστικά λάθη.

Το SSH Remote είναι μια δωρεάν εφαρμογή ανοιχτού κώδικα που σας επιτρέπει να ελέγχετε υπολογιστές απομακρυσμένα μέσω SSH.

Μπορείτε να προσαρμόσετε πλήρως τις εντολές που εκτελούνται και υπάρχουν προεπιλογές για συνηθισμένες ρυθμίσεις.

Χρησιμοποιώ αυτή την εφαρμογή για να ελέγχω τη ρύθμιση HTPC μου, που εκτελεί Raspberry Pi OS. Ο έλεγχος ενός HTPC είναι το βασικό σενάριο για το οποίο είναι βελτιστοποιημένη η εφαρμογή.

Αυτή η εφαρμογή δεν είναι εξομοιωτής τερματικού, αλλά θα σας επιτρέψει να εκτελέσετε `apt-get install` σε περίπτωση ανάγκης.

## Ξεκινώντας

Αν θέλετε να χρησιμοποιήσετε κλειδί SSH για σύνδεση, ανοίξτε πρώτα τις ρυθμίσεις της εφαρμογής και εισαγάγετε ή δημιουργήστε ένα κλειδί.

Προσθέστε έναν νέο κεντρικό υπολογιστή πατώντας το κουμπί `+` κάτω δεξιά. Εισαγάγετε τα στοιχεία σύνδεσης και αποθηκεύστε.

Την πρώτη φορά που θα συνδεθείτε, θα σας ζητηθεί να επιλέξετε μια προεπιλογή. Αυτή η επιλογή ρυθμίζει τα κουμπιά του τηλεχειριστηρίου ώστε να λειτουργούν καλά σε διάφορους τύπους υπολογιστών. Δείτε παρακάτω για περιγραφή των διαθέσιμων προεπιλογών. Αν προτιμάτε, μπορείτε να ξεκινήσετε χωρίς καμία ρύθμιση επιλέγοντας `Χωρίς προεπιλογή`.

Αν δεν γνωρίζετε αν ο υπολογιστής Linux σας εκτελεί X11 ή Wayland, εκτελέστε το παρακάτω σε ένα τερματικό:

```shell
echo $XDG_SESSION_TYPE
```

Αυτό θα πρέπει να εμφανίσει `x11` ή `wayland`. Πρέπει να το εκτελέσετε μέσα από το περιβάλλον επιφάνειας εργασίας.

## Τηλεχειριστήριο

Μόλις συνδεθείτε, μπορείτε να χρησιμοποιήσετε το περιβάλλον του τηλεχειριστηρίου για να στείλετε εντολές. Αλλάξτε καρτέλες για να αποκτήσετε πρόσβαση σε διάφορες μεθόδους εισόδου.

Κάθε πάτημα κουμπιού θα εκτελεί μια εντολή στον κεντρικό υπολογιστή. Αυτό έχει μεγάλο κόστος για κάτι τόσο απλό όσο ένα πάτημα πλήκτρου και ίσως παρατηρήσετε αρκετά μεγάλη καθυστέρηση σε σύγκριση με ένα κανονικό πληκτρολόγιο. Ελπίζω να το βελτιώσω σε μελλοντικές εκδόσεις.

Χρησιμοποιήστε το μενού για να μπείτε σε λειτουργία επεξεργασίας. Προς το παρόν δεν είναι δυνατή η επεξεργασία της διάταξης ή των εικονιδίων των κουμπιών. Ελπίζω να γίνει αυτό δυνατό σε μελλοντική έκδοση.

## Προεπιλογές

Θα χρειαστεί να εγκαταστήσετε το εργαλείο που απαιτείται για το περιβάλλον επιφάνειας εργασίας σας.

Συνιστώ το `ydotool` επειδή στις δοκιμές μου έχει την καλύτερη απόδοση και λειτουργεί τόσο σε X11 όσο και σε Wayland.

### ydotool

Το `ydotool` θα πρέπει να λειτουργεί με οποιονδήποτε διαχειριστή παραθύρων, αλλά χρειάζεται μια υπηρεσία παρασκηνίου σε λειτουργία. Αν η διανομή σας παρέχει υπηρεσία χρήστη systemd, ξεκινήστε την εκτελώντας:

```shell
systemctl start --user ydotool
```

Ενεργοποιήστε την αυτόματη εκκίνηση της υπηρεσίας κατά τη σύνδεση εκτελώντας:

```shell
systemctl enable --user ydotool
```

Παρακαλώ βεβαιωθείτε ότι εγκαθιστάτε αρκετά πρόσφατη έκδοση του `ydotool`. Οι εκδόσεις Ubuntu πριν από την 26.04 παρέχουν εκδόσεις που είναι υπερβολικά παλιές. Δείτε το νήμα συζήτησης για έναν εναλλακτικό τρόπο.

Παρακαλώ συζητήστε για το `ydotool` σε αυτό το νήμα συζήτησης: <https://github.com/stefansundin/SSHRemote/discussions/5>

### xdotool

Το `xdotool` είναι για υπολογιστές που εκτελούν X11. Το X11 είναι αυτό που χρησιμοποιούσαν ιστορικά οι περισσότεροι υπολογιστές Linux, αν και το Wayland γίνεται όλο και πιο δημοφιλές.

Μια ιδιομορφία του X11 είναι ότι ίσως χρειαστεί να επιτρέψετε πρόσβαση στον διακομιστή X. Αυτό συμβαίνει αν λαμβάνετε σφάλματα "Authorization required". Έχετε μερικές επιλογές για να διορθώσετε αυτό το πρόβλημα· εδώ είναι δύο επιλογές που έχουν λειτουργήσει για μένα:

Αν το `xauth list` δεν εμφανίζει καμία καταχώριση, δοκιμάστε να δημιουργήσετε ένα αρχείο `.Xauthority`:

```shell
xauth generate :0 . trusted
```

Αν αυτό δεν λειτούργησε, τότε δοκιμάστε να παραχωρήσετε πρόσβαση χρησιμοποιώντας το `xhost`:

```shell
xhost +local:$USER
```

Θα χρειαστεί να εκτελείτε την εντολή `xhost` μετά από κάθε εκκίνηση. Μπορείτε να το αυτοματοποιήσετε δημιουργώντας ένα σενάριο bash και ρυθμίζοντάς το να εκκινείται αυτόματα κατά τη σύνδεση.

Παρακαλώ συζητήστε για το `xdotool` σε αυτό το νήμα συζήτησης: <https://github.com/stefansundin/SSHRemote/discussions/3>

### wtype

Το `wtype` είναι σαν το `xdotool`, αλλά για Wayland.

Κανονικά δεν υποστηρίζει έλεγχο ποντικιού, αλλά έχω δημιουργήσει μια τροποποιημένη έκδοση που προσθέτει υποστήριξη ποντικιού. Παρακαλώ εγκαταστήστε την αν χρειάζεστε υποστήριξη ποντικιού: <https://github.com/stefansundin/wtype>

Αν λάβετε το σφάλμα `Compositor does not support the virtual keyboard protocol`, τότε προτείνω να δοκιμάσετε κάποιο άλλο εργαλείο, όπως το `ydotool`.

Παρακαλώ συζητήστε για το `wtype` σε αυτό το νήμα συζήτησης: <https://github.com/stefansundin/SSHRemote/discussions/4>

### dotool

Το `dotool` θα πρέπει να λειτουργεί με οποιονδήποτε διαχειριστή παραθύρων, παρόμοια με το `ydotool`. Στις περιορισμένες δοκιμές μου ήταν πολύ πιο αργό από το `ydotool`.

Παρακαλώ συζητήστε για το `dotool` σε αυτό το νήμα συζήτησης: <https://github.com/stefansundin/SSHRemote/discussions/6>

### cec-client

Αυτή η προεπιλογή είναι πειραματική, καθώς δεν έχω καταφέρει να την επαληθεύσω στο δικό μου υλικό. Τα σχόλια είναι ευπρόσδεκτα.

Παρακαλώ συζητήστε για το `cec-client` σε αυτό το νήμα συζήτησης: <https://github.com/stefansundin/SSHRemote/discussions/7>

### macOS

Υπάρχει μια προεπιλογή για τον έλεγχο του VLC στο macOS, χρησιμοποιώντας εντολές AppleScript. Δεν γνωρίζω κάποιο εργαλείο που να υποστηρίζει αποστολή συμβάντων πληκτρολογίου ή ποντικιού.

Παρακαλώ συζητήστε για το macOS σε αυτό το νήμα συζήτησης: <https://github.com/stefansundin/SSHRemote/discussions/8>

### Android

Αν η συσκευή Android σας διαθέτει διακομιστή SSH, ίσως μπορείτε να συνδεθείτε σε αυτή και να στείλετε συμβάντα εισόδου. Αυτό είναι πιο πιθανό αν η συσκευή σας εκτελεί μια προσαρμοσμένη ROM, όπως η KonstaKANG στο Raspberry Pi.

Δεν έχω καταφέρει να βρω πώς να λειτουργήσει η υποστήριξη ποντικιού.

Παρακαλώ συζητήστε για το Android σε αυτό το νήμα συζήτησης: <https://github.com/stefansundin/SSHRemote/discussions/9>

## Έξυπνος έλεγχος έντασης

Όταν επεξεργάζεστε το τηλεχειριστήριο, μπορείτε να βρείτε τις ρυθμίσεις «έξυπνου» ελέγχου έντασης στο μενού. Αυτό μπορεί να εμφανίζει την τρέχουσα ένταση του υπολογιστή στην εφαρμογή και να σας επιτρέπει να ορίζετε γρήγορα την ένταση χρησιμοποιώντας ένα ρυθμιστικό. Μπορείτε επίσης να χρησιμοποιείτε τα φυσικά κουμπιά της συσκευής σας για να στέλνετε γρήγορα εντολές αύξησης/μείωσης έντασης.

Η ανάγνωση της τρέχουσας έντασης και ο ορισμός νέας έντασης με το ρυθμιστικό είναι προς το παρόν σταθερά ρυθμισμένα να χρησιμοποιούν το `pactl`.

Το πακέτο που περιέχει το `pactl` συνήθως ονομάζεται `pulseaudio-utils` ή `libpulse`.

## Κλειδιά SSH

Μπορείτε να εισαγάγετε ή να δημιουργήσετε κλειδιά SSH στις ρυθμίσεις της εφαρμογής. Η σύνδεση με κλειδί SSH είναι πιο ασφαλής από τη χρήση κωδικών πρόσβασης.

Ο ευκολότερος τρόπος να εισαγάγετε ένα υπάρχον κλειδί SSH από υπολογιστή είναι να σαρώσετε έναν κωδικό QR. Μπορείτε να χρησιμοποιήσετε το πρόγραμμα `qrencode` για να δημιουργήσετε την εικόνα του κωδικού QR. Εκτελέστε μια εντολή όπως η παρακάτω για να δημιουργήσετε τον κωδικό QR:

```shell
# Μεταβείτε στα κλειδιά SSH σας:
cd ~/.ssh

# Εμφανίστε τον κωδικό QR στο τερματικό:
qrencode -r id_ed25519 -t ansiutf8

# Εναλλακτικά, δημιουργήστε ένα αρχείο εικόνας:
qrencode -r id_ed25519 -o qr.png

# Τα κλειδιά RSA 4096 bit είναι πολύ μεγάλα για κωδικό QR. Μπορείτε να χρησιμοποιήσετε gzip για να χωρέσει οριακά ένα:
cat id_rsa_4096 | gzip | base64 | qrencode -o qr.png
```

Μπορείτε να στείλετε δημόσια κλειδιά SSH σε έναν διακομιστή χρησιμοποιώντας τη λειτουργία `Αποστολή δημόσιου κλειδιού` στο μενού. Αυτό θα προσαρτήσει το επιλεγμένο κλειδί SSH στο αρχείο `~/.ssh/authorized_keys`. Έτσι μπορείτε εύκολα να μεταβείτε από σύνδεση με κωδικό πρόσβασης σε σύνδεση με κλειδί SSH.

Μπορείτε να εισάγετε και να χρησιμοποιήσετε κρυπτογραφημένα κλειδιά SSH, αλλά προς το παρόν δεν μπορείτε να δημιουργήσετε τέτοια μέσα από την εφαρμογή.

## Ασφάλεια

Δεν είναι δυνατό να εξαχθεί ή να αποσπαστεί το ιδιωτικό μέρος των κλειδιών SSH ή οι αποθηκευμένοι κωδικοί πρόσβασης από την εφαρμογή. Αυτά τα δεδομένα κρυπτογραφούνται με AES 256 bit και το κλειδί κρυπτογράφησης αποθηκεύεται στο Android Keystore. Τα κρυπτογραφημένα δεδομένα εξαιρούνται από τα αντίγραφα ασφαλείας Android.

Δεν υπάρχει λογισμικό αναφοράς σφαλμάτων σε αυτή την εφαρμογή. Δεν υπάρχει τηλεμετρία. Δεν υπάρχουν διαφημίσεις. Δεν υπάρχουν δικτυακά αιτήματα εκτός από τη σύνδεση SSH.

Η ασφάλεια αυτής της εφαρμογής δεν έχει ελεγχθεί με audit. Αν έχετε εμπειρία με την ασφάλεια Android ή την ασφάλεια SSH, παρακαλώ ρίξτε μια ματιά στον πηγαίο κώδικα και αναφέρετε τα ευρήματά σας σε αυτό το issue στο GitHub:

<https://github.com/stefansundin/SSHRemote/issues/1>
